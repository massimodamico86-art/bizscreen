# Requirements: BizScreen v2.3 Production Hardening

**Defined:** 2026-02-07
**Core Value:** Screens reliably display the right content at the right time, even when offline

## v2.3 Requirements

Requirements for production hardening milestone. Each maps to roadmap phases.

### E2E Test Stability

- [x] **TEST-01**: E2E test pass rate reaches 90%+ — achieved 92.7% (279/301)
- [x] **TEST-02**: Timeout-related test failures identified and resolved — 172 waitForTimeout removed
- [x] **TEST-03**: Flaky tests stabilized with proper waits and assertions — Promise.race + element waits
- [x] **TEST-04**: Test infrastructure issues fixed (shared state, setup/teardown) — custom fixtures
- [x] **TEST-05**: Test execution time is reasonable (no excessive slowdowns) — timeouts configured

### Error Monitoring

- [x] **MON-01**: Sentry SDK integrated for error tracking — React 19 hooks, Router v7 tracing
- [x] **MON-02**: Frontend errors captured with user context, route, and state — Sentry.setUser + breadcrumbs
- [x] **MON-03**: API errors captured with request context — Proxy-based Supabase instrumentation
- [x] **MON-04**: Alerting configured for critical errors — GitHub secrets set, Slack deferred
- [x] **MON-05**: Source maps uploaded for readable stack traces — @sentry/vite-plugin with hidden maps

### Feature Flag Cleanup

- [x] **FLAG-01**: OnboardingWizard component de-wired (file not deleted, confirmed broken)
- [x] **FLAG-02**: WelcomeModal legacy code de-wired (file not deleted)
- [x] **FLAG-03**: Obsolete localStorage keys — out of scope for flag cleanup, deferred
- [x] **FLAG-04**: VITE_USE_UNIFIED_ONBOARDING flag removed from env schema, config, components

## Future Requirements

Deferred to v2.4 or later. Tracked but not in current roadmap.

### Performance Optimization

- **PERF-01**: Bundle analysis baseline established
- **PERF-02**: Core Web Vitals measured and documented
- **PERF-03**: Performance budget defined and enforced

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| RTL language support | Deferred to v3.0 — requires UI/content mirroring |
| CJK language support | Deferred to v3.0 — font/rendering complexity |
| User template marketplace | Deferred to v3.0 — complex moderation/payment |
| TypeScript migration | Too disruptive; JavaScript works |
| New feature development | This is a hardening milestone |
