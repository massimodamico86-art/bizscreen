### Vite build + preview commands (from package.json)

- `npm run build` → `vite build` → produces `dist/` (minified JS/CSS, tree-shaken, no HMR runtime)
- `npm run preview` → `vite preview` → serves `dist/` on port 4173 by default (Vite's standard)

Operator can override port: `npx vite preview --port <port> --strictPort`.

### Playwright base URL override

`playwright.config.js` line 37 reads:
```js
baseURL: process.env.PLAYWRIGHT_BASE_URL || 'http://localhost:5173',
```

So `PLAYWRIGHT_BASE_URL=http://localhost:4173 npx playwright test ...` redirects the runner to the prod-preview server.

**CRITICAL:** `playwright.config.js` lines 67-78 has a `webServer` block that auto-starts `npm run dev` on port 5173 if not already running. To run against `npm run preview` on 4173:
- Either start `npm run preview` manually BEFORE running Playwright (preview MUST be running before Playwright launches)
- AND set `PLAYWRIGHT_BASE_URL=http://localhost:4173`
- The `webServer` block's `reuseExistingServer: true` will attempt to start dev on 5173 if 4173 isn't there — that's harmless (5173 stays idle), but ensure `PLAYWRIGHT_BASE_URL` is set so the runner uses 4173 not 5173.

### Post-180-07 ARIA caveat

180-07 added `role='gridcell'` to per-card wrappers. This MIGHT marginally increase render time (extra DOM attribute writes). In practice, the impact is negligible (1 string per card × ~500 cards = 500 attribute writes — sub-millisecond). The perf re-run should not measurably change due to this alone.

### Post-180-09 helpers caveat

180-09 added 4 new entries to closeButtonSelectors in dismissAnyModals. Each adds ~100ms to the dismissal loop in the worst case. The perf spec calls `loginAndPrepare` in beforeEach (helpers.js:23) which calls `dismissAnyModals` (helpers.js:63). The perf measurement starts AFTER login (line 48: `await page.evaluate(() => performance.mark('gallery-paint-start'))` is the first call after `loginAndPrepare`). So the new dismissAnyModals time does NOT count toward the first-paint measurement. The login itself becomes ~400ms slower in worst case, but that's outside the SC-2 budget window.
