### Current row structure in VirtualizedTemplateGrid.jsx (lines 82-141)

The outer `<div role="grid" aria-rowcount={rows.length} ...>` wraps the virtualizer; each virtual row is a `<div ... role="row" aria-rowindex={vRow.index + 1} ...>`; each per-card child inside that row is currently a bare `<div key={t.id} className="relative">` at line 112. Axe-core's `aria-required-children` rule fires here because `[role='grid']` expects `[role='row']` direct children with `[role='gridcell']` descendants, and the existing per-card wrapper has NO ARIA role at all.

Concrete excerpt of the per-card render block (lines 109-135):

```jsx
{row.map((t, idxInRow) => {
  const absoluteIndex = vRow.index * safeCols + idxInRow;
  return (
    <div key={t.id} className="relative">
      {isNew && isNew(t.created_at) && (
        <div className="absolute top-2 left-2 z-10">
          <Badge variant="success" size="sm">New</Badge>
        </div>
      )}
      {(t.use_count ?? 0) >= (popularityThreshold ?? 0) && (popularityThreshold ?? 0) > 0 && (
        <div className="absolute top-2 right-2 z-10">
          <Badge variant="default" size="sm">Popular</Badge>
        </div>
      )}
      <TemplateCard
        title={t.name}
        ...
      />
    </div>
  );
})}
```

The fix: add `role="gridcell"` to the per-card wrapper `<div>`. No other ARIA attributes needed — `aria-required-children` is satisfied by the presence of one descendant gridcell per row.

The fix: change `<h3 className="font-medium text-gray-900 text-sm truncate">{title}</h3>` to `<h4 className="font-medium text-gray-900 text-sm truncate">{title}</h4>` (open AND close tag).
