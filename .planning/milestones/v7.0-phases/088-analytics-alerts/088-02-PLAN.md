Fix critical layout and component issues in AlertsCenterPage:

**BUG 1: PageLayout misuse (CRITICAL)**
The page passes `title`, `description`, `icon`, and `actions` props directly to `<PageLayout>`:
```jsx
<PageLayout title={...} description={...} icon={Bell} actions={...}>
```
`PageLayout` does NOT accept these props — they are spread into `...props` on the wrapper div and silently ignored. The page renders NO header at all.

**Fix:** Restructure to use `PageLayout` > `PageHeader` + `PageContent` pattern:
1. Add `PageHeader`, `PageContent`, `Button`, `Card` to the design-system import (currently only `PageLayout` is imported)
2. Move the `title`, `description`, and `actions` to a `<PageHeader>` component inside `<PageLayout>`
3. Wrap all body content in `<PageContent>`
4. The `icon` prop on PageHeader should use the `icon` and `iconBackground` pattern: `icon={<Bell className="w-5 h-5 text-blue-600" />}` and `iconBackground="bg-blue-100"`

**BUG 2: Raw button elements instead of Button component**
The page uses raw `<button>` elements for:
- "Notification Settings" button in header actions
- "Refresh" button in header actions
- Bulk action buttons (Acknowledge, Resolve, Cancel)
- Individual alert action buttons (acknowledge, resolve, view details)
- Pagination previous/next buttons
- Clear filters link

**Fix:** Replace the header action buttons with design-system `<Button>`:
- Notification Settings: `<Button variant="secondary" onClick={() => onNavigate?.('notification-settings')}><Settings ... /> Notification Settings</Button>`
- Refresh: `<Button variant="ghost" onClick={loadAlerts} disabled={loading}><RefreshCw ... /> Refresh</Button>`
Leave inline action buttons (per-row acknowledge/resolve/details) and pagination buttons as raw elements since they are small icon-only buttons where Button component would add unnecessary size.

**BUG 3: Hand-rolled modal**
The `AlertDetailModal` uses a raw `<div className="fixed inset-0 z-50 ...">` overlay instead of the design-system `<Modal>` component.

**Fix:** Replace the hand-rolled modal with the design-system `<Modal>` component:
```jsx
<Modal isOpen={showDetailModal} onClose={onClose} title={alert.title} size="lg">
  {/* Keep all existing modal body content */}
  {/* Move footer actions inside modal body since Modal doesn't have a separate footer slot */}
</Modal>
```
Remove the manual overlay div, the manual close X button (Modal provides its own), and adapt the header styling.

**Implementation steps:**
1. Update the import from `../design-system` to include: `PageLayout, PageHeader, PageContent, Button, Card, Modal`
2. Restructure the return JSX: `<PageLayout>` wraps `<PageHeader>` (with title/description/actions) + `<PageContent>` (with summary cards, filters, bulk actions, alerts table, detail modal)
3. Replace the header action raw buttons with `<Button>` components

5. Keep the modal body content (message, details grid, recovery details, related items, meta data, resolution notes) intact
