## Overview

Rebuilt the template experience end-to-end — modernized gallery UI, strong discovery, polished preview→apply→customize flow, and a substantially expanded SVG content library — so first-time users find and ship great-looking screens fast. Closes the legacy three-source merge in `svgTemplateService.fetchSvgTemplates`, retires `SvgTemplateGalleryPage`, replaces sessionStorage handoffs with URL params, and adds first-class Starter Packs, Favorites, and a driver.js-powered first-visit tour.

## Milestone Summary

**Decimal Phases:**

- Phase 172.1: Fix SVG Apply RPC (inserted after Phase 172 to repair regression surfaced by live E2E UAT)

**Key Decisions:**

- Phase 170 (data layer) mandated before all UI phases — three-source merge bug and RLS gap are latent defects that become critical when new content is added
- Phase 175 (content + quality) depends only on Phase 170 (data layer), not on UI phases — could begin as soon as the VIEW and service were stable
- driver.js (MIT) chosen for onboarding tour over react-joyride (React 19 incompatible) and Shepherd.js (AGPL)
- fuse.js for client-side search — catalog fits in-memory at <500 records; no Algolia dependency
- sessionStorage template handoff replaced with URL params in Phase 172 — fragile in multi-tab scenarios
- Wave 0 RED tests committed BEFORE production code in every phase from 173 onward — Nyquist sampling continuity gate satisfied (no 3-task gap without automated verify)
- Atomic single-RPC apply pattern (migration 168 for Polotno, 170 for SVG) — closes clone-then-patch race
- Decimal phase numbering (172.1) for unplanned regression fix — preserves linear roadmap progression
- Migration 175 applied via direct Supabase Management API call — file too large (84 KB) for MCP `apply_migration` round-trip
- TemplateCard FavoriteButton wrapper given `z-20 pointer-events-auto` — hover overlay was intercepting clicks (correctness fix, not test-only workaround)
- Self-verify directive applied to Phase 173 visual UAT — automated E2E covers every SC structurally; no manual checkpoint required

**Issues Resolved:**

- Three-source JS merge in `svgTemplateService.fetchSvgTemplates` collapsed to single VIEW query
- Cross-tenant RLS gap on `svg_templates` closed via migration 167
- `LOCAL_SVG_TEMPLATES` hardcoded array purged from source; entries seeded to DB
- Legacy `SvgTemplateGalleryPage` deleted (no longer in import graph)
- `sessionStorage('pendingTemplate')` handoff removed end-to-end
- Modal inner-height bug surfaced by 172.1 E2E (Apply button below viewport at 1280×720) — fixed via `h-[calc(100vh-2rem)] sm:h-[90vh]`
- Two worktree-merge incidents recovered (different commits — no Phase 163 safeguard yet, but recovery patterns now reproducible)
- AuthContext post-reload hang in TFAV-01 persistence test — bypassed via cookie-clear + re-login (TFAV-03 pattern), follow-up tech-debt issue noted

**Issues Deferred (carried into v20.1+):**

- 11 manual UAT items across Phases 172 + 174 — visual proportions, mobile layout, native pickers, live editor-return round-trip, first-visit tour visual, onboarding starter_pack apply flow
- Phase 163 Worktree Merge Safeguard still not built — third milestone in a row touched by worktree-merge data loss (v18.0 + v19.0 + minor in v20.0)
- `content_templates` pack completeness audit — migration 110 (v19.0) touched `apply_pack_template`; not exercised in v20.0 packs work but worth verifying once
- Polotno QuickCustomize (TPRV-F1) — Polotno JSON mutation primitives still pending
- Gallery virtualization (TGAL-F1) — `@tanstack/react-virtual` installed but inactive; activate when catalog > ~200 templates
- Personalization (TPER-F1) — requires reliable `profiles.business_type` population

- TQAL-05 audit found Phase 171 TDSC-04 product-routing test, Phase 174 gallery-tour state test, and Phase 173 packs Layouts-button test as pre-existing failures (logged in `175/deferred-items.md`, not regressions of v20.0)
