### Phase 177: AI Generation Pipeline + Admin Queue UI
**Goal**: Admins can generate SVG templates from a natural-language prompt, have validator failures auto-retried with error feedback to the LLM, and review pending drafts in a queue UI — with the guarantee that no malformed or unvalidated SVG ever reaches the queue
**Depends on**: Phase 176
**Requirements**: TGEN-01, TGEN-02, TGEN-03, TGEN-04, TGEN-05, TGEN-06, TADM-01, TADM-02, TADM-03, TADM-04
**Success Criteria** (what must be TRUE):
  1. An admin submits a prompt; the Edge Function calls the LLM, runs `svgValidator` on the raw LLM output **before writing to `template_drafts`** — a unit/integration test asserts `validateSvg()` is called prior to any INSERT; a malformed SVG (injected as mock LLM response) is rejected at ingest and never appears in the queue
  2. When `svgValidator` fails on attempt 1, the pipeline retries up to 2 times with the validator's specific error messages fed back into the LLM prompt; after 2 consecutive failures the draft is marked `needs_human_review` with the failure reasons logged — verified by a test that mocks two successive validator failures
  3. The `ANTHROPIC_MODEL_ID` (or equivalent) env var — not a hardcoded string literal — is the sole model reference in the Edge Function; a Playwright or CI smoke test asserts ≥1 canonical prompt produces a structurally valid SVG (viewBox present, no `currentColor`, ≥1 colored element)
  4. A non-admin user attempting to call the generation Edge Function receives a 403; no LLM API key is present in the browser-side bundle (verified by `grep -r "ANTHROPIC" dist/` finding zero matches)
  5. `AdminTemplateQueuePage` lists pending drafts with rendered thumbnail and originating prompt; Approve rasterizes the PNG via `@resvg/resvg-js`, uploads to S3, flips `published = true`, and the template appears in `gallery_templates` within the same request; Reject removes the draft with an audit trail; Edit navigates to `AdminEditTemplatePage`; page is inaccessible to non-admin users
  6. **Per-template-type prompt library (TGEN-06):** at least 6 curated base prompts exist as version-controlled assets — one per primary template type (Menu / Promo / Announcement / Reminder / Wayfinding / Health Tip) — each covering the three named verticals (Restaurants/Retail/Healthcare). The admin UI exposes a template-type selector; the Edge Function prepends the matching base prompt to the admin's freeform input before calling the LLM. An A/B comparison test (5 generations per template type, with vs without the base prompt) demonstrates measurably higher first-pass `svgValidator` success when the base prompt is applied — exact threshold (e.g., ≥30 percentage-point improvement) to be set during phase planning. Pattern observed in OptiSigns AI Designer (`/ai`) where each "Explore AI Prompts" example is tagged with a category — see `.planning/research/OPTISIGNS-WALKTHROUGH.md`.
**Plans**: 11 plans (6 original + 5 gap-closure 180-07..180-11)
- [x] 177-01-PLAN.md — Wave 0: Edge Function spike + 5 RED tests + Docker/CLI prereqs (resolves D-17 Deno DOMParser landmine)
- [x] 177-02-PLAN.md — Wave 1: Prompt library (JSON + JS parity) + generate handler (validator-at-ingest + retry-with-feedback) + frontend service layer
- [x] 177-03-PLAN.md — Wave 2: Approve handler (rasterize via resvg-wasm + S3 PUT + INSERT svg_templates + UPDATE draft) + reject handler
- [x] 177-04-PLAN.md — Wave 3: AdminTemplateQueuePage shell + Pending tab (sanitized previews + 3 row actions) + App.jsx route registration
