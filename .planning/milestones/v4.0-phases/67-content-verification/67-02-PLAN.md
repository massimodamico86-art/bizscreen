**1. Create `src/player/hooks/useContentVerification.js`:**

A hook that manages content verification state and provides transition-aware re-sync:

```javascript
import { useRef, useCallback, useEffect } from 'react';
import { useLogger } from '../../hooks/useLogger.js';
import { calculateBackoff } from '../../services/playerService';
```

**Hook signature:**
```javascript
export function useContentVerification({ screenId, loadContentRef, content })
```

**State (all refs, no useState to avoid re-renders):**
- `pendingSyncRef` (boolean) -- whether a re-sync is queued
- `retryCountRef` (number) -- current retry attempt count
- `MAX_SYNC_RETRIES = 3`

**Exposed functions:**

a. `onMismatchDetected(expectedVersion, reportedVersion)` -- Called by heartbeat when mismatch detected. Sets `pendingSyncRef.current = true` if not already pending. Logs the mismatch. Does NOT trigger any immediate action (play-then-verify per VERI-04).

b. `checkAndSync()` -- Called at playlist item transition points (between items). If `pendingSyncRef.current` is false, return false (no sync needed). If true, attempt content reload via `loadContentRef.current?.(screenId, false)`.
   - On success: reset `pendingSyncRef` and `retryCountRef` to 0, log success, return true.
   - On failure: increment `retryCountRef`. If retries >= MAX_SYNC_RETRIES, log warning ("retries exhausted, keeping current content"), reset both refs (give up gracefully -- stale content is better than blank). Return false.
   - Use `calculateBackoff(retryCountRef.current)` to determine delay before retry, but since this runs at transition points (not in a loop), the backoff is implicit via the heartbeat cycle detecting mismatch again on the next beat.

c. `hasPendingSync` -- getter that returns `pendingSyncRef.current`

**Auto-reset effect:** When content identity changes (content?.mode, content?.source, content?.playlist?.id, content?.layout?.id), reset `pendingSyncRef` and `retryCountRef` to 0. This handles the case where content was successfully reloaded through another path (manual publish, needs_refresh push, etc.).

**1. In `src/services/screenDiagnosticsService.js`:**

Add a `forceDeviceSync` exported function that triggers a content reload on the device by setting `needs_refresh = true`:

```javascript
/**
 * Force a content sync on a specific device by setting needs_refresh flag.
 * The player will reload content on its next heartbeat cycle.
 * @param {string} deviceId - The device UUID
 * @returns {Promise<void>}
 */
export async function forceDeviceSync(deviceId) {
  const { error } = await supabase
    .from('tv_devices')
    .update({
      needs_refresh: true,
      content_version_status: 'pending',
    })
    .eq('id', deviceId);
  if (error) throw error;
}
```

Import supabase at the top if not already imported. This sets `needs_refresh = true` (which the player checks on every heartbeat) and transitions the status to 'pending' so the mismatch warning clears immediately in the UI.
