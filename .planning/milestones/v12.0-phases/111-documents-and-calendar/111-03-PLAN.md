Calendar OAuth tokens DB table (created in 111-01 migration 164):

CREATE TABLE public.calendar_oauth_tokens (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  owner_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE NOT NULL,
  provider TEXT NOT NULL CHECK (provider IN ('gcal', 'outlook_cal')),
  calendar_id TEXT NOT NULL,
  label TEXT,
  access_token TEXT NOT NULL,
  refresh_token TEXT,
  token_expiry TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE(owner_id, provider, calendar_id)
);

Following the `googleDriveService.js` pattern exactly:

1. Imports: `CLOUD_PROVIDERS, generateCodeVerifier, generateCodeChallenge, saveOAuthState, validateOAuthState, saveTokens, getTokens, clearTokens, isTokenExpired` from `./cloudOAuthService`; `createScopedLogger` from `../loggingService`

2. Constants:
   - `PROVIDER = CLOUD_PROVIDERS.GOOGLE_CALENDAR` (= `'gcal'`)
   - `GOOGLE_CALENDAR_CLIENT_ID = import.meta.env.VITE_GOOGLE_CALENDAR_CLIENT_ID || ''`
   - `GOOGLE_OAUTH_URL = 'https://accounts.google.com/o/oauth2/v2/auth'`
   - `GOOGLE_TOKEN_URL = 'https://oauth2.googleapis.com/token'`
   - `GOOGLE_SCOPES = 'https://www.googleapis.com/auth/calendar.events.readonly'`

3. `startGoogleCalendarOAuth()`: Generate PKCE verifier and challenge; generate random state via `crypto.randomUUID()`; save state via `saveOAuthState(PROVIDER, { codeVerifier, state })`; build URL params: `response_type=code`, `client_id`, `redirect_uri=${window.location.origin}/auth/cloud/callback?provider=${PROVIDER}`, `scope=GOOGLE_SCOPES`, `state`, `code_challenge`, `code_challenge_method=S256`, `access_type=offline`, `prompt=consent`; redirect via `window.location.href`.

4. `handleGoogleCalendarCallback(code, state)`: Validate state via `validateOAuthState(PROVIDER, state)` -- throws if invalid. Exchange code for tokens via `fetch(GOOGLE_TOKEN_URL, ...)`. **IMPORTANT: Do NOT call `saveTokens(PROVIDER, ...)` from cloudOAuthService -- calendar tokens must be persisted to the database, not localStorage. Instead, return `{ accessToken: data.access_token, refreshToken: data.refresh_token, expiresIn: data.expires_in }` so the caller (App.jsx callback handler) can pass them to `calendarService.saveCalendarSource()`.** This is the key difference from the cloud storage OAuth pattern.

5. `refreshGoogleCalendarToken()`: Get stored tokens via `getTokens(PROVIDER)`. If no refresh token, throw. POST to `GOOGLE_TOKEN_URL` with `grant_type=refresh_token`. Save updated tokens. Return new access token. NOTE: This function is only used as a fallback for the editor UI; the calendar-proxy Edge Function handles its own token refresh server-side.

6. `getGoogleCalendarToken()`: Get tokens, check expiry via `isTokenExpired(PROVIDER)`. If expired, call `refreshGoogleCalendarToken()`. Return access token.

7. `disconnectGoogleCalendar()`: `clearTokens(PROVIDER)`

8. `isGoogleCalendarConnected()`: Return `!!getTokens(PROVIDER)?.accessToken`

Export all functions.

Service for managing calendar connections and fetching events:

1. Imports: `supabase` from `../supabase`; `createScopedLogger` from `./loggingService`

2. `saveCalendarSource({ provider, calendarId, label, accessToken, refreshToken, tokenExpiry })`:
   - Get current user: `(await supabase.auth.getUser()).data.user.id`
   - Upsert into `calendar_oauth_tokens` table with columns: `owner_id`, `provider`, `calendar_id`, `label`, `access_token`, `refresh_token`, `token_expiry`, `updated_at: new Date().toISOString()`

3. `getCalendarSources()`: Select from `calendar_oauth_tokens` where owner_id = current user. Return array of `{ id, provider, calendarId: calendar_id, label, created_at }`. Do NOT return access_token or refresh_token to the UI.

5. `fetchCalendarEvents(sourceId)`: Invoke the `calendar-proxy` Edge Function: `await supabase.functions.invoke('calendar-proxy', { body: { action: 'fetch', sourceId } })`. Return `data.data` (the events array) or throw on error.

6. `fetchAllCalendarEvents(sourceIds)`: `Promise.all(sourceIds.map(id => fetchCalendarEvents(id)))`. Merge all events arrays, sort by `startTime` ascending. Return merged and sorted array.

**File: `supabase/functions/calendar-proxy/index.ts`** (CREATE)

Following the `rss-proxy/index.ts` pattern closely:

1. Imports: `createClient` from `npm:@supabase/supabase-js@2`; `corsHeaders` from `../_shared/cors.ts`

2. Constants: `CACHE_TTL_MINUTES = 5`, `FETCH_TIMEOUT_MS = 10_000`, `MAX_EVENTS = 50`, `EVENT_WINDOW_DAYS = 7`

3. Helper `jsonResponse(body, status)` -- same as rss-proxy

4. `normalizeGoogleEvent(event)`: Map Google Calendar event to common shape: `{ id, title: event.summary || '(No title)', description: event.description || '', startTime: event.start?.dateTime || event.start?.date, endTime: event.end?.dateTime || event.end?.date, allDay: !event.start?.dateTime, location: event.location || null }`

5. `normalizeOutlookEvent(event)`: Map Microsoft Graph event to same common shape: `{ id, title: event.subject || '(No title)', description: event.bodyPreview || '', startTime: event.start?.dateTime, endTime: event.end?.dateTime, allDay: event.isAllDay || false, location: event.location?.displayName || null }`

6. `fetchGoogleCalendarEvents(calendarId, accessToken)`: Build URL `https://www.googleapis.com/calendar/v3/calendars/${encodeURIComponent(calendarId)}/events` with query params: `timeMin` (now ISO), `timeMax` (now + EVENT_WINDOW_DAYS ISO), `singleEvents=true`, `orderBy=startTime`, `maxResults=MAX_EVENTS`. Fetch with `Authorization: Bearer ${accessToken}`, timeout via AbortController. Parse JSON, map `data.items` through `normalizeGoogleEvent`. Return normalized events array.

7. `fetchOutlookCalendarEvents(accessToken)`: Build URL `https://graph.microsoft.com/v1.0/me/calendarView` with query params: `startDateTime` (now ISO), `endDateTime` (now + EVENT_WINDOW_DAYS ISO), `$top=MAX_EVENTS`, `$orderby=start/dateTime`, `$select=subject,start,end,location,isAllDay,bodyPreview`. Header: `Authorization: Bearer ${accessToken}`, `Prefer: outlook.timezone="UTC"`. Parse JSON, map `data.value` through `normalizeOutlookEvent`.

8. `handleFetchEvents(supabaseAdmin, userId, params: { sourceId })`:
   - Look up `calendar_oauth_tokens` by `id = sourceId` AND `owner_id = userId` (RLS + explicit check)
   - If not found, return 404 error
   - Extract `provider, calendar_id, access_token, refresh_token`
   - **Cache check:** Query `calendar_event_cache` where `calendar_id` and `provider` match, `expires_at > now()`. If hit, return cached events with `meta: { cached: true }`
   - **Fetch from provider:** If `provider === 'gcal'`: call `fetchGoogleCalendarEvents(calendar_id, access_token)`. If `provider === 'outlook_cal'`: call `fetchOutlookCalendarEvents(access_token)`. On 401 response: attempt token refresh (see below), retry once. On other error: return 502 with error details.
   - **Token refresh on 401:** For Google: POST to `https://oauth2.googleapis.com/token` with `grant_type=refresh_token`, `refresh_token`, `client_id` from env. For Outlook: POST to `https://login.microsoftonline.com/common/oauth2/v2.0/token` with same pattern. Update `calendar_oauth_tokens` **in the database** with new `access_token` and `token_expiry`. Retry the event fetch with the new token.
   - **Cache write:** Upsert into `calendar_event_cache` with `calendar_id, provider, events, fetched_at, expires_at`
   - Return events with `meta: { cached: false, eventCount }`

9. Main `Deno.serve` handler: CORS preflight (same as rss-proxy). JWT auth verification (same as rss-proxy). Extract `userId` from verified user. Parse body: `{ action, ...params }`. Route: `action === 'fetch'` -> `handleFetchEvents(supabaseAdmin, userId, params)`. Default: 400 error for unknown action.

Environment variables needed (from Deno.env): `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `GOOGLE_CALENDAR_CLIENT_ID` (or `VITE_GOOGLE_CALENDAR_CLIENT_ID`), `MICROSOFT_CALENDAR_CLIENT_ID` (or `VITE_MICROSOFT_CALENDAR_CLIENT_ID`).
