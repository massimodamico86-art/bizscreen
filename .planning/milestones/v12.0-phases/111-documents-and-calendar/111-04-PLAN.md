**File: `src/player/components/widgets/CalendarWidget.jsx`** (CREATE)

Component: `CalendarWidget({ props = {} })`

Props from registry defaultProps:
- `sources` (array of `{ id, provider, calendarId, label }`) -- calendar source references
- `refreshIntervalMinutes` (number, default 5) -- minutes between event refreshes
- `maxEvents` (number, default 10) -- max events to display
- `showEndTime` (boolean, default true) -- show end time for events
- `textColor` (string, default '#ffffff')
- `accentColor` (string, default '#3b82f6') -- color for time indicators and current/next event highlight
- `theme` (string, default 'dark') -- 'dark' or 'light'

Implementation:

1. State: `events` (array), `loading` (boolean), `error` (string|null), `lastRefresh` (Date|null)

2. **Event fetching effect:** Extract `sourceIds` from `props.sources` (map to `s.id`). If no sourceIds, set events to empty and return. Call the calendar-proxy Edge Function directly (player may not have calendarService access):
```javascript
const results = await Promise.all(
  sourceIds.map(sourceId =>
    supabase.functions.invoke('calendar-proxy', {
      body: { action: 'fetch', sourceId },
    })
  )
);
```
Merge all events, sort by startTime, slice to `maxEvents`. Set `events` state, clear loading. On error: set error message, keep previous events (graceful degradation).

3. **Auto-refresh effect:** Set interval at `refreshIntervalMinutes * 60 * 1000`. Re-run the fetch on each tick. Clean up interval on unmount.

4. **Render:**
   - Container: full width/height, `overflow-y: auto`, padding 16px
   - If loading (initial): show centered spinner with "Loading events..." text
   - If error and no events: show centered error icon with message
   - If no sources: show centered `CalendarDays` icon with "No calendars connected" text
   - If no events (but sources connected): show "No upcoming events" message
   - **Event list:** Each event as a row. Left column (accent-colored time block): All-day events show "ALL DAY" badge with accent background. Timed events show start time formatted as "HH:mm" (or "h:mm A" for 12h). If `showEndTime`, show end time below start in smaller text. Right column (event details): Title in `textColor`, bold, truncated to 2 lines. Location (if present) in dimmed text with MapPin icon, truncated to 1 line. Divider line between events.
   - **Date grouping:** Group events by date. Show date headers: "Today" for current date, "Tomorrow" for next day, "Wed, Mar 5" format for other dates. Date headers in accent color, smaller text, with subtle top border.
   - **Styling:** Dark theme: `bg-transparent` (inherits from zone), white text. Light theme: same layout but with dark text colors. Font sizes: date header 12px, time 14px bold, title 16px, location 12px. Smooth scroll for long event lists.

5. **Time formatting helper:** `formatEventTime(isoString, allDay)` -- returns formatted time string. Handle both ISO datetime and date-only strings (all-day events). Use `Intl.DateTimeFormat` for locale-aware formatting.

Import: `useState, useEffect` from react; `CalendarDays, MapPin, AlertCircle` from lucide-react; `supabase` from `../../../supabase`.
