Create migration `164_document_conversion_calendar_cache.sql` that adds the calendar support tables needed by both document and calendar features.

The document conversion metadata uses the existing `config_json` JSONB column on `media_assets` (no schema change needed). But the calendar feature (Plan 111-03/04) needs two new tables, and we create them here so both wave-1 plans share a single migration.

Add:

1. `calendar_event_cache` table:
   - `id UUID PRIMARY KEY DEFAULT uuid_generate_v4()`
   - `calendar_id TEXT NOT NULL` -- the external calendar identifier
   - `provider TEXT NOT NULL CHECK (provider IN ('gcal', 'outlook_cal'))`
   - `events JSONB NOT NULL DEFAULT '[]'::jsonb` -- cached normalized events
   - `fetched_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
   - `expires_at TIMESTAMPTZ NOT NULL` -- TTL for cache invalidation
   - `UNIQUE(calendar_id, provider)`
   - Index: `idx_calendar_cache_expiry ON calendar_event_cache(calendar_id, provider, expires_at)`

2. `calendar_oauth_tokens` table:
   - `id UUID PRIMARY KEY DEFAULT uuid_generate_v4()`
   - `owner_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE NOT NULL`
   - `provider TEXT NOT NULL CHECK (provider IN ('gcal', 'outlook_cal'))`
   - `calendar_id TEXT NOT NULL` -- external calendar ID
   - `label TEXT` -- user-friendly name
   - `access_token TEXT NOT NULL`
   - `refresh_token TEXT`
   - `token_expiry TIMESTAMPTZ`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
   - `updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
   - `UNIQUE(owner_id, provider, calendar_id)`
   - RLS enabled. Policy: `"Users can manage own calendar tokens" FOR ALL USING (owner_id = auth.uid())`
   - Index: `idx_calendar_tokens_owner ON calendar_oauth_tokens(owner_id, provider)`

**File: `src/services/documentService.js`** (CREATE)

1. `DOCUMENT_MIME_TYPES` constant array listing all 7 accepted MIME types (PDF + 6 Office variants)

2. `isDocumentMimeType(mimeType)` -- returns boolean, checks against the array

3. `uploadDocument(file, { onStatusChange })`:
   - Upload file to Supabase Storage at path `documents/originals/${userId}/${timestamp}-${file.name}`
   - Get the public URL via `supabase.storage.from('media').getPublicUrl(path)`
   - Create a `media_assets` record with: `name` (filename without extension), `type: MEDIA_TYPES.DOCUMENT`, `url` (public URL), `mime_type: file.type`, `file_size: file.size`, `config_json: { conversionStatus: 'pending', convertedPages: [], pageCount: 0, originalFormat: getFileExtension(file.name) }`
   - Call `supabase.functions.invoke('doc-converter', { body: { mediaId: asset.id, storageUrl: publicUrl, mimeType: file.type } })`
   - Return the media asset record immediately (conversion is async)
   - Call `onStatusChange?.('uploading')` then `onStatusChange?.('converting')` at appropriate points

4. `getConversionStatus(mediaId)`: Fetch media asset by ID, return `{ status, pageCount, convertedPages, error }` from `config_json`

5. `pollConversionStatus(mediaId, { onComplete, onError, maxAttempts = 30, intervalMs = 2000 })`: Poll `getConversionStatus` with exponential backoff starting at `intervalMs`, capped at 15000ms. Call `onComplete(result)` when `conversionStatus === 'complete'`. Call `onError(error)` when `conversionStatus === 'error'` or max attempts reached. Return a cancel function `() => clearTimeout(timeoutId)`.

Import `supabase` from `../supabase`, `MEDIA_TYPES` and `getFileExtension` from `./mediaService.js`, and `createScopedLogger` from `./loggingService`.

**File: `supabase/functions/doc-converter/index.ts`** (CREATE)

Structure following `rss-proxy/index.ts` pattern:

1. Import `createClient` from `npm:@supabase/supabase-js@2` and `corsHeaders` from `../_shared/cors.ts`

2. Main `Deno.serve` handler with: CORS preflight handling, JWT auth verification, parse body: `{ mediaId, storageUrl, mimeType }`

3. Conversion logic in a try/catch:
   a. Download the document from `storageUrl` using `fetch()`
   b. For Office files (`mimeType !== 'application/pdf'`): Call Gotenberg API (URL from `Deno.env.get('GOTENBERG_URL') || 'http://localhost:3000'`). POST to `${gotenbergUrl}/forms/libreoffice/convert`. If Gotenberg unavailable, update media asset with `conversionStatus: 'error'` and `conversionError: 'Office conversion service unavailable.'`
   c. For PDF: Store the original PDF URL as the single "page" in `convertedPages` and set `conversionStatus: 'complete'`. Add TODO comment noting where magick-wasm page splitting will be integrated.
   d. Upload each converted page image to `documents/pages/${mediaId}/page-${index.toString().padStart(3, '0')}.png`
   e. Update `media_assets` record: On success: `config_json = { conversionStatus: 'complete', convertedPages: [...pageUrls], pageCount, originalFormat }`. Set `thumbnail_url` to first page URL. On error: `config_json = { conversionStatus: 'error', conversionError: err.message }`

4. Helper `jsonResponse(body, status)` and `updateMediaAsset(supabaseAdmin, mediaId, configJson, thumbnailUrl?)` functions
