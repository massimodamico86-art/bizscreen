**File: `src/player/components/widgets/DocumentWidget.jsx`** (CREATE)

Component: `DocumentWidget({ props = {} })`

Props from registry defaultProps:
- `mediaId` (string) -- the media asset ID containing converted pages
- `pageIntervalSeconds` (number, default 10) -- seconds between page advances
- `loop` (boolean, default true) -- loop back to first page after last
- `transition` (string, default 'crossfade') -- transition type between pages

Implementation:

1. State: `pages` (array of URLs), `currentPage` (index), `prevPage` (index for crossfade)

2. Effect to load pages: When `mediaId` changes, fetch the media asset via `supabase.from('media_assets').select('config_json').eq('id', mediaId).maybeSingle()`. Extract `config_json.convertedPages` array. Set `pages` state. If `convertedPages` is empty or `conversionStatus !== 'complete'`, show a "Converting..." placeholder.

3. Effect for auto-advance: When `pages.length > 1`, create `setInterval` at `pageIntervalSeconds * 1000`. On each tick: set `prevPage` to current, increment `currentPage`. If at end and `loop`, reset to 0. If at end and not `loop`, clear interval.

4. Render:
   - Container `div` with `width: 100%, height: 100%, position: relative, overflow: hidden, backgroundColor: '#000'`
   - If no pages: show centered placeholder with `FileText` icon and "No document selected" text (gray on black)
   - If pages but `conversionStatus === 'pending'`: show centered "Converting document..." with a spinner
   - If pages available: Current page `<img>` with `src={pages[currentPage]}`, `objectFit: 'contain'`, full size, `position: absolute`, `opacity: 1`, `transition: 'opacity 500ms ease'`. Previous page `<img>` (if `prevPage !== currentPage`) with `opacity: 0`, same transition -- this creates the crossfade.
   - Page indicator in bottom-right corner: `{currentPage + 1} / {pages.length}` in a semi-transparent rounded badge

Import `useState, useEffect` from react. Import `supabase` from `../../../supabase`. Import `FileText` from `lucide-react` for the empty state icon. Do NOT import from documentService -- the widget reads directly from supabase to avoid circular dependencies in the player bundle.
