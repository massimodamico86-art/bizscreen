  <action>
Add to `src/services/playlistService.js`:

1. **New export function `addNestedPlaylist(playlistId, nestedPlaylistId, duration = null)`:**
   - Fast path: if `playlistId === nestedPlaylistId`, throw `new Error('Cannot add a playlist to itself')`.
   - Call `supabase.rpc('check_playlist_nesting_valid', { p_parent_id: playlistId, p_child_id: nestedPlaylistId })`.
   - If error, throw it. If `data === false`, throw `new Error('Adding this playlist would create a circular reference or exceed the 5-level nesting limit')`.
   - Call `addPlaylistItem(playlistId, { itemType: 'playlist', itemId: nestedPlaylistId, duration })` and return the result.

2. **New export function `getNestedPlaylistInfo(playlistId)`:**
   - Fetches the nested playlist's name and item count for display in the editor strip.
   - `const { data } = await supabase.from('playlists').select('id, name, playlist_items(count)').eq('id', playlistId).single();`
   - Returns `{ id: data.id, name: data.name, itemCount: data.playlist_items?.[0]?.count || 0 }`.

3. **Update the JSDoc `@typedef PlaylistItem`** at the top of the file:
   - Change `@property {'media'|'app'} item_type` to `@property {'media'|'app'|'layout'|'web_page'|'playlist'} item_type`.

Do NOT modify the existing `addPlaylistItem` function -- it already works for any item_type string. The `addNestedPlaylist` wrapper adds the validation layer on top.
