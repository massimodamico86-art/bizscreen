  <action>
Create migration file `supabase/migrations/156_nested_playlists_audio_working_hours.sql` with these sections:

**Section 1: Extend playlist_items item_type CHECK constraint**
- Drop existing CHECK constraint: `ALTER TABLE playlist_items DROP CONSTRAINT IF EXISTS playlist_items_item_type_check;`
- Add new CHECK: `ALTER TABLE playlist_items ADD CONSTRAINT playlist_items_item_type_check CHECK (item_type IN ('media', 'app', 'layout', 'web_page', 'playlist'));`

**Section 2: Create circular reference prevention trigger**
- Create function `check_playlist_nesting()` as BEFORE INSERT/UPDATE trigger on playlist_items.
- Only activates when `NEW.item_type = 'playlist'`.
- Fast path: reject `NEW.item_id = NEW.playlist_id` (direct self-reference) with error 'Cannot add a playlist to itself'.
- Walk ancestry using recursive CTE: start from `NEW.playlist_id`, walk UP through `playlist_items` rows where `item_type = 'playlist'` to find all ancestor playlists. If `NEW.item_id` appears in ancestry, raise exception 'Circular playlist reference detected' with ERRCODE 'P0001'.
- Walk descendants: start from `NEW.item_id`, walk DOWN through nested playlists. Compute total depth = ancestry depth + descendant depth.
- If total depth > 5, raise exception 'Playlist nesting depth exceeds maximum of 5 levels' with ERRCODE 'P0002'.
- Safety brake: `AND depth < 6` in both CTEs to prevent runaway recursion.
- Create trigger: `CREATE TRIGGER check_playlist_nesting_trigger BEFORE INSERT OR UPDATE ON playlist_items FOR EACH ROW EXECUTE FUNCTION check_playlist_nesting();`
- Add index: `CREATE INDEX IF NOT EXISTS idx_playlist_items_nesting ON playlist_items(playlist_id, item_type) WHERE item_type = 'playlist';`

**Section 3: Add background audio columns to playlists**
- `ALTER TABLE playlists ADD COLUMN IF NOT EXISTS background_audio_id UUID REFERENCES media_assets(id) ON DELETE SET NULL;`
- `ALTER TABLE playlists ADD COLUMN IF NOT EXISTS background_audio_volume INTEGER DEFAULT 100 CHECK (background_audio_volume >= 0 AND background_audio_volume <= 100);`
- Index: `CREATE INDEX IF NOT EXISTS idx_playlists_background_audio ON playlists(background_audio_id) WHERE background_audio_id IS NOT NULL;`

**Section 4: Add working_hours to tv_devices**
- `ALTER TABLE tv_devices ADD COLUMN IF NOT EXISTS working_hours JSONB DEFAULT NULL;`
- Add comment: `COMMENT ON COLUMN tv_devices.working_hours IS 'Per-day working hours schedule. Shape: {"0":{"enabled":true,"start":"08:00","end":"18:00"},...,"6":{...}} where keys are DOW (0=Sun..6=Sat). NULL = always on.';`

**Section 5: Create validation RPC for service-layer pre-check**
- Create function `check_playlist_nesting_valid(p_parent_id UUID, p_child_id UUID) RETURNS BOOLEAN` that runs the same ancestry/descendant check as the trigger but returns false instead of raising an exception. This gives the service layer a fast pre-check before attempting the insert.
- Grant execute to authenticated.
