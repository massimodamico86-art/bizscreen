<task type="auto">
  <name>Task 1: Database migration for Canva tokens and video wall tables</name>
  <files>supabase/migrations/165_canva_tokens_video_walls.sql</files>
  <action>
Create migration 165 with three tables and RLS policies:

1. **canva_oauth_tokens** table:
   - id UUID PK default gen_random_uuid()
   - user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE
   - tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE
   - access_token TEXT NOT NULL
   - refresh_token TEXT
   - token_expiry TIMESTAMPTZ NOT NULL
   - scopes TEXT[]
   - created_at TIMESTAMPTZ DEFAULT now()
   - updated_at TIMESTAMPTZ DEFAULT now()
   - UNIQUE(user_id)
   - RLS: "Users manage own Canva tokens" ON canva_oauth_tokens FOR ALL USING (user_id = auth.uid())

2. **video_walls** table:
   - id UUID PK default gen_random_uuid()
   - tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE
   - name TEXT NOT NULL
   - rows INTEGER NOT NULL CHECK (rows >= 1 AND rows <= 10)
   - cols INTEGER NOT NULL CHECK (cols >= 1 AND cols <= 10)
   - bezel_gap_x NUMERIC DEFAULT 0 (horizontal bezel gap in mm)
   - bezel_gap_y NUMERIC DEFAULT 0 (vertical bezel gap in mm)
   - playlist_id UUID REFERENCES playlists(id) ON DELETE SET NULL
   - is_active BOOLEAN DEFAULT true
   - created_at TIMESTAMPTZ DEFAULT now()
   - updated_at TIMESTAMPTZ DEFAULT now()
   - RLS: tenant members manage via user_profiles tenant_id match

3. **video_wall_screens** junction table:
   - id UUID PK default gen_random_uuid()
   - wall_id UUID NOT NULL REFERENCES video_walls(id) ON DELETE CASCADE
   - screen_id UUID NOT NULL REFERENCES tv_devices(id) ON DELETE CASCADE
   - row_position INTEGER NOT NULL CHECK (row_position >= 0)
   - col_position INTEGER NOT NULL CHECK (col_position >= 0)
   - is_leader BOOLEAN DEFAULT false
   - created_at TIMESTAMPTZ DEFAULT now()
   - UNIQUE(wall_id, row_position, col_position)
   - UNIQUE(wall_id, screen_id)
   - RLS: via wall_id join to video_walls tenant_id

Enable RLS on all three tables. Grant SELECT, INSERT, UPDATE, DELETE to authenticated role on all three.

<task type="auto">
  <name>Task 2: canva-proxy Edge Function for server-side Canva API calls</name>
  <files>supabase/functions/canva-proxy/index.ts</files>
  <action>
Create the canva-proxy Edge Function following the exact pattern of calendar-proxy/index.ts. The function handles 4 actions:

1. **exchange_token**: Receives {code, codeVerifier, redirectUri}. Calls Canva token endpoint (POST https://api.canva.com/rest/v1/oauth/token) with grant_type=authorization_code, code, code_verifier, redirect_uri, client_id (from CANVA_CLIENT_ID env), client_secret (from CANVA_CLIENT_SECRET env). Stores tokens in canva_oauth_tokens table (upsert on user_id). Returns {ok: true}.

2. **list_designs**: Reads user's Canva token from DB. Refreshes if expired (token_expiry < now). Calls GET https://api.canva.com/rest/v1/designs with query params {query, continuation, ownership=owned, sort_by=modified_descending}. On 401, refresh token and retry (same pattern as calendar-proxy). Returns {ok: true, data: {items, continuation}}.

3. **export_design**: Receives {designId, format='png'}. Calls POST https://api.canva.com/rest/v1/exports with body {design_id: designId, format: {type: format}}. Polls GET https://api.canva.com/rest/v1/exports/{exportId} every 2 seconds, max 15 attempts (30s total). Returns {ok: true, data: {urls}} on success. Returns job ID if still in_progress after timeout for client-side polling.

4. **check_connection**: Checks if user has a valid (non-expired) Canva token in DB. Returns {ok: true, data: {connected: boolean}}.

Token refresh logic: If token_expiry < now() + 5 minutes, POST to https://api.canva.com/rest/v1/oauth/token with grant_type=refresh_token, refresh_token, client_id, client_secret. Update DB row with new tokens.

Use corsHeaders from ../_shared/cors.ts. JWT auth verification identical to calendar-proxy pattern. jsonResponse helper for consistent response format.

Environment variables needed: CANVA_CLIENT_ID, CANVA_CLIENT_SECRET (set via supabase secrets).
