<objective>
Wire the document upload pipeline so that PDF and Office files uploaded via the media library are routed through `documentService.uploadDocument()` (which uploads to Supabase Storage, creates the media_assets record with conversion status, and invokes the doc-converter Edge Function) instead of the generic S3 upload path.

Purpose: Without this wiring, document files go through the generic S3 upload and are saved as regular media assets with no server-side conversion. The doc-converter pipeline exists (Phase 111) but is never triggered by the upload flow. WebOS/Tizen smart TVs cannot render raw PDFs/Office files -- they need pre-converted PNG images.

Output: Modified `useS3Upload.jsx` that intercepts document MIME types before S3 upload, and updated `useMediaLibrary.js` that handles document upload results with conversion status feedback.
</objective>

2. In `handleFileSelectImpl`, inside the `for (const file of fileArray)` loop, after the `validateFile()` check (line ~94) and BEFORE the metadata gathering + `uploadFileToS3()` call (line ~98), add a document interception branch:

   ```javascript
   // Route document files through documentService pipeline (bypasses S3)
   if (isDocumentMimeType(file.type)) {
     setCurrentFile(file.name);
     try {
       const asset = await uploadDocument(file, {
         onStatusChange: (status) => {
           // Map to progress for UI consistency
           if (status === 'uploading') setProgress(Math.round(((completedFiles + 0.3) / totalFiles) * 100));
           if (status === 'converting') setProgress(Math.round(((completedFiles + 0.7) / totalFiles) * 100));
         },
       });

       // Build a result object matching the shape useS3Upload.onSuccess expects
       const docResult = {
         url: asset.url,
         name: asset.name,
         originalFilename: file.name,
         type: file.type,
         size: file.size,
         mediaType: 'document',
         format: file.name.split('.').pop()?.toLowerCase(),
         resourceType: 'document',
         // No width/height/duration for documents
         width: null,
         height: null,
         duration: null,
       };

       setUploadedFiles(prev => [...prev, docResult]);
       onSuccess?.(docResult);
       completedFiles++;
     } catch (error) {
       logger.error(`Error uploading document ${file.name}:`, error);
       setErrors(prev => [...prev, { file: file.name, errors: [error.message] }]);
       onError?.(error);
     }
     continue; // Skip the S3 upload path for this file
   }
   ```

3. The `continue` statement ensures the existing S3 upload code below is skipped for document files. Non-document files (images, videos, audio) proceed through the existing path unchanged.

**Important:** Do NOT move or restructure the existing S3 upload code. Only ADD the document branch before it, with a `continue` to skip the rest of the loop iteration.
