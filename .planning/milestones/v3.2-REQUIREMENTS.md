# Requirements: BizScreen v3.2 Display Toolkit

**Defined:** 2026-02-13
**Core Value:** Screens reliably display the right content at the right time, even when offline

## v3.2 Requirements

Requirements for Display Toolkit milestone. Each maps to roadmap phases.

### Clock & Date

- [x] **CLOCK-01**: User can configure clock widget timezone (IANA timezone selector, e.g., "America/New_York") — Phase 56
- [x] **CLOCK-02**: User can toggle 12h/24h time format on clock widget — Phase 56
- [x] **CLOCK-03**: User can enable seconds display on clock widget — Phase 56
- [x] **CLOCK-04**: User can select analog clock style with customizable accent color — Phase 56
- [x] **CLOCK-05**: User can use combined clock+date widget showing time and date together in one zone — Phase 56
- [x] **CLOCK-06**: Clock/date widgets use screen's assigned timezone instead of browser timezone — Phase 56 + 63

### Weather

- [x] **WTHR-01**: Weather API key is proxied through server-side Edge Function (not exposed in client bundle) — Phase 58
- [x] **WTHR-02**: Weather widget displays forecast mode (multi-day forecast in zone widget, not just current conditions) — Phase 58
- [x] **WTHR-03**: Weather widget uses screen's timezone for display formatting — Phase 58 + 63
- [x] **WTHR-04**: Weather widget caches data to IndexedDB for offline display with "last updated" indicator — Phase 58

### QR Code

- [x] **QR-01**: User can select QR code type (URL, WiFi, plain text) in layout editor sidebar — Phase 57
- [x] **QR-02**: User can configure WiFi QR code with SSID, password, and encryption type — Phase 57
- [x] **QR-03**: User can set error correction level (L/M/Q/H) on QR code widget — Phase 57
- [x] **QR-04**: User can add brand logo overlay to QR code center (auto-sets error correction to H) — Phase 57
- [x] **QR-05**: QR code import bug fixed (QRCodeSVG import added to prevent player crash) — Phase 57

### Video Playback

- [x] **VIDEO-01**: User can add video (MP4) as an element in layout zones — Phase 59
- [x] **VIDEO-02**: Video elements play with autoplay, muted, and loop in player — Phase 59
- [x] **VIDEO-03**: Player supports HLS adaptive streaming (.m3u8 URLs) via hls.js — Phase 59
- [x] **VIDEO-04**: Video elements show poster frame/thumbnail in editor (not autoplay in editor) — Phase 59
- [x] **VIDEO-05**: Video playback integrates with existing stuck detection for stall recovery — Phase 59

### Portrait Mode

- [x] **PORT-01**: User can set screen orientation (landscape/portrait) per device in screen settings — Phase 61
- [x] **PORT-02**: Layout editor supports portrait canvas (9:16 aspect ratio) for content design — Phase 61
- [x] **PORT-03**: Player applies CSS rotation when content orientation differs from device hardware orientation — Phase 61
- [x] **PORT-04**: At least 3 portrait-oriented templates available in template marketplace — Phase 61
- [x] **PORT-05**: Orientation mismatch warning shown when scheduling portrait content to landscape screens (and vice versa) — Phase 61

### Screen Groups & Tags

- [x] **GROUP-01**: User can add/remove tags on screen groups with tag chip UI — Phase 60
- [x] **GROUP-02**: User can filter screen groups by tag in the groups list — Phase 60
- [x] **GROUP-03**: User can push a playlist to all screens in a group — Phase 60
- [x] **GROUP-04**: User can perform bulk actions on screen groups (delete, tag, assign content) — Phase 60
- [x] **GROUP-05**: Tag queries use GIN index for efficient array matching — Phase 60

### Menu Board

- [x] **MENU-01**: User can create a menu board with structured categories and items (name, description, price, image) — Phase 62
- [x] **MENU-02**: User can reorder menu categories and items via drag-and-drop — Phase 62
- [x] **MENU-03**: Menu board renders as themed widget in player with category sections, item names, prices, and descriptions — Phase 62
- [x] **MENU-04**: Menu board supports multiple price columns (e.g., Small/Medium/Large) — Phase 62
- [x] **MENU-05**: Menu board auto-paginates for long menus with smooth page transitions — Phase 62
- [x] **MENU-06**: User can toggle item availability (show/hide items without deleting) — Phase 62
- [x] **MENU-07**: Menu board widget updates in near-real-time via Supabase Realtime subscription — Phase 62
- [x] **MENU-08**: User can add dietary/allergen tags to menu items (rendered as icons/badges) — Phase 62
- [x] **MENU-09**: Menu board respects tenant locale for currency formatting — Phase 62

### Infrastructure

- [x] **INFRA-01**: Widget registry pattern replaces switch statement duplication across EditorCanvas, LivePreviewWindow, and SceneRenderer — Phase 56
- [x] **INFRA-02**: Widget type switching in editor resets props to new type's defaults (prevents stale prop cross-contamination) — Phase 56

## Future Requirements

Deferred to subsequent milestones. Tracked but not in current roadmap.

### Advanced Widgets
- **ADV-01**: Video wall sync across screen groups via WebSocket time-sync
- **ADV-02**: Weather-triggered content rules (show specific content based on temperature/conditions)
- **ADV-03**: Interactive QR code analytics with scan tracking via redirect URLs
- **ADV-04**: POS integration for menu boards (Toast, Square, Clover APIs)

### Advanced Screen Management
- **ADV-05**: Screen group scheduling (different content at different times per group)
- **ADV-06**: Auto-discovery of screens via mDNS/SSDP on local network

## Out of Scope

| Feature | Reason |
|---------|--------|
| Video transcoding on upload | Massive infrastructure cost; users upload web-ready MP4 |
| Custom video player controls (play/pause/seek) | Digital signage is passive; no viewer interaction needed |
| Real-time weather alerts (NOAA) | Liability concerns; emergency override system handles severe weather |
| Portrait auto-detection via accelerometer | Browser accelerometer access is unreliable; screens are physically mounted once |
| Menu board POS integration | Complex APIs, OAuth flows, frequent changes; defer to premium feature |
| Screen group auto-discovery | Network discovery unreliable across subnets/VLANs |
