**Categories seeded by migration 080** (lines 451-461) — slug values are: `retail`, `restaurant`, `corporate`, `healthcare`, `education`, `hospitality`, `fitness`, `events`. Resolve each to a UUID via `(SELECT id FROM template_categories WHERE slug = '<slug>')`.

**Consumer contract — QuickCustomizePanel expects `detail.metadata.svgContent`** (src/components/QuickCustomizePanel.jsx line 54):
```js
const svgContent = detail?.metadata?.svgContent;
```
If `metadata.svgContent` is missing, the panel renders the no-SVG fallback message (line 167). The 8 category-representative rows MUST have `metadata->>'svgContent'` populated. The 9th fallback-test row MUST omit `svgContent` from its `metadata` so the fallback path is reachable from seed data.

**Consumer contract — `installWithCustomization()` fetches the FIRST SLIDE** to get the SVG content for cloning. So `template_library_slides.design_json->>'svgContent'` must match `template_library.metadata->>'svgContent'` for each populated-SVG template, and each template must have at least one slide at `position = 0`. The fallback-test row still requires a slide row (for FK integrity and listing), but its `design_json` also omits `svgContent`.

**Idempotency:** This is a dev seed — executed repeatedly in `supabase db push`. Use a stable WHERE clause with a deterministic `name` to make the INSERT idempotent. Pattern from migration 091 (lines 9-22) uses `ON CONFLICT ... DO NOTHING`. For `template_library` there is no unique constraint on name, so use a guard pattern:
```sql
INSERT INTO template_library (...)
SELECT ... WHERE NOT EXISTS (SELECT 1 FROM template_library WHERE name = '<name>' AND metadata->>'seed_source' = 'dev-166.1');
```
Tag every seeded row with `metadata->>'seed_source' = 'dev-166.1'` for safe re-run detection and easy cleanup.
