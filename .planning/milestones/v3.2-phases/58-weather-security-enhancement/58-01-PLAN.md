<objective>
Create a weather-proxy Supabase Edge Function that proxies all OpenWeatherMap API calls server-side, then migrate all client-side weather and geocoding code to use the proxy, eliminating the exposed VITE_OPENWEATHER_API_KEY from the client bundle.

Purpose: The OpenWeatherMap API key is currently exposed in the client JavaScript bundle (VITE_OPENWEATHER_API_KEY). This is the last remaining client-side API key exposure after the Unsplash and RSS proxies were completed in earlier phases. Moving it server-side eliminates the security risk.

Output: weather-proxy Edge Function, weather_cache migration, weatherProxyService.js client, migrated weatherService.js and geolocationService.js.
</objective>

  **1. Create `supabase/migrations/145_create_weather_cache.sql`:**
  Follow the `144_create_rss_feed_cache.sql` pattern exactly.

  ```sql
  CREATE TABLE IF NOT EXISTS weather_cache (
    cache_key TEXT PRIMARY KEY,
    action TEXT NOT NULL,
    location TEXT NOT NULL,
    units TEXT NOT NULL DEFAULT 'imperial',
    response_data JSONB NOT NULL,
    fetched_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    expires_at TIMESTAMPTZ NOT NULL DEFAULT (now() + interval '15 minutes')
  );
  CREATE INDEX IF NOT EXISTS idx_weather_cache_expires ON weather_cache (expires_at);
  ALTER TABLE weather_cache ENABLE ROW LEVEL SECURITY;
  ```

  No RLS policies needed -- only service role accesses this table (same as rss_feed_cache).

  **2. Create `supabase/functions/weather-proxy/index.ts`:**
  Follow the `rss-proxy/index.ts` structure closely but with these differences:

  - Import `createClient` from `npm:@supabase/supabase-js@2` and `corsHeaders` from `../_shared/cors.ts`
  - Constants: `CACHE_TTL_MINUTES = 15`, `FETCH_TIMEOUT_MS = 10_000`, `OWM_BASE = 'https://api.openweathermap.org/data/2.5'`, `GEO_BASE = 'https://api.openweathermap.org/geo/1.0'`
  - Helper: `jsonResponse(body, status)` -- identical to rss-proxy
  - Auth: Accept BOTH authenticated user JWTs AND the anon key. This is critical because player devices are NOT authenticated users -- they use the anon key. Check `supabase.auth.getUser(token)` first; if that fails (authError or no user), verify `token === Deno.env.get('SUPABASE_ANON_KEY')`. If neither matches, return 401. This prevents player screens from breaking.
  - Actions (via switch on `body.action`):
    - `'current'`: Fetch current weather. Params: `{ city, units, lat, lon }`. Build URL: if lat/lon provided use `${OWM_BASE}/weather?lat=${lat}&lon=${lon}&units=${units}&appid=${apiKey}`, else `${OWM_BASE}/weather?q=${encodeURIComponent(city)}&units=${units}&appid=${apiKey}`. Cache key: `current:${city||lat+','+lon}:${units}`.
    - `'forecast'`: Fetch 5-day forecast. Params: `{ city, units, lang, lat, lon }`. Build URL similarly with `/forecast` endpoint. Cache key: `forecast:${city||lat+','+lon}:${units}`.
    - `'geocode'`: Forward geocoding. Params: `{ query, limit }`. URL: `${GEO_BASE}/direct?q=${encodeURIComponent(query)}&limit=${limit||5}&appid=${apiKey}`. Cache key: `geocode:${query}:${limit}`.
    - `'reverse-geocode'`: Reverse geocoding. Params: `{ lat, lon }`. URL: `${GEO_BASE}/reverse?lat=${lat}&lon=${lon}&limit=1&appid=${apiKey}`. Cache key: `reverse:${lat.toFixed(2)},${lon.toFixed(2)}`.
  - For each action: check weather_cache first (by cache_key, where expires_at > now()). If cache hit, return cached response_data. If miss, fetch from OWM with timeout (AbortController, 10s), upsert into weather_cache, return data.
  - Get API key via `Deno.env.get('OPENWEATHER_API_KEY')` (NOT the VITE_ prefix -- this is a server-side secret).
  - Return format: `{ ok: true, data: <OWM response JSON>, meta: { cached: boolean } }`
  - Error handling: return `{ ok: false, error: { code, message } }` with appropriate HTTP status codes (400, 401, 502).

  ```javascript
  import { supabase } from '../supabase.js';
  import { createScopedLogger } from './loggingService.js';

  const logger = createScopedLogger('WeatherProxyService');
  const FUNCTION_NAME = 'weather-proxy';
  ```

  Export these functions:
  - `fetchCurrentWeather(city, options = {})` -- calls proxy with `{ action: 'current', city, units: options.units || 'imperial' }`. If options.lat/lon provided, pass those instead of city. Return `data.data`.
  - `fetchWeatherForecast(city, options = {})` -- calls proxy with `{ action: 'forecast', city, units: options.units || 'imperial', lang: options.lang || 'en' }`. If options.lat/lon provided, pass those. Return `data.data`.
  - `fetchGeocode(query, limit = 5)` -- calls proxy with `{ action: 'geocode', query, limit }`. Return `data.data`.
  - `fetchReverseGeocode(lat, lon)` -- calls proxy with `{ action: 'reverse-geocode', lat, lon }`. Return `data.data`.

  Each function: call `supabase.functions.invoke(FUNCTION_NAME, { body })`, check for error, check `data.ok`, throw on failure with descriptive message. Log errors via logger.
