**ClockWidget.jsx** -- Rewrite to support all CLOCK requirements while keeping the same component signature `({ props, timezone })`:

1. Add a `resolveTimezone(widgetTimezone, screenTimezone)` helper function at top of file:
   - If `widgetTimezone` exists and is not `'screen'`, return it (explicit IANA timezone override)
   - If `screenTimezone` exists, return it (screen's assigned timezone from DB)
   - Otherwise return `Intl.DateTimeFormat().resolvedOptions().timeZone` (browser fallback)

2. Accept `timezone` as a second prop (passed by parent ZonePlayer/SceneRenderer): `export function ClockWidget({ props = {}, timezone })`

3. Support these props (all with sensible defaults):
   - `textColor` (default: '#ffffff')
   - `format` (default: '12h', options: '12h' | '24h') -- CLOCK-02
   - `showSeconds` (default: false) -- CLOCK-03
   - `timezone` (default: 'screen') -- CLOCK-01, CLOCK-06 -- the prop `props.timezone` is the widget-level override; the component prop `timezone` is the screen-level timezone
   - `style` (default: 'digital', options: 'digital' | 'analog') -- CLOCK-04
   - `accentColor` (default: '#3b82f6') -- CLOCK-04
   - `size`, `customFontSize` (keep existing behavior)

4. For **digital** style: Use `Intl.DateTimeFormat` with `timeZone` option (from resolveTimezone) to format time. Pass `hour12: format === '12h'`, include `second: '2-digit'` when showSeconds is true. Use `toLocaleTimeString('en-US', options)` where options includes `{ timeZone: resolvedTz, hour: '2-digit', minute: '2-digit', hour12, second }`.

5. For **analog** style: Create an inline `AnalogClock` sub-component using SVG (see research Pattern 4). Extract hours, minutes, seconds from a timezone-aware date. Use the research example as a starting point:
   - SVG viewBox="0 0 100 100", clock face circle, 12 hour markers, hour/minute/second hands
   - Hour markers: thicker at 12/3/6/9 positions
   - Hour hand: shorter and thicker, minute hand: longer and thinner
   - Second hand: only shown when `showSeconds` is true, use red color (`#ef4444`)
   - Use `accentColor` for face, markers, hour/minute hands
   - Center dot in `accentColor`
   - For getting timezone-aware hour/minute/second values for analog: use `Intl.DateTimeFormat` with `{ hour: 'numeric', minute: 'numeric', second: 'numeric', hour12: false, timeZone: resolvedTz }` to extract numeric values, then parse them to calculate angles

6. Update interval: use `setInterval` at 1000ms. For analog with seconds, this creates a "tick" motion which is acceptable (many real clocks tick). Do NOT use requestAnimationFrame -- the tick look is fine and avoids unnecessary CPU usage.

7. Keep the existing `sizeMap` and `getFontSize` helpers. Size applies to the digital text. For analog, the SVG fills its container naturally.

**DateWidget.jsx** -- Enhance with timezone awareness:

1. Add the same `resolveTimezone` helper (or import from ClockWidget -- but to avoid circular deps, just duplicate the 5-line helper; it is trivial).

2. Accept `timezone` as a second prop: `export function DateWidget({ props = {}, timezone })`

3. Support these props:
   - `textColor` (default: '#ffffff')
   - `format` (default: 'long', options: 'long' | 'short' | 'numeric') -- 'long' = "Thursday, January 23", 'short' = "Jan 23, 2025", 'numeric' = "1/23/2025"
   - `timezone` (default: 'screen') -- widget-level timezone override
   - `size`, `customFontSize` (keep existing)

4. Use `Intl.DateTimeFormat` with `timeZone` option for formatting. Map format prop to Intl options:
   - 'long': `{ weekday: 'long', month: 'long', day: 'numeric', timeZone: resolvedTz }`
   - 'short': `{ month: 'short', day: 'numeric', year: 'numeric', timeZone: resolvedTz }`
   - 'numeric': `{ month: 'numeric', day: 'numeric', year: 'numeric', timeZone: resolvedTz }`

5. Change the update interval from 1000ms to 60000ms (every minute). Date changes are not time-sensitive. This reduces unnecessary renders.

IMPORTANT: Both components must work correctly when `timezone` prop is undefined (backward compatibility with scenes/layouts that don't pass timezone). The resolveTimezone helper handles this by falling back to browser timezone.
