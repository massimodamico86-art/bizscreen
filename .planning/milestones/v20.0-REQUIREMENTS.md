# Requirements: BizScreen v20.0 Templates Reimagined (ARCHIVED)

**Defined:** 2026-04-15
**Shipped:** 2026-05-03
**Status:** ✅ COMPLETE — 39/39 requirements satisfied (with 11 manual UAT items deferred as known gaps)
**Core Value:** Screens reliably display the right content at the right time, even when offline
**Milestone Goal:** Rebuild the template experience end-to-end — modernized gallery UI, strong discovery, a polished preview→apply→customize flow, and a substantially expanded SVG content library — so first-time users find and ship great-looking screens fast.

## v20.0 Requirements

### Data Layer Foundation

- [x] **TDAT-01**: Gallery UI reads from a single `gallery_templates` Postgres VIEW that UNIONs `template_library` and `svg_templates` (single-query gallery fetches)
- [x] **TDAT-02**: A dedicated `templateGalleryService.js` is the sole data-access point for gallery UI, replacing the three-source JS merge in `svgTemplateService.fetchSvgTemplates`
- [x] **TDAT-03**: `svg_templates` RLS audit completes; any cross-tenant access gap is fixed via migration before new content is uploaded
- [x] **TDAT-04**: `LOCAL_SVG_TEMPLATES` hardcoded array is seeded into `svg_templates` and removed from source

### Gallery UX

- [x] **TGAL-01**: User can browse all templates in a redesigned gallery with modern card layout, orientation badges, and hover states
- [x] **TGAL-02**: User sees distinct empty, loading (skeletons), and error states on the gallery page
- [x] **TGAL-03**: User can sort templates by Newest, Most Popular (`use_count`), Alphabetical, and Recently Used
- [x] **TGAL-04**: User sees a "New" badge on templates added within a configurable recency window
- [x] **TGAL-05**: Gallery is responsive — usable density and legibility on mobile (touch) and desktop

### Discovery & Search

- [x] **TDSC-01**: User can free-text search templates by name, description, and tags with instant (sub-second) client-side filtering
- [x] **TDSC-02**: User can filter templates by category, tags, and orientation (landscape/portrait) with visible, dismissible filter chips
- [x] **TDSC-03**: User can clear all active filters and search in a single action
- [x] **TDSC-04**: Gallery filter, sort, and search state persist in URL query params for shareable/deep-linkable views
- [x] **TDSC-05**: User sees a helpful "no results" empty state (with suggestions) when filters yield zero templates

### Preview & Apply Flow

- [x] **TPRV-01**: User can open a full-screen preview modal from any template card, with prev/next navigation across the filtered result set
- [x] **TPRV-02**: User can customize brand colors, logo placement, and text overrides directly in the preview modal via an integrated QuickCustomizePanel
- [x] **TPRV-03**: Preview modal shows live SVG preview updates as customization values change
- [x] **TPRV-04**: User can apply a template to a new scene in one click; the flow routes by template type (SVG templates → FabricSvgEditor / QuickCustomize; Polotno entries → Polotno scene editor)
- [x] **TPRV-05**: Apply flow is race-safe — clone-and-patch executes atomically (single RPC or transaction) so customized values never get lost
- [x] **TPRV-06**: Apply flow eliminates the `sessionStorage` handoff in favor of direct navigation-state or URL params

### Starter Packs

- [x] **TPCK-01**: User can browse curated starter packs as dedicated bundle cards on the gallery page
- [x] **TPCK-02**: User can apply an entire starter pack to their tenant library in one click (bulk clone)
- [x] **TPCK-03**: Admin can create and edit starter packs via the existing admin surface (reuses preserved `marketplaceService.js`)
- [x] **TPCK-04**: Pack cards display template count, thumbnail mosaic, and industry/use-case label

### Template Content

- [x] **TCTN-01**: At least 100 net-new SVG templates ship at v20.0 launch across a defined category taxonomy (delivered: 103 net-new → 127 active total across 15 categories)
- [x] **TCTN-02**: All net-new templates pass an SVG validation gate (well-formed, expected `data-customize-*` attributes, sanitized markup) — `svgValidator` 6-rule + admin upload gate
- [x] **TCTN-03**: Template taxonomy (categories, tags, industries) is documented and enforced at admin-upload time — `chk_svg_templates_category_enum` CHECK constraint + admin form
- [x] **TCTN-04**: Admin pipeline generates template thumbnails automatically (replacing the LayoutTemplate icon placeholder) — 127/127 PNGs uploaded to `s3://bizscreen-media/thumbnails/system/`

### Scene Editor Integration

- [x] **TEDR-01**: User can open the template gallery from inside the scene editor via a "Browse Templates" action
- [x] **TEDR-02**: User returning from gallery→editor lands back on the origin scene with the new template applied
- [x] **TEDR-03**: Gallery deep-link (e.g. `?editorReturn=true`) preserves editor context across the round-trip

### Onboarding Integration

- [x] **TONB-01**: New-user onboarding includes a skippable starter-pack selection step that surfaces curated packs inline
- [x] **TONB-02**: Selecting a pack during onboarding bulk-applies it to the tenant's library without leaving the wizard
- [x] **TONB-03**: Onboarding state tracks whether starter pack was chosen, skipped, or not reached (`completed_starter_pack`)
- [x] **TONB-04**: A lightweight feature tour (driver.js) introduces gallery discovery and preview/apply flow on first visit post-onboarding

### Favorites

- [x] **TFAV-01**: User can favorite/unfavorite templates from the gallery card and preview modal
- [x] **TFAV-02**: User can filter the gallery by "Favorites only" via a dedicated toggle/chip
- [x] **TFAV-03**: Favorites persist per-user across sessions (DB-backed `template_favorites` + `gallery_templates_with_favorites` VIEW)

### Test & Quality

- [x] **TQAL-05**: Gallery E2E tests use structural assertions (no count-exact or screenshot-diff against dynamic template lists) so they stay green as the catalog grows — TQAL-05 audit lint shows 0 matches across all gallery specs

## Future Requirements (deferred beyond v20.0)

### Polotno QuickCustomize

- **TPRV-F1**: QuickCustomize-before-apply extended to Polotno templates (`template_library` entries with `design_json`) — requires Polotno JSON mutation primitives; out of scope for v20.0

### Gallery scale

- **TGAL-F1**: Virtualized gallery grid (`@tanstack/react-virtual`) activated when total catalog exceeds ~200 templates — stack is installed but gating is progressive enhancement

### Personalization

- **TPER-F1**: Template recommendations based on industry/business type and usage patterns — requires reliable `profiles.business_type` population

## Out of Scope (excluded explicitly)

| Feature | Reason |
|---------|--------|
| User-created template marketplace (buy/sell) | Already out-of-scope per PROJECT.md; requires moderation/payments |
| AI-generated templates | Already out-of-scope per PROJECT.md; unpredictable results |
| Reviving Template Marketplace UI concepts | v19.0 pivot — SVG gallery aliased is the direction forward |
| Algolia / hosted search service | Catalog size (≤500) fits in-memory; adds cost, violates offline friendliness of the stack |
| Shepherd.js / Intro.js onboarding tours | AGPL-licensed; driver.js (MIT) is the compatible choice |
| react-joyride | Incompatible with React 19 (`unmountComponentAtNode` removed) |
| Headless CMS for template content | Over-engineered; existing admin + Supabase + S3 pattern covers the need |
| Live multi-user template collaboration | Not core to the signage use case |
