In `src/components/layout-editor/LayoutPropertiesPanel.jsx`:

1. Add imports at the top (after the existing QRCodeWidgetControls import):
   ```javascript
   import { DataTableWidgetControls } from '../scene-editor/DataTableWidgetControls.jsx';
   import { RssWidgetControls } from '../scene-editor/RssWidgetControls.jsx';
   ```

2. Inside the `WidgetControls` function, after the `{widgetType === 'qr' && ...}` block (around line 584-589) and BEFORE the `<ColorPicker>` for Text Color (around line 591), add three new conditional render blocks:

   For data-table:
   ```jsx
   {widgetType === 'data-table' && (
     <DataTableWidgetControls
       props={props}
       onPropChange={(key, value) => onPropsUpdate({ [key]: value })}
       onMultiPropChange={onPropsUpdate}
     />
   )}
   ```

   For rss-ticker and rss-card (both use the same controls component with widgetType prop):
   ```jsx
   {(widgetType === 'rss-ticker' || widgetType === 'rss-card') && (
     <RssWidgetControls
       widgetType={widgetType}
       props={props}
       onPropChange={(key, value) => onPropsUpdate({ [key]: value })}
     />
   )}
   ```

The scene-editor controls already handle all the properties listed in the WIDGET_REGISTRY defaultProps for these types plus additional configuration. The `onPropChange(key, value)` adapter wraps the layout editor's `onPropsUpdate({})` pattern. The `onMultiPropChange` maps directly to `onPropsUpdate` since both accept an object of prop updates.

IMPORTANT: Do NOT move the shared `<ColorPicker label="Text Color" ...>` that's at the bottom of WidgetControls. The RSS and DataTable controls already include their own text color inputs inline, so the shared one serves as a fallback for widget types that don't have inline text color controls (like clock, weather). This is fine — having a redundant text color picker is acceptable and matches the existing pattern.
