<task type="auto">
  <name>Task 1: Create screenshot comparison report generator</name>
  <files>scripts/screenshot-compare.cjs</files>
  <action>
Create a Node.js script that generates an HTML visual regression report by comparing current screenshots against a committed baseline.

**How it works:**
1. The script takes two directory arguments: `--baseline=screenshots/` (committed baseline) and `--current=screenshots/` (after test run). In CI, both will point to the same `screenshots/` directory since the baseline IS the committed screenshots. The comparison is: git HEAD (baseline) vs working tree (current after tests).
2. For the CI use case: use `git show HEAD:{file}` to get the baseline version of each screenshot, and compare against the current file on disk.
3. For each .png file in `screenshots/` (recursively):
   - If the file exists in git HEAD AND on disk: compare pixel-by-pixel using a pure-JS pixel comparison (no external dependency -- use raw PNG buffer comparison via built-in `crypto` module to compute SHA-256 hashes; if hashes differ, mark as "changed")
   - If the file exists on disk but NOT in git: mark as "new"
   - If the file exists in git but NOT on disk: mark as "removed"
4. Generate an HTML report at `test-results/screenshot-report.html` with:
   - Summary: total screenshots, unchanged count, changed count, new count, removed count
   - A table listing all changed/new/removed screenshots with their paths
   - For changed screenshots: show the current screenshot inline as a base64 data URI
   - Color coding: green for unchanged (just count, not listed), yellow for changed, blue for new, red for removed
5. Exit code: always 0 (report is informational, not a gate)

**Implementation details:**
- Use `child_process.execSync('git ls-tree -r --name-only HEAD -- screenshots/')` to get baseline file list
- Use `child_process.execSync('git show HEAD:{filepath}')` to get baseline file content for hash comparison
- Use `crypto.createHash('sha256')` for fast comparison without pixel-level diffing (no external deps needed)
- Use `fs.readdirSync` recursively to get current file list
- Handle gracefully: no git repo, no screenshots dir, first run (no baseline)
- Script must work with no additional npm dependencies (Node.js built-ins only)

**Do NOT:**
- Add pixelmatch, sharp, or any image processing library (hash comparison is sufficient for detecting changes)
- Make this a blocking gate (it is informational only)
- Require any external services or APIs
  </action>

  </verify>
  <done>Running `node scripts/screenshot-compare.cjs` produces test-results/screenshot-report.html with a summary of screenshot changes. Script exits 0 regardless of changes found.</done>
