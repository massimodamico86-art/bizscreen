  <name>Task 2: Replace PageLoader spinner with page-appropriate skeletons in App.jsx</name>

  <action>
In `src/App.jsx`:

1. Import `getSkeletonForPage` and the individual skeleton components from `./components/PageSkeletons`

2. Replace the `PageLoader` spinner component. Keep it as a fallback but rename to `FallbackLoader`. Create a new `PageSkeleton` component that takes a `pageId` prop:
   ```jsx
   const PageSkeleton = ({ pageId }) => {
     const SkeletonComponent = getSkeletonForPage(pageId);
     return <SkeletonComponent />;
   };
   ```

3. Update ALL entries in the `pages` map to use `<PageSkeleton pageId="..." />` as the Suspense fallback instead of `<PageLoader />`. For example:
   ```
   // Before:
   dashboard: <RouteErrorBoundary name="Dashboard"><Suspense fallback={<PageLoader />}><DashboardPage ... /></Suspense></RouteErrorBoundary>,

   // After:
   dashboard: <RouteErrorBoundary name="Dashboard"><Suspense fallback={<PageSkeleton pageId="dashboard" />}><DashboardPage ... /></Suspense></RouteErrorBoundary>,
   ```

4. For dynamic routes in the fallback chain (playlist-editor-*, layout-editor-*, etc.), use the EditorSkeleton directly:
   ```
   <Suspense fallback={<PageSkeleton pageId="playlist-editor-" />}>
   ```

5. Keep the FallbackLoader (old PageLoader) for use in the admin/super-admin dashboard fallbacks and the special routes (Canva callback, password reset) where a generic spinner is appropriate.

6. Also update the `PageLoader` prop passed to AppContent to be `PageSkeleton` instead (or keep both, passing PageSkeleton as the primary). The AppContent function signature should accept `PageSkeleton` and use it for Suspense fallbacks.

Important: This task builds on Plan 01 which already added RouteErrorBoundary wrapping. Preserve that wrapping while changing the Suspense fallback.
