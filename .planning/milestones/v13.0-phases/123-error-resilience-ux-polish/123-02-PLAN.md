  <files>src/hooks/useApiCall.js</files>
  <action>
Create `src/hooks/useApiCall.js` that composes `useRetryWithBackoff` with data management:

```
useApiCall(fetchFn, options?) -> { data, loading, error, retryCount, maxedOut, retry, refetch }
```

Implementation:
- Accept `fetchFn` -- an async function that RETURNS data (unlike useRetryWithBackoff which expects a void function)
- Accept `options` matching useRetryWithBackoff options plus: `initialData` (default null), `enabled` (default true), `onSuccess(data)` callback, `onError(error)` callback
- Internally, maintain `data` state via useState
- Create a wrapper function around fetchFn that calls it, stores the result in data state, and throws on error (so useRetryWithBackoff can catch it)
- Delegate retry/backoff logic entirely to useRetryWithBackoff
- `refetch` -- alias for `retry` that also clears current data (useful for force-refresh)
- Default maxRetries: 3 (not 5, since this is for user-facing API calls that should fail fast)
- Default baseDelay: 1000ms
- Default maxDelay: 10000ms
- Default pollInterval: 0 (no auto-polling by default, unlike useRetryWithBackoff's 30s default)
- Export as named export

Add JSDoc with usage example showing how pages should use it:
```js
const { data: screens, loading, error, maxedOut, retry } = useApiCall(
  () => fetchScreens(tenantId),
  { enabled: !!tenantId }
);
```
