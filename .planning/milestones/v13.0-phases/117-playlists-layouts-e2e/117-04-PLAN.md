**Fix 1: navigateToLayoutEditor route**

The current code uses `__setCurrentPage('layout-editor')` which shows "Page not found". The correct format is `__setCurrentPage('layout-editor-{id}')`.

Strategy to get a valid layout ID:
a. First, navigate to Templates page (layouts page) via sidebar
b. Click "Your Designs" to see user-created layouts
c. If layouts exist, extract an ID by intercepting the navigation:
   ```javascript
   // Intercept the onNavigate call when clicking a layout card
   const layoutCards = page.locator('[class*="aspect-video"][class*="cursor-pointer"]');
   if (await layoutCards.count() > 0) {
     // Set up a listener for page change
     const pagePromise = page.evaluate(() => {
       return new Promise(resolve => {
         const orig = window.__setCurrentPage;
         window.__setCurrentPage = (page) => { orig(page); resolve(page); };
       });
     });
     await layoutCards.first().click();
     const newPage = await pagePromise;
     // newPage should be 'layout-editor-{id}'
   }
   ```
d. If no user designs exist, create a layout programmatically via the app's internal API:
   ```javascript
   const layoutId = await page.evaluate(async () => {
     // Import the layoutService to create a layout
     const { createLayout } = await import('/src/services/layoutService.js');
     const layout = await createLayout({ name: 'E2E Test Layout' });
     return layout.id;
   });
   // Then navigate
   await page.evaluate((id) => window.__setCurrentPage('layout-editor-' + id), layoutId);
   ```
   Note: Dynamic imports may not work in evaluate context. Alternative approach -- just navigate with a UUID and accept that the editor shows "Layout not found". BUT this defeats the purpose since "Layout not found" has minimal UI.

   BEST approach: Try to create a layout by calling supabase directly:
   ```javascript
   const layoutId = await page.evaluate(async () => {
     const { supabase } = await import('/src/supabase.js');
     const { data } = await supabase.from('layouts').insert({
       name: 'E2E Test Layout',
       description: 'Created for E2E screenshot tests'
     }).select().single();
     return data?.id;
   });
   ```
   If this works, navigate to `layout-editor-{id}`.

   If Supabase calls fail in test context, as a LAST RESORT fallback: Navigate to the layout editor with a placeholder and capture whatever state renders (which may be "Layout not found" -- this is still better than "Page not found: layout-editor").

   IMPORTANT: Even "Layout not found" shows the LayoutEditorPage component's not-found state (which includes "Back to Layouts" button), which is a valid layout editor page screenshot. It proves the route works. But ideally we want the full editor with zones.
