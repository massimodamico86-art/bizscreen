Create `tests/e2e/helpers/screenshots.js` with these exports:

1. **VIEWPORTS constant** -- Object defining the three viewport presets:
```javascript
export const VIEWPORTS = {
  desktop: { width: 1440, height: 900 },
  tablet: { width: 768, height: 1024 },
  mobile: { width: 375, height: 667 },
};
```

2. **cleanScreenshots(area)** -- Deletes and recreates the `screenshots/{area}/` directory. Uses `fs.rmSync` with `{ recursive: true, force: true }` followed by `fs.mkdirSync` with `{ recursive: true }`. If no area is provided, cleans the entire `screenshots/` directory but recreates it empty. Uses `path.resolve(process.cwd(), 'screenshots', area)` for the path.

3. **screenshotStep(page, area, step, options = {})** -- The main helper function:
   - Parameters:
     - `page` -- Playwright Page object
     - `area` -- string, the feature area (e.g., 'auth', 'dashboard', 'media'). Used as subdirectory name.
     - `step` -- string or number, the step identifier (e.g., '01-login-page', '02-filled-form'). Recommend zero-padded numbers with description.
     - `options.viewport` -- optional string key from VIEWPORTS ('desktop', 'tablet', 'mobile'). Defaults to 'desktop'.
     - `options.fullPage` -- optional boolean, defaults to false.
   - Implementation:
     - Derive viewport label: if `options.viewport` is provided, use it. Otherwise detect current viewport size from `page.viewportSize()` and match against VIEWPORTS entries. If no match, use 'desktop' as fallback.
     - Build filename: `${area}-${step}-${viewportLabel}.png`
     - Build directory: `path.resolve(process.cwd(), 'screenshots', area)`
     - Ensure directory exists: `fs.mkdirSync(dir, { recursive: true })`
     - Take screenshot: `await page.screenshot({ path: path.join(dir, filename), fullPage: options.fullPage || false })`
     - Return the full path to the screenshot file (useful for logging/debugging).
   - The function must NOT change the page viewport size. It just labels the screenshot based on the current viewport.

4. **detectViewport(page)** -- Internal helper (exported for testing but not the primary API):
   - Gets `page.viewportSize()`, matches width against VIEWPORTS entries (exact match or closest), returns the key string ('desktop', 'tablet', 'mobile').

All imports: `import fs from 'fs'` and `import path from 'path'` (Node built-ins, available in Playwright context).

```javascript
 * Screenshot Helpers for E2E Tests

 * Provides consistent screenshot capture at every test step.
 * Screenshots are saved to screenshots/{area}/{area}-{step}-{viewport}.png
 *
 * @module e2e/helpers/screenshots
 */
```
