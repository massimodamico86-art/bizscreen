<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated client → RPC | Supabase JWT verified at PostgREST; `auth.uid()` is derived server-side. RPC preamble re-checks IS NOT NULL. |
| RPC (SECURITY DEFINER) → svg_templates | RPC runs as definer (superuser) but self-enforces the svg_templates SELECT RLS predicate in its WHERE clause. SECURITY DEFINER bypass of table-level RLS is intentional — the predicate is replicated in-function. |
| RPC → scene_slides.design_json | `p_customized_svg` is written raw into `jsonb_build_object('svgContent', v_resolved_svg)`. The RPC is NOT a sanitizer; client-side DOMPurify is the XSS defense boundary (D-17 from Phase 172, unchanged). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-172.1-04 | S (Spoofing) | `auth.uid() IS NULL` check | mitigate | Preamble: `IF v_user_id IS NULL THEN RAISE EXCEPTION 'Not authenticated'` — mirrors 168:58-61. Blocks unauthenticated invocations despite SECURITY DEFINER. |
| T-172.1-05 | E (Elevation of Privilege) | cross-tenant row read | mitigate | Primary SELECT replicates svg_templates SELECT RLS (167:39-45) verbatim: `WHERE id = p_template_id AND is_active = TRUE AND (tenant_id IS NULL OR created_by = v_user_id)`. super_admin fallback (D-10) gated on `profiles.role = 'super_admin'` only — reads profiles.role server-side, not any client-supplied value. |
| T-172.1-06 | T (Tampering) | SVG XSS via p_customized_svg | accept | RPC is a dumb persistor per Phase 172 D-10/D-17. Client-side DOMPurify (templateApplyService.js:54-56) and the 500KB `SVG_MAX_BYTES` cap (templateApplyService.js:32,47-49) are the defense boundary. Mirrors 168 threat model T-172-01 disposition. |
| T-172.1-07 | D (Denial of Service) | SVG payload size bomb | accept | No RPC-layer size cap. Compensating control: `SVG_MAX_BYTES=500_000` enforced client-side at templateApplyService.js:47-49 (D-12). An attacker with a raw SQL connection could bypass the cap, but that requires DB-level credentials; at the PostgREST boundary the client enforces. Accepted risk for ASVS L1. |
| T-172.1-08 | I (Information Disclosure) | svg_templates row leakage | mitigate | The RPC returns only `v_new_scene_id` (uuid). The template row is never returned to the client; only derived fields land in scene_slides under the caller's own tenant_id. |
| T-172.1-09 | R (Repudiation) | scene creation audit | accept | No explicit audit log; scenes.created_at + tenant_id = v_user_id provides a forensic trail. Matches 168 posture (no audit log added there either). |
