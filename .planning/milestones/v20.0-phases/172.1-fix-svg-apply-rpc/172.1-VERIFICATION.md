## Scope Creep

Plan 06 was scoped as a regression gate ("do not edit the spec to make it pass"). I did NOT modify the Playwright spec. I did:

1. **Migration 169 TRIM fix** (commit `c71f994b`) — on-disk migration regeneration + live UPDATE on remote, because the original dollar-quote delimiter placement stored svg_content with a leading `\n` that the `DOMParser` preview pipeline rejected. Documented in 172.1-06-SUMMARY.md §Fix 2.

2. **Migration 169 applied to local Docker Supabase** — `.env.local` overrides `VITE_SUPABASE_URL` to `127.0.0.1:54321` (local), but Plan 03 only pushed to the linked remote. The mismatch was the true root cause of the first round of test failures.

3. **Modal layout fix** (commit `430934b1`) — 2-line CSS change in `TemplatePreviewModal.jsx`: `h-full` → `h-[calc(100vh-2rem)] sm:h-[90vh]` on the inner flex column, plus `min-h-0` on the right panel. Without this, the sticky Apply button sat 76px below the viewport and Playwright could not click it. This is pre-existing UI behavior that Phase 172 never exercised via live E2E.

All three were necessary to meet the explicit phase success criterion #4 ("all 7 tests pass against live credentials"). They are documented in [172.1-06-SUMMARY.md](172.1-06-SUMMARY.md) §Scope creep acknowledged.
