## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local CLI → remote Supabase | `SUPABASE_ACCESS_TOKEN` env var authenticates the push. Token is scoped to the project owner. |
| push command → production DB | This migration is applied to the linked project (`gdxizdiltfqeugbsgtpx`), which is the production project per STATE.md §Decisions Phase 170. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-172.1-10 | T (Tampering) | applying wrong migrations | mitigate | `supabase db push --linked` applies only migrations listed in `supabase/migrations/` that are newer than the remote's `supabase_migrations.schema_migrations` table. The only two unapplied files after this phase's Wave 1 are 169 and 170; a dry-run/diff step confirms exactly these two before push. |
| T-172.1-11 | D (Denial of Service) | partial push leaves DB in broken state | mitigate | Each migration file is wrapped in a transaction by the Supabase CLI. If 169's ASSERT fails or 170 errors, the transaction rolls back and the DB remains at its pre-push state. Post-push smoke SELECT confirms both migrations registered. |
| T-172.1-12 | E (Elevation of Privilege) | running push with elevated credentials | accept | `SUPABASE_ACCESS_TOKEN` is required to push. Token storage is the developer's responsibility; this plan does NOT log or print the token. |
</threat_model>

```bash
  echo "=== fallback exit code: $? ==="

```
**Ordering is mandatory — 169 must run before 170.** If 169's ASSERT fails (raises `expected 0 active svg_templates rows with svg_content IS NULL...`), the transaction aborts and you must re-check Plan 01's UPDATE SQL against the seed-migration `svg_url` values before retrying. **Step 2 — Post-push smoke SELECTs** (append to the same evidence file):
```bash
{
  echo "=== Smoke: pg_proc has clone_svg_template_to_scene ==="
  supabase db query --linked \
    "SELECT proname, pronargs, prorettype::regtype AS returns FROM pg_proc WHERE proname = 'clone_svg_template_to_scene';" 2>&1

  echo "=== Smoke: 168 unchanged — clone_template_with_customization still exists ==="
  supabase db query --linked \
    "SELECT proname, pronargs FROM pg_proc WHERE proname = 'clone_template_with_customization';" 2>&1

  echo "=== Smoke: all 24 active svg_templates rows have svg_content ==="
  supabase db query --linked \
    "SELECT COUNT(*) AS total_active,
            COUNT(svg_content) AS with_content,
            COUNT(*) FILTER (WHERE svg_content IS NULL) AS missing_content
     FROM svg_templates
     WHERE is_active = TRUE;" 2>&1

  echo "=== Smoke: GRANT EXECUTE landed for authenticated role ==="
  supabase db query --linked \
    "SELECT grantee, privilege_type
     FROM information_schema.routine_privileges
     WHERE routine_name = 'clone_svg_template_to_scene' AND grantee = 'authenticated';" 2>&1

  date -u +"%Y-%m-%dT%H:%M:%SZ"
} | tee -a .planning/phases/172.1-fix-svg-apply-rpc/172.1-03-push-evidence.txt
```
