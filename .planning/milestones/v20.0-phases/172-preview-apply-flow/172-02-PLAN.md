<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client ↔ RPC | `p_customized_svg TEXT` is user-supplied (via Supabase JS client). The RPC writes it verbatim to `scene_slides.design_json.svgContent`. |
| auth.uid() ↔ SECURITY DEFINER | RPC runs as function owner (typically `postgres` role), bypassing RLS on read/write. Function MUST re-verify caller identity. |
| Free/Pro/Enterprise tiers ↔ template access | License gate prevents lower-tier users from cloning templates they are not entitled to. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-172-01 | Tampering | `p_customized_svg` — SVG XSS via `<script>` or `on*` handlers embedded in the SVG string | accept (at RPC layer) | Client-side DOMPurify sanitization in Plan 03 (`templateApplyService.applyTemplate`) is the sole mitigation. This RPC is a dumb persistor per D-10. RPC disposition = accept because mitigation is client-side. Defense-in-depth server sanitization would require a PG extension and is explicitly out-of-scope. |
| T-172-02 | Tampering | Clone-then-patch race — two separate writes from the client | mitigate | CLOSED BY DESIGN: single PL/pgSQL function = single transaction. The scene INSERT + slide-LOOP INSERTs are atomic. No separate UPDATE from the client path. |
| T-172-03 | Elevation of Privilege | License-tier bypass via direct RPC call | mitigate | Auth check at top (`auth.uid() IS NULL` → RAISE EXCEPTION). License-tier switch (free/pro/enterprise) inherited verbatim from migration 080:207-234. Super_admin override preserved. Any tier-mismatch raises exception before the scene INSERT runs. |
| T-172-04 | Denial of Service | Huge `p_customized_svg` payload | accept | PostgreSQL TEXT column accepts up to ~1GB. Practical signage SVGs are < 200KB. Client-side cap (≥500KB reject) enforced in Plan 03; no server-side cap here. Risk: low — single-tenant DB, authenticated callers, no public write path. |
| T-172-05 | Tampering | search_path injection | mitigate | `SET search_path = public` at function definition (mirrored from migration 080). Prevents any future table/function resolution against attacker-controlled schemas. |
| T-172-06 | Spoofing | CSRF via forged RPC call | mitigate | Supabase JS client attaches the JWT automatically; `SECURITY DEFINER` still re-verifies `auth.uid()`. No additional token needed. Inherited mitigation. |
| T-172-07 | Tampering | SQL injection in `p_customized_svg` | mitigate | Parameterized through `jsonb_set(..., to_jsonb(p_customized_svg))`. No string concatenation; `to_jsonb` treats the TEXT as a JSON string value. Low risk. |
</threat_model>
