## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-172-01 | Tampering | `applyTemplate`: user-crafted `<script>` / `on*` / `javascript:` URL in `customizedSvg` | mitigate | `DOMPurify.sanitize(customizedSvg, { USE_PROFILES: { svg: true, svgFilters: true } })` runs BEFORE the RPC call. Unit test (Task 2 Test 5) asserts the sanitizer is invoked AND the post-sanitization payload reaches the RPC. |
| T-172-04 | Denial of Service | `applyTemplate`: huge `customizedSvg` payload | mitigate | Client-side 500KB hard cap rejects oversize inputs before DOMPurify runs (avoids pathological sanitizer CPU cost too). Unit test (Task 2 Test 8) asserts the cap throws. |
| T-172-08 | Tampering | `editorRouteFor` — template-controlled route string | accept | Output is scoped to `svg-editor?sceneId={sceneId}` or `scene-editor-{sceneId}`. `sceneId` comes from the RPC return (server-generated uuid), not user input. Unknown `editor_type` throws rather than returning an attacker-controlled string. |
