## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px | 400 (regular) | 1.4 |
| Body | 14px | 400 (regular) | 1.5 |
| Heading | 16px | 600 (semibold) | 1.3 |
| Display | 20px | 600 (semibold) | 1.2 |

Exactly 2 weights declared: 400 (regular) and 600 (semibold). Size contrast (12px vs 14px) differentiates Label from Body without a third weight.

Notes:
- Label (12px/400) covers input sub-labels, color swatch labels, section group headers within QuickCustomizePanel
- Body (14px/400) covers QuickCustomizePanel field labels, Polotno info block body text, and nav counter ("3 / 12")
- Heading (16px/600) covers template name in the modal toolbar and panel section titles ("Colors", "Logo", "Text")
- Display (20px/600) is not used in the modal interior but is reserved for any full-screen hero text that may appear in edge-case empty state
- Source: matches existing `TemplateCard` body at `text-sm` (14px) and title at `font-medium text-sm`; toolbar `text-lg font-medium` from `LayoutPreviewModal.jsx` is overridden here to 16px semibold for smaller visual footprint in split view

Modal backdrop: `bg-black/85 backdrop-blur-sm` — matches `LayoutPreviewModal.jsx` pattern (`bg-black/90`) but softened to 85% to let the split layout breathe.

## Interaction Contract

### Modal Open / Close

| Trigger | Behavior |
|---------|----------|
| `TemplateCard.onSelect` | Opens modal with `initialIndex` = card's position in filtered results array |
| Escape key | Closes modal (delegated to design system `Modal` `closeOnEscape=true`) |
| Backdrop click | Does NOT close modal (split-view backdrop is structural — `closeOnOverlay=false`) |
| Close button (X) in toolbar | Closes modal |

### Prev / Next Navigation

| Trigger | Behavior |
|---------|----------|
| Left arrow button | Navigate to previous template; wraps to last if at index 0 |
| Right arrow button | Navigate to next template; wraps to first if at last index |
| Keyboard `ArrowLeft` | Same as left arrow button — active ONLY when modal is open AND no `<input>` or `<textarea>` is focused |
| Keyboard `ArrowRight` | Same as right arrow button — active only when modal open and no text input focused |
| Navigation (any method) | Silently resets QuickCustomizePanel to new template defaults (D-04 from CONTEXT.md) |
| Navigation (any method) | Cross-fades SVG preview using `fadeIn` motion primitive (150ms) |

Focus-trap strategy: Arrow key handler checks `document.activeElement.tagName` — if `INPUT` or `TEXTAREA`, skip the navigation binding. Escape propagates normally.

### Arrow Button Visual Treatment

- Ghost button variant wrapping `ChevronLeft` / `ChevronRight` (20px icon)
- Positioned absolute on left/right flanks of the preview pane: `absolute left-3 top-1/2 -translate-y-1/2` and `absolute right-3 top-1/2 -translate-y-1/2`
- Minimum hit area: 44×44px (`p-3` around 20px icon = 44px)
- Style: `bg-white/80 backdrop-blur-sm hover:bg-white shadow-card rounded-full text-gray-700 hover:text-gray-900 transition-all`
- Disabled appearance when only 1 template in filtered set: `opacity-40 pointer-events-none`
- Hidden entirely on mobile when nav happens via swipe (swipe is a Phase 174 enhancement — not in scope; buttons remain visible on mobile for this phase)

### QuickCustomizePanel Sections

Section order: **Colors → Logo → Text**. All sections always visible (no accordion/collapse). Scroll within the panel if content overflows.

#### Colors Section

- Header: "Colors" — 16px/600, `text-gray-900`, `mb-3`
- Up to 3 color controls: Primary Color, Secondary Color, Accent Color (only controls matching `data-customize-color` attributes present in the SVG are rendered; minimum 1, maximum 3)
- Each control: horizontal row — label (14px/400 text-gray-700 flex-1), color swatch button (32×32px rounded-md, `ring-offset-2`, active state: `ring-2 ring-brand-500`), "Clear" text button (12px/400 text-gray-400 hover:text-gray-700)
- Default fill: auto-prefill from `brandThemeService` active theme (`primary_color`, `secondary_color`, `accent_color`) where available; fall back to template's embedded default color if brand theme unavailable
- Color picker: native `<input type="color">` — absolutely positioned off-screen, triggered by swatch click. No third-party color picker library.
- Live preview update: `svgCustomizeService.swapColor()` called on every `onChange` event (debounced 50ms to prevent excessive re-renders)

#### Logo Section

- Header: "Logo" — same header style as Colors
- Current state: thumbnail of active logo from `brandThemeService.logoUrl` if available, else placeholder (`LayoutTemplate` icon 24px gray-300, `bg-gray-100 rounded-md w-16 h-16`)
- "Upload logo" button: `Button variant="secondary" size="sm"` — triggers file input (accept="image/*", max 2MB, enforced client-side before any upload)
- "Remove logo" link: 12px/400 text-gray-400 hover:text-red-600 — sets logo to null in SVG (calls `svgCustomizeService.replaceLogo(null)`); `aria-label="Remove logo"`
- Upload in progress: swatch area shows `Loader2` spinner (16px)

#### Text Section

- Header: "Text" — same header style
- Each text node matching `data-customize-text` rendered as: label (field name from attribute, 12px/400 text-gray-500) + `Input` component (14px, full-width, `size="sm"`)
- Maximum 4 text fields shown; additional fields clipped (out-of-scope controls exist for future phase)
- Live preview update: `svgCustomizeService.updateText()` called on `onBlur` (not onChange — avoids mid-word re-renders disrupting focus)

### Apply CTA

| State | Appearance |
|-------|------------|
| Idle | `Button variant="primary" size="lg" fullWidth` — "Apply to new scene" |
| Applying | `loading=true` prop on Button — shows `Loader2` spinner, label becomes "Applying…", button disabled |
| Success | Button stays disabled for 200ms (navigation begins); modal closes as router pushes to editor route |
| Failure | Button re-enables; `Alert variant="error"` appears above the button inside the panel (not a floating toast) |

Apply failure Alert:
- Text: "Couldn't apply template. Your customizations are saved — tap Apply to try again."
- Dismiss: `dismissible=true` on Alert; `onDismiss` clears the error state
- Position: `mt-4` above the Apply button at the bottom of the right panel

Scene naming: auto-derived as `"${template.name} scene"` — matches existing `installTemplateAsScene` convention. No user prompt. Scene name is passed as `p_scene_name` to the RPC.

### Loading States

| Scenario | Treatment |
|----------|-----------|
| Modal open, SVG not yet parsed | Preview pane: `Skeleton` block at `aspect-video` or `aspect-[9/16]` (matches template orientation); QuickCustomizePanel: 3 skeleton rows for color controls |
| Template navigation (prev/next) | Fade-out current SVG (100ms), show `Skeleton` in pane, fade-in new SVG once parsed (100ms); panel resets to new defaults immediately |
| Logo upload in progress | Swatch area shows `Loader2` spinner, "Upload logo" button disabled |
| Apply in progress | Button spinner (see above); nav arrows disabled (`pointer-events-none opacity-40`) |

### Thumbnail Fallback

| Scenario | Fallback |
|----------|----------|
| `thumbnail` URL 404 or load error | Gray `bg-gray-100` pane with `LayoutTemplate` icon (40px, `text-gray-300`) centered |
| SVG `svg_content` parse error | Same gray placeholder + inline label "Preview unavailable" (14px/400 text-gray-500) |
| No `svg_content` and no `thumbnail` | Same gray placeholder — no error thrown to user; Apply still works if template record is valid |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Modal role | `role="dialog" aria-modal="true"` — delegated to design system `Modal` component |
| Modal label | `aria-labelledby` pointing to template name heading in toolbar |
| Focus trap | Design system `Modal` restores focus on close (`previousActiveElement.current`) |
| Arrow nav buttons | `aria-label="Previous template"` / `aria-label="Next template"` |
| Keyboard nav | ArrowLeft/ArrowRight active when modal open and no input focused |
| Escape | Closes modal — provided by `Modal closeOnEscape=true` |
| Color swatches | `aria-label="Primary color: {hex}"` on swatch button; updates dynamically |
| Color input | `aria-hidden="true"` on the native `<input type="color">` (swatch button is the accessible control) |
| Apply button | `aria-busy="true"` when loading=true |
| Error Alert | `role="alert"` (built into design system Alert component) |
| Touch targets | All interactive controls ≥ 44×44px |
