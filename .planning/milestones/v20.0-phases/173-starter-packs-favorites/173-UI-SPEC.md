## Spacing Scale

Declared values (sourced from `src/design-system/tokens.css` â€” 8-point grid):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--space-1`) | Icon gaps, badge padding, heart-icon inset |
| sm | 8px (`--space-2`) | Compact element spacing, card info padding gap |
| md | 16px (`--space-4`) | Default element spacing, pack strip horizontal padding |
| lg | 24px (`--space-6`) | Section padding, modal body padding |
| xl | 32px (`--space-8`) | Layout gaps, strip-to-grid gap |
| 2xl | 48px (`--space-12`) | Major section breaks |
| 3xl | 64px (`--space-16`) | Page-level spacing |

Exceptions:
- Heart icon (`FavoriteButton`) touch target: minimum 48px Ã— 48px tap area (`--space-12`), even though the visual icon is 20px â€” apply invisible padding via `p-[14px]` on the wrapping button (20px icon + 28px padding = 48px total). Use `min-h-12 min-w-12` Tailwind classes. Note: the `14px` value is implementation arithmetic to reach a 48px grid-aligned tap target â€” it is NOT a reusable spacing token and must not be used elsewhere.
- Pack card mosaic tiles: gap between the 4 mosaic cells is 4px (`--space-1`, `gap-1`) to keep the grid tight and visually unified; this is the minimum grid-aligned value.
- Modal close button and nav arrows: 48px tap target (`--space-12`) â€” use `min-h-12 min-w-12` (Tailwind `h-12 w-12`).

## Typography

Sourced from `src/design-system/tokens.css`. Phase 173 uses a strict subset of 4 sizes and **2 weights** across all new components.

| Role | Size | Weight | Line Height | Token |
|------|------|--------|-------------|-------|
| Body / card description | 14px (`--text-sm`, `0.875rem`) | 400 (`--font-normal`) | 1.5 (`--leading-normal`) | `text-sm font-normal leading-normal` |

Notes:

- `font-medium` (500) is NOT used anywhere in this phase. Roles previously mapped to 500 are promoted to 600.

## Color

Sourced from `tailwind.config.js` + `src/design-system/tokens.css`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (`--color-bg-primary`) / `#fafafa` (`--color-bg-secondary`) | Page background, modal surface, card base, strip background |
| Secondary (30%) | `#f4f4f5` (`--color-bg-tertiary`) / `#e4e4e7` (`--color-border-primary`) | Pack card container, mosaic placeholder tiles, filter chip inactive, admin table rows |
| Accent (10%) | `#F26F26` (`brand-500`) | Heart icon filled state, "Apply pack" CTA button background, "Favorites" chip active state indicator, count badge background on pack card |
| Destructive | `#DC2626` (`--color-error-600`) | Delete pack action in admin (confirmation dialog button only) |

Accent (`#F26F26` / `brand-500`) is reserved for exactly:
1. `FavoriteButton` filled/active heart icon (filled red: use `text-brand-500`)
2. Primary CTA button in `PackPreviewModal` ("Apply pack (N templates)")
3. "Favorites" filter chip active/selected state â€” left border or background tint (`brand-50` bg + `brand-500` text)
4. Count badge on `PackCard` mosaic corner (`bg-brand-500 text-white`)
5. Focus ring on interactive elements (`shadow-focus-brand` from tailwind config)

Brand accent is NOT used on: generic card hover states, strip background, mosaic tiles, industry label badges (those use `Badge variant="default"` â€” gray).

Secondary semantic color note: The heart icon in its unfavorited state is `text-gray-400` (outline-only via lucide `Heart` with `fill-none stroke-current`). Favorited state: `text-brand-500 fill-current`. Error toast for favorites failure: uses `--color-error-500`.

## Accessibility Contracts

- `FavoriteButton`: `<button type="button">` with `aria-label` (see Copywriting), `aria-pressed={isFavorited}`. Keyboard: Space/Enter toggles.
- `PackPreviewModal`: focus trap active while open (same as `TemplatePreviewModal`). First focus target: close button. Escape closes.
- Pack modal close button: `aria-label="Close"` on the icon-only X button.
- Pack prev/next arrows: `aria-label="Previous pack"` / `aria-label="Next pack"`.
- Mosaic images: `alt={template.name}` for each of the 4 thumbnail cells. Placeholder tiles: `aria-hidden="true"`.
- Pack card: `<article>` with `aria-label={pack.name}`. Click target covers full card.
- Favorites filter chip: `role="checkbox"` or `aria-pressed` on the chip button.
- Count badge: visually visible; no additional aria (content is decorative supplement to the pack name).
