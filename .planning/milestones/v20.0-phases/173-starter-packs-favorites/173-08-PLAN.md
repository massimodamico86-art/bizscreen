```jsx
// Phase 173 TFAV-01 — optimistic favorite toggle
const handleToggleFavorite = useCallback(async (template, nextValue) => {
  // Optimistically flip in local state
  setAllTemplates((prev) =>
    prev.map((t) =>
      t.id === template.id && t.editor_type === template.editor_type
        ? { ...t, is_favorited: nextValue }
        : t
    )
  );
  try {
    await toggleFavorite({
      templateId: template.id,
      editorType: template.editor_type,
      nextValue,
    });
  } catch (err) {
    console.error('[TemplateGalleryPage] toggleFavorite failed:', err);
    // Revert
    setAllTemplates((prev) =>
      prev.map((t) =>
        t.id === template.id && t.editor_type === template.editor_type
          ? { ...t, is_favorited: !nextValue }
          : t
      )
    );
    showToast?.({ variant: 'error', message: 'Failed to update favorite' });
  }
}, [showToast]);
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL param ?favorites=1 → filter state | URL is browser-controlled but does not grant access — RLS on template_favorites is the actual control. Spoofed URL only changes UI rendering. |
| optimistic local state → server state | onToggleFavorite flips local allTemplates immediately; on error, reverts and toasts. |
| pack modal → page-level showToast / onNavigate | Modal calls these via prop; failure to wire (e.g. showToast undefined) is silent (optional chain). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-173-08-01 | Tampering | URL `?favorites=1` injected to view another user's favorites | mitigate | The filter is purely client-side over `is_favorited` column, which the gallery_templates_with_favorites VIEW computes per caller (RLS via auth.uid() in JOIN predicate). Other users' favorites cannot leak into the result set. |
| T-173-08-02 | Tampering | handleToggleFavorite race: rapid toggle on same row | mitigate | FavoriteButton's `if (busy) return` guard at the primitive level prevents per-row re-entry. Local optimistic state is the only mutation; on error reverts to opposite of `nextValue`. |
| T-173-08-03 | Information disclosure | Pack apply error message displayed inline | accept | Error message comes from PackPreviewModal — generic "Couldn't apply this pack." UX-friendly + no internal details surfaced. |
| T-173-08-04 | Tampering | TemplateCard onSelect fires when user clicks heart | mitigate | TemplateCard's existing button-guard at lines 60-64 + FavoriteButton's e.stopPropagation cover the case (covered in Plan 06 threat model — re-stated for traceability). |
| T-173-08-05 | Denial of service | Strip fetches packs on every TemplateGalleryPage mount | accept | Single-query, indexed by display_order/is_active; cost equivalent to existing template fetch. No caching layer needed at this scale. |
</threat_model>
