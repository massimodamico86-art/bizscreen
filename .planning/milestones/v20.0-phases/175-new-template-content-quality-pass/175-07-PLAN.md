**Step 1 — Run the canonical lint:**
```bash
grep -rnE "toHaveCount\(\s*[1-9]" tests/e2e/template-*.spec.js tests/e2e/gallery-*.spec.js 2>/dev/null | grep -v '^[^:]*:[[:space:]]*//' | grep -v '^[^:]*:[[:space:]]*\*' || echo "CLEAN"
```

If the output is `CLEAN` or empty: skip Step 2 — TQAL-05 already satisfied.
If the output contains matches: Step 2 transforms each.

**Step 2 — For each match, choose one of these transforms:**

(a) Replace `await expect(loc).toHaveCount(N)` (where N >= 1) with `await expect(loc.first()).toBeVisible()` plus a comment explaining why exact count is no longer asserted.

(b) If the count is meaningful for the test's intent (e.g., "exactly 3 chips"), use `await expect(loc).toHaveCount({ greaterThanOrEqual: N })` if Playwright supports it — actually Playwright does NOT support that variant. Instead use:
```javascript
const count = await loc.count();
expect(count).toBeGreaterThanOrEqual(N);
```

(c) If the test was checking "no results" (count = 0), KEEP `toHaveCount(0)` — that's the only allowed exact-count form per TQAL-05.

**Step 3 — Pitfall 4 audit (template-name pinning):**

Run:
```bash
grep -rnE "page\.goto\([^)]*svg-templates[^)]*\)" tests/e2e/template-*.spec.js tests/e2e/gallery-*.spec.js
```

For each navigation that does NOT include `?search=`, scan the lines below it for `getByText('<specific template name>')` or `getByRole('link', { name: '<name>' })`. If present, modify the goto URL to include `?search=<name-prefix>` so the test pins the target to page 1.

Example transform:
```javascript
// before
await page.goto(`${base}/?page=svg-templates`);
await expect(page.getByText('Restaurant Menu')).toBeVisible();

// after
await page.goto(`${base}/?page=svg-templates&search=Restaurant`);
await expect(page.getByText('Restaurant Menu')).toBeVisible();
```

**Step 4 — Replace ad-hoc grid-card assertions with the new helper:**

```javascript
const cards = page.locator('main h3').first();
```

```javascript
import { expectAtLeastOneTemplateCard } from './helpers.js';
// ...
await expectAtLeastOneTemplateCard(page);
```

This consolidates the structural contract in one place per RESEARCH Pattern 3.
