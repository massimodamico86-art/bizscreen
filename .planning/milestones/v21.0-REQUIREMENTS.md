# Requirements: BizScreen v21.0 Templates at Scale

**Defined:** 2026-05-06
**Core Value:** Screens reliably display the right content at the right time, even when offline.

**Milestone goal:** Reach ~500 active templates with OptiSigns-class breadth across three high-volume verticals (Restaurants & QSR, Retail & e-commerce, Healthcare & wellness), an admin-driven AI generation pipeline that keeps quality gated, and the gallery infrastructure required to keep search and scroll snappy at that catalog size.

**Production lanes for the catalog scale-up:** AI generation (admin-only) + hand-authoring. Third-party bulk SVG import was scoped out during requirements definition (deferred to v21.x with attribution form + display).

## Out of Scope (v21.0)

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Self-serve AI generation for end users | Quality control — admin-only pipeline this milestone; expand only after gen+validator quality is proven in production |
| Animated / dynamic templates (motion, Lottie, video) | Player rendering, thumbnail rasterization, and offline-player compatibility (WebOS/Tizen) all need work; out of scope per PROJECT.md |
| Brand Kit persistence across templates | Stateful brand kit needs new schema, UI, and changes to all apply RPCs; too large for v21.0 |
| User-submitted templates / marketplace | Moderation, IP clearance, payment rails are a separate workstream |
| Corporate vertical templates | Less differentiating than the 3 targeted verticals; would dilute authoring focus; defer |
| POS / ERP live data integration for menu boards | Existing Google Sheets widget already serves the live-data use case; a separate workstream from template content |
| Template versioning / rollback | No evidence target users request this; adds DB schema complexity |
| Phase 163 Worktree Merge Safeguard | Overdue tech debt but unrelated to template scaling — handle in a focused tech-debt phase |
| 11 v20.0 manual UAT items (Phases 172 + 174) | Partially obsoleted by `df2926e2` reversal (modal-side items no longer apply); re-audit at v21.0 close |
