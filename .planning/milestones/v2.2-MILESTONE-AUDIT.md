## Tech Debt Inventory

### Phase 31: Human Verification Deferred

The following manual tests were NOT performed:

- [ ] Fresh user flow through all 5 steps
- [ ] Skip flow with confirmation dialog
- [ ] Resume flow with returning users
- [ ] Feature flag off (old flow unchanged)
- [ ] Animation smoothness

**Recommendation:** Run manual verification before enabling `VITE_USE_UNIFIED_ONBOARDING=true` in production.

### Phase 33: Screenshot Proof Deferred

**What was planned:** Show screenshot from real device when screen paired

**What exists:** Text message "Your content is now live!" (no image)

**Why deferred:** Research confirmed screenshot capture requires 30+ seconds (device polling, content load, capture, upload). Infeasible for immediate post-pairing display.

**Impact:** Low — users still receive clear confirmation via text and confetti celebration.

### Phase 35: Human Verification Needed

Visual tests requiring human interaction:

- [ ] Editor loads within 10 seconds with template preview
- [ ] Unsaved changes dialog triggers on close with dirty state
- [ ] Mobile warning banner appears on small viewports
- [ ] Timeout error recovery with retry/fallback buttons

### Minor Improvement Opportunity

**Issue:** `get_unified_onboarding_state()` RPC doesn't return `screen_pairing_completed_at` field

**Impact:** SuccessStep shows generic message instead of "Your content is now live!" even when screen was paired

**Fix:** Add `screen_pairing_completed_at` to RPC response

**Severity:** LOW — cosmetic only
