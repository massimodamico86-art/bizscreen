### Broken Flows (2/9)

#### Flow 1: Document Upload → Display on Screen (Critical)

- **Breaks at:** `src/pages/hooks/useMediaLibrary.js` — `handleUploadSuccess` (line 250)
- **Issue:** `documentService.uploadDocument()` is never called from the media upload flow. The `doc-converter` Edge Function is never invoked. When a user uploads a PDF/Office file, it gets stored as a raw file. `config_json.convertedPages` is never populated, so `DocumentWidget` shows empty/pending state.
- **Affected requirements:** DOC-01, DOC-02, DOC-03, DOC-06
- **Fix:** Add conditional branch in `handleUploadSuccess` — when `mediaType === MEDIA_TYPES.DOCUMENT`, call `documentService.uploadDocument()` instead of `createMediaAsset()` directly

#### Flow 2: Embed Widget in Layout Zone → Plays on Screen (Moderate)

- **Breaks at:** `src/player/components/ZonePlayer.jsx` (line 116)
- **Issue:** `ZonePlayer` uses a mediaType-based dispatch that predates the widget registry. It has no render branch for `youtube`, `vimeo`, or `google-slides` widget types. Embed widgets work correctly in scene editor (via `SceneRenderer` + `getWidgetComponent()`), but render as blank placeholders in layout zones.
- **Affected requirements:** EMBED-01, EMBED-02, EMBED-03, EMBED-05, EMBED-06, EMBED-07, SLIDES-01, SLIDES-02, SLIDES-03
- **Fix:** Add widget-type dispatch in `ZonePlayer` using `getWidgetComponent()` from registry, matching `SceneRenderer`'s pattern
