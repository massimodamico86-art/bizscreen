# External Integrations

**Analysis Date:** 2026-01-29

## APIs & External Services

**Authentication & Database:**
- Supabase - Postgres database, authentication, real-time subscriptions
  - SDK/Client: `@supabase/supabase-js` (2.80.0)
  - Auth: `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY` (required)
  - Client: `src/supabase.js` (singleton instance)
  - Features: PKCE auth flow, auto-refresh tokens, session persistence
  - RLS: Row Level Security policies enforced in database

**Payment Processing:**
- Stripe - Subscription billing and payment processing
  - SDK/Client: `stripe` (20.0.0) for frontend
  - Auth: `VITE_STRIPE_PUBLISHABLE_KEY` (production-required)
  - Implementation: `src/services/billingService.js`
  - Features: Checkout sessions, billing portal, subscription management
  - API endpoints: `/api/billing/checkout`, `/api/billing/portal` (custom Vite middleware)

**Weather Data:**
- OpenWeatherMap - Weather widget data
  - SDK/Client: Direct fetch API calls
  - Auth: `VITE_OPENWEATHER_API_KEY` (optional)
  - Implementation: Weather components in `src/components/WeatherWall/`
  - Free tier: 1,000 calls/day

**Email Delivery:**
- Resend - Email notifications and alerts
  - SDK/Client: `resend` (6.8.0)
  - Auth: `VITE_RESEND_API_KEY` (optional)
  - Implementation: Email service integration
  - From address: alerts@bizscreen.com
  - Use case: Device alerts, status notifications

## Data Storage

**Databases:**
- Supabase PostgreSQL
  - Connection: `VITE_SUPABASE_URL` (required)
  - Client: `@supabase/supabase-js`
  - Migrations: `supabase/migrations/` (135+ migration files)
  - Local development: Supabase CLI with local postgres

**File Storage:**
- AWS S3 - Media file storage (images, videos, documents)
  - Connection: `AWS_REGION`, `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_S3_BUCKET`
  - Client: `@aws-sdk/client-s3` (3.946.0)
  - Implementation: `src/services/s3UploadService.js`
  - Upload strategy: Presigned URLs generated by custom API route (`/api/media/presign`)
  - Folder structure: `uploads/{mediaType}/{uuid}.{ext}` where mediaType = image|video|audio|document
  - Supported formats: Images (jpeg, png, gif, webp, svg), Videos (mp4, webm, mov, avi), Audio (mp3, wav, ogg), Documents (pdf, docx, pptx, xlsx)
  - Optional CDN: CloudFront (`AWS_CLOUDFRONT_URL`)

**Offline Storage:**
- IndexedDB - Client-side offline data caching
  - Client: `idb` (8.0.3)
  - Implementation: `src/player/offlineService.js`
  - Use case: TV player offline mode

**Caching:**
- Browser LocalStorage - Session data, user preferences
- IndexedDB - Large offline datasets for player

## Authentication & Identity

**Auth Provider:**
- Supabase Auth - Primary authentication system
  - Implementation: `src/contexts/AuthContext.jsx`
  - Flow type: PKCE (Proof Key for Code Exchange)
  - Features: Email/password, magic links, session management
  - Tokens: JWT with auto-refresh
  - User metadata: Stored in Supabase auth.users table

**Authorization:**
- Role-based access control (RBAC)
  - Roles: super_admin, reseller, admin, client, screen_viewer
  - Implementation: `src/services/permissionsService.js`
  - Enforcement: Row Level Security (RLS) in Supabase + client-side checks

## Monitoring & Observability

**Error Tracking:**
- Sentry - Error monitoring and performance tracking
  - Client: `@sentry/react` (10.36.0)
  - Auth: `VITE_SENTRY_DSN` (optional, production recommended)
  - Implementation: `src/utils/errorTracking.jsx`
  - Provider config: `VITE_ERROR_TRACKING_PROVIDER` (console | sentry)
  - Features: Exception capture, breadcrumbs, session replay, performance tracing
  - Sampling: 10% trace rate in production, 10% replay rate, 100% error replay
  - Environment: Tracks via `VITE_APP_ENV` (local | staging | production)
  - Fallback: Console provider for local development

**Logs:**
- Centralized logging service: `src/services/loggingService.js`
  - Scoped loggers per module
  - Levels: debug, info, warn, error
  - Remote endpoint: `VITE_LOG_ENDPOINT` (optional)
  - Production: Console statements stripped by terser in build

**Performance:**
- Web Vitals - Performance metrics
  - Client: `web-vitals` (5.1.0)
  - Metrics: CLS, FID, LCP, FCP, TTFB
  - Analytics endpoint: `VITE_ANALYTICS_ENDPOINT` (optional)

## Environment Configuration

**Required env vars:**
- `VITE_SUPABASE_URL` - Supabase project URL (https://[project].supabase.co)
- `VITE_SUPABASE_ANON_KEY` - Supabase anonymous key (JWT)
- `AWS_REGION` - AWS region for S3 (server-side)
- `AWS_ACCESS_KEY_ID` - AWS access key (server-side)
- `AWS_SECRET_ACCESS_KEY` - AWS secret key (server-side)
- `AWS_S3_BUCKET` - S3 bucket name (server-side)

**Production-required:**
- `VITE_STRIPE_PUBLISHABLE_KEY` - Stripe publishable key (pk_live_* or pk_test_*)
- `VITE_CLOUDINARY_CLOUD_NAME` - Cloudinary cloud name (legacy, may be deprecated)

**Optional:**
- `VITE_OPENWEATHER_API_KEY` - Weather API key
- `VITE_SENTRY_DSN` - Sentry project DSN
- `VITE_ERROR_TRACKING_ENABLED` - Enable/disable error tracking
- `VITE_RESEND_API_KEY` - Resend email API key
- `VITE_DEVICE_TOKEN` - TV device pairing token
- `AWS_CLOUDFRONT_URL` - CloudFront CDN URL
- `VITE_API_BASE_URL` - Override API base URL
- `VITE_APP_ENV` - Environment (local | staging | production)
- `VITE_ENABLE_AI` - Enable AI features
- `ANTHROPIC_API_KEY` - Claude API key for AI auto-tagging (server-side)

**Secrets location:**
- Development: `.env` and `.env.local` files (gitignored)
- Production: Hosting platform environment settings
- Test: `.env.example` provides template with test credentials

**Validation:**
- Centralized: `src/config/env.js`
- Strategy: Fail fast in production, warn in development
- Pattern validation: URL formats, key prefixes (pk_live_, etc.)

## Webhooks & Callbacks

**Incoming:**
- Stripe webhooks (payment events)
  - Endpoint: `/api/billing/webhook` (serverless function needed)
  - Signature verification: Required in production
  - Events: checkout.session.completed, customer.subscription.updated, etc.

**Outgoing:**
- Custom webhook system
  - Implementation: `src/services/webhookService.js`
  - Events: device.online, device.offline, campaign.activated, campaign.ended, content.approved, etc.
  - Features: Retry logic (max 5 attempts with exponential backoff), signature generation (HMAC), dead letter queue for failed deliveries
  - Storage: Webhook endpoints and delivery history in Supabase
  - Validation: HTTPS-only, blocks private/local IPs
