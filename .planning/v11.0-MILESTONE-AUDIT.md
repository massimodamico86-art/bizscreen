### Active Crash Bugs Found (2)

These are the same `action={object}` pattern that Phase 104 fixed in DemoToolsPage but missed in two other files:

**BUG-A: `ResellerDashboardPage.jsx:207`**
- EmptyState `action` receives plain `{ label, onClick }` object instead of JSX
- Crash condition: any user navigating to reseller-dashboard without existing resellerAccount
- Fix: convert to `action={<Button onClick={...}>Apply Now</Button>}`

**BUG-B: `ScreenGroupsPage.jsx:297`**
- EmptyState `action` receives `canEdit && { label, onClick, icon }` (evaluates to object when canEdit=true)
- Crash condition: admin/editor user with no screen groups (common new-account state)
- Fix: convert to `action={canEdit ? <Button ...>Create First Group</Button> : undefined}`
