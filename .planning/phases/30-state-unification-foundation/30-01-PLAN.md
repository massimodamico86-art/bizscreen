Create migration file `139_unified_onboarding_state.sql` with:

**Schema Extensions:**
```sql
-- Add columns to onboarding_progress table
ALTER TABLE public.onboarding_progress
  ADD COLUMN IF NOT EXISTS current_unified_step TEXT DEFAULT 'welcome_tour',
  ADD COLUMN IF NOT EXISTS onboarding_version INTEGER DEFAULT 2,
  ADD COLUMN IF NOT EXISTS screen_pairing_completed_at TIMESTAMPTZ;
```

**Column Comments:**
- `current_unified_step`: Current position in unified onboarding flow (welcome_tour, industry_selection, starter_pack, screen_pairing, complete)
- `onboarding_version`: Schema version for handling future migrations (2 = v2.2 unified flow)
- `screen_pairing_completed_at`: Timestamp when screen pairing step was completed (for activation tracking)

**RPC Function: get_unified_onboarding_state**
- Returns JSONB with:
  - `current_step`: TEXT (current unified step)
  - `can_resume`: BOOLEAN (true if not complete and not skipped)
  - `progress_percent`: INTEGER (0-100 based on completed steps)
  - `is_complete`: BOOLEAN
  - `skipped_at`: TIMESTAMPTZ or NULL
- Calculates progress_percent from existing completed_* booleans
- Uses SECURITY DEFINER with auth.uid()

**RPC Function: advance_onboarding_step**
- Parameters: p_completed_step TEXT
- Validates step is in allowed list: ('welcome_tour', 'industry_selection', 'starter_pack', 'screen_pairing', 'complete')
- Updates current_unified_step to next step in sequence
- If completing screen_pairing, also sets screen_pairing_completed_at = now()
- If advancing to 'complete', also sets is_complete = true
- Returns JSONB with success, current_step, next_step, is_complete

**RPC Function: complete_unified_onboarding**
- Sets is_complete = true, current_unified_step = 'complete'
- Sets completed_at = now() if not already set
- Returns JSONB with success

**Permissions:**
- GRANT EXECUTE to authenticated role for all three functions

**Index:**
- CREATE INDEX on current_unified_step WHERE current_unified_step != 'complete'

Follow the exact pattern from migrations 136 and 137 for:
- Header comment format
- SECURITY DEFINER functions
- ON CONFLICT handling for upserts
- Completion notice at end
