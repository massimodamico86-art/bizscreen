1. Create `src/components/RouteErrorBoundary.jsx` -- a lightweight React class component error boundary designed for per-route use:
   - Catches errors via getDerivedStateFromError and componentDidCatch
   - Shows a compact fallback UI (not full-page like the root ErrorBoundary): centered card with warning icon, "This section encountered an error" message, error message text, and a "Try Again" button that resets the boundary state
   - Also include a "Go to Dashboard" link that calls `window.__setCurrentPage?.('dashboard')` as a secondary CTA
   - Accept an optional `name` prop for identifying which section crashed (shown in the error message)
   - Call `handleReactError` from `../utils/errorTracking.jsx` in componentDidCatch for error reporting
   - Export as both named and default export

2. In `src/App.jsx`, import `RouteErrorBoundary` and wrap every page entry in the `pages` map object. The pattern changes from:
   ```
   dashboard: <Suspense fallback={<PageLoader />}><DashboardPage ... /></Suspense>,
   ```
   to:
   ```
   dashboard: <RouteErrorBoundary name="Dashboard"><Suspense fallback={<PageLoader />}><DashboardPage ... /></Suspense></RouteErrorBoundary>,
   ```
   Apply this pattern to ALL entries in the `pages` object (~70 entries) AND all dynamic route entries in the fallback chain (playlist-editor, layout-editor, schedule-editor, campaign-editor, admin-tenant, scene-detail, screen-group-detail, scene-editor, admin-template, yodeck-layout-preview, yodeck-layout, design-editor, svg-editor).

   Do NOT change the root ErrorBoundary in main.jsx -- it remains as the last-resort catch-all.

1. Create `src/hooks/useNetworkStatus.js`:
   - Track three states: 'online', 'offline', 'reconnecting'
   - Use `navigator.onLine` for initial state
   - Listen to window 'online' and 'offline' events
   - When transitioning from offline to online, set state to 'reconnecting' for 2 seconds (to confirm connection stability), then set to 'online'
   - Return `{ status, isOnline, isOffline, isReconnecting }`
   - Clean up event listeners on unmount

   - Shows a small pill/badge in the header:

     - **reconnecting**: Yellow/amber pill with "Reconnecting..." text and a loader icon (Loader2 with animate-spin)
     - **online**: Green pill with "Online" text that auto-fades after 3 seconds (only shown after recovering from offline/reconnecting, not on initial load)
   - Use Tailwind classes consistent with the existing design system (rounded-full, text-xs, px-2 py-1 style pills)
