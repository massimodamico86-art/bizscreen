    **Data wiring fix — fetchLocations return shape:**
    The `loadData` function (line 111-113) calls:
    ```js
    const [groupsData, locationsData, tagsData] = await Promise.all([
      fetchScreenGroupsWithScenes(),
      fetchLocations(),
      fetchAllGroupTags()
    ]);
    ```
    Then does `setLocations(locationsData)` (line 127).

    BUT `fetchLocations()` from locationService returns `{ data: [...], error: ... }` — NOT a plain array. This means `locations` state will be `{ data: [...], error: null }` instead of an array, causing the location `<select>` `locations.map(...)` to fail silently or crash.

    **Fix:** Change `setLocations(locationsData)` to `setLocations(locationsData?.data || locationsData || [])`. Use defensive extraction: if the result has a `.data` property, use that; otherwise treat the result itself as the array (for backward compatibility).
