**EmptyState.jsx -- Make icon rendering handle both component refs and JSX elements:**

In the `icon` rendering block (line 63-66), replace the direct `{icon}` child render with logic that detects whether `icon` is a React component (function or forwardRef object) vs. an already-instantiated JSX element:

```jsx
{icon && (
  <div className={`${styles.icon} text-gray-300 mb-4`}>
    {typeof icon === 'function'
      ? React.createElement(icon, { className: `${styles.icon}`, 'aria-hidden': 'true' })
      : (icon?.$$typeof
        ? React.createElement(icon, { className: `${styles.icon}`, 'aria-hidden': 'true' })
        : icon)}
  </div>
)}
```

The logic:
- `typeof icon === 'function'` catches regular function components
- `icon?.$$typeof` catches forwardRef objects (which have $$typeof: Symbol(react.forward_ref))
- Otherwise it's already a JSX element -- render as-is

Import `createElement` from React at the top: `import { forwardRef, createElement } from 'react';`

Do NOT change the existing icon prop API or any call sites. The fix is purely in EmptyState -- both patterns (`icon={Users}` and `icon={<Users />}`) must work after this fix.
