**DataTableWidget.jsx:**
1. Import `{ useWidgetData }` from `../../hooks/useWidgetData.js` and `{ SyncStatusIndicator }` from './SyncStatusIndicator.jsx'.
2. Import `{ getDataSource }` and `{ cacheDataSource, getCachedDataSource }` remain for use as fetchFn and cacheFn.
3. Define sourceKey: `const sourceKey = dataSourceId ? \`data-source:\${dataSourceId}\` : null;`
4. Define fetchFn: `const fetchFn = useCallback(async () => { const result = await getDataSource(dataSourceId); return result; }, [dataSourceId]);`
5. Define cacheFn: `const cacheFn = useCallback(async (key, data) => { await cacheDataSource(dataSourceId, data); }, [dataSourceId]);`
6. Call: `const { data: orchestratorData, lastFetchedAt, isLoading } = useWidgetData(sourceKey, fetchFn, refreshIntervalMinutes * 60 * 1000, cacheFn);`
7. REMOVE the entire "Initial data load" useEffect (lines ~57-93 in current file). REMOVE the entire "Refresh timer" useEffect (lines ~96-128). These are replaced by the orchestrator.
8. Add a fallback for offline: if orchestratorData is null on mount, try getCachedDataSource as a one-time fallback:
   ```
   useEffect(() => {
     if (!dataSourceId || orchestratorData) return;
     let cancelled = false;
     getCachedDataSource(dataSourceId).then(cached => {
       if (!cancelled && cached && !orchestratorData) setData(cached);
     }).catch(() => {});
     return () => { cancelled = true; };
   }, [dataSourceId, orchestratorData]);
   ```
9. Use `orchestratorData || data` (where data is the local cache fallback state) as the active data source for rendering.
10. Add a fade transition for data REFRESH (not pagination, which already has its own fade from Plan 02). When orchestratorData reference changes (new fetch), apply a brief 200ms opacity dip. Use a `dataVersion` ref that increments when orchestratorData changes. Only animate when dataVersion > 1 (skip first load). DataTable needs this because a full table of rows swapping at once is visually jarring without it.
11. Wrap the entire widget return in a `<div style={{ position: 'relative', width: '100%', height: '100%' }}>` and add `<SyncStatusIndicator lastRefreshedAt={lastFetchedAt} />` as a sibling after the table content.
