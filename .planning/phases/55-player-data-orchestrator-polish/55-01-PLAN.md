Create `src/player/contexts/DataRefreshContext.js`:
- Create and export a React context `DataRefreshContext` with default value null
- Export a `DataRefreshProvider` component that wraps children with the context provider, receiving `value` prop from the orchestrator hook

Create `src/player/hooks/useDataRefreshOrchestrator.js`:
- Export `useDataRefreshOrchestrator()` hook that returns `{ register, getData, getAll }`
- Internal state: `registryRef` (useRef Map) keyed by sourceKey (string). Each entry: `{ fetchFn, intervalMs, cacheFn, lastFetchedAt, isLoading, subscriberCount }`
- Internal state: `dataStoreRef` (useRef Map) keyed by sourceKey. Each entry: `{ data, lastFetchedAt, isLoading, error }`
- Internal state: `versionRef` (useRef number) + `[version, setVersion] = useState(0)` -- increment version to trigger re-renders when data changes
- Master tick loop: single `setInterval` at 10_000ms (10 seconds). On each tick, iterate registryRef entries. For each entry where `Date.now() - entry.lastFetchedAt >= entry.intervalMs` and `!entry.isLoading` and `entry.subscriberCount > 0`: set isLoading=true, call fetchFn(), on success update dataStoreRef with new data + lastFetchedAt=Date.now(), call cacheFn if provided, increment version. On error, log warning, keep existing data. Always set isLoading=false after.
- Add jitter on registration: when a source registers, set its `lastFetchedAt` to `Date.now() - intervalMs + random(0, 30_000)` so not all sources with the same interval fire simultaneously. But ALSO do an immediate fetch on first registration (subscriberCount goes from 0 to 1).
- `register(sourceKey, fetchFn, intervalMs, cacheFn)`: If sourceKey already in registry, just increment subscriberCount and return unregister function. If new, add to registry with subscriberCount=1, do immediate fetch, add jitter offset for subsequent refreshes. Return `unregister` function that decrements subscriberCount; when subscriberCount reaches 0, stop polling that source (but keep cached data in dataStoreRef).
- `getData(sourceKey)`: Read from dataStoreRef -- returns `{ data, lastFetchedAt, isLoading, error }` or `{ data: null, lastFetchedAt: null, isLoading: false, error: null }` if not found.
- `getAll()`: Return the full dataStoreRef Map (for debugging).
- Cleanup: clear the master interval on unmount.

Create `src/player/hooks/useWidgetData.js`:
- Export `useWidgetData(sourceKey, fetchFn, intervalMs, cacheFn)` hook
- Calls `useContext(DataRefreshContext)` to get orchestrator
- On mount (or when sourceKey/fetchFn/intervalMs change): call `orchestrator.register(sourceKey, fetchFn, intervalMs, cacheFn)`. Store the returned unregister function. On cleanup, call unregister.
- Return `orchestrator.getData(sourceKey)` -- i.e. `{ data, lastFetchedAt, isLoading, error }`
- If orchestrator is null (context not provided, e.g. widget rendered outside provider), fall back gracefully: do a standalone fetch on mount and return the data without orchestration. Log a warning.

Update `src/player/hooks/index.js`:
