**src/main.jsx changes:**
1. Add `import { initObservability } from './utils/observability';` at top of file.
2. Call `initObservability();` BEFORE the `createRoot()` call (line ~36). This must happen before React renders so Sentry captures errors during initial render.
3. Remove the entire `window.addEventListener('unhandledrejection', ...)` block (lines 14-34). This is now handled by `setupGlobalErrorHandlers()` inside `errorTracking.jsx` which is called via `initObservability()`. Keeping both causes duplicate error reports.
4. Add `import * as Sentry from '@sentry/react';` at top.
5. Change the `createRoot` call to pass React 19 error hook options:
   ```javascript
   const root = createRoot(document.getElementById("root"), {
     onUncaughtError: Sentry.reactErrorHandler((error, errorInfo) => {
       console.warn("Uncaught error", error, errorInfo.componentStack);
     }),
     onCaughtError: Sentry.reactErrorHandler(),
     onRecoverableError: Sentry.reactErrorHandler(),
   });
   root.render(
     <React.StrictMode>
       ...existing providers...
     </React.StrictMode>
   );
   ```
   This ensures errors that bypass ErrorBoundary (async errors, event handlers) are still captured by Sentry.

**src/utils/errorTracking.jsx changes:**

   ```javascript
   import {
     useLocation,
     useNavigationType,
   ```

2. In the `sentry.init()` method (inside `providers.sentry.init`), replace the `Sentry.browserTracingIntegration()` in the integrations array with:

   ```javascript
     /^safari-extension:\/\//i,
   ],
   ```
