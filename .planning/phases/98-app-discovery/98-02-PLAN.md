# Plan 98-02: Authenticated App Pages Discovery

## Goal

Navigate to every authenticated page inside the BizScreen app (the `/app` shell), take a screenshot of each page's initial load state, and capture a browser_snapshot accessibility tree for each to discover all interactive elements. This covers all ~63 static page IDs in the `pages` object plus dynamic editor routes.

## Context

- The dev server should already be running from plan 98-01 at `http://localhost:5173`
- If it is NOT running, start it with `npm run dev` via Bash (run_in_background: true), wait 5 seconds
- VITE_DEV_BYPASS_AUTH=true is set in `.env.local` -- navigating to `/app` auto-logs in as a mock dev user (client role)
- The app uses React state-based navigation inside `/app/*` -- pages are NOT separate URL paths. To navigate, click sidebar items or use browser_evaluate to call `setCurrentPage('page-id')`
- The mock user has role=client, plan=enterprise (all features enabled)
- All screenshots go to `./screenshots/` with prefix `98-`
- Use MCP Playwright tools: browser_navigate, browser_take_screenshot, browser_snapshot, browser_click, browser_evaluate

## How to Navigate to App Pages

The app's `/app` route renders a sidebar + content area. Pages are switched by setting the `currentPage` React state. Two approaches:

**Approach A (preferred -- most reliable):** Use `browser_click` on sidebar navigation items visible in the `browser_snapshot` accessibility tree.

**Approach B (for pages not in sidebar):** Use `browser_evaluate` to execute JavaScript that dispatches navigation:
```js
// Find the React fiber root and call setCurrentPage
// This works because setCurrentPage is passed down through the component tree
() => {
  // Use the window.__setCurrentPage if exposed, or click programmatically
  const event = new CustomEvent('navigate', { detail: { page: 'PAGE_ID' } });
  window.dispatchEvent(event);
}
```
If Approach B does not work (no global navigation hook), navigate by finding and clicking links/buttons in the UI that lead to each page. Many pages are accessible from the Header user menu, settings dropdowns, or from links within other pages.
