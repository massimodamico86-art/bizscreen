# Plan 98-01: Public Routes & Auth Pages Discovery

## Goal

Navigate to every public (unauthenticated) route in the BizScreen app, take a screenshot of each page's initial load state, and capture a browser_snapshot accessibility tree for each to discover all interactive elements.

## Context

- The dev server runs on `http://localhost:5173` (Vite default port)
- Start the dev server with `npm run dev` in background before navigating
- VITE_DEV_BYPASS_AUTH=true is set in `.env.local` -- the login page will auto-redirect to `/app` when visited directly. To screenshot the login page itself, navigate to `/auth/login` (it renders before redirect fires)
- All screenshots go to `./screenshots/` with prefix `98-`
- Public routes do NOT require authentication
- Use MCP Playwright tools: browser_navigate, browser_take_screenshot, browser_snapshot

## Public Routes to Visit

These are all routes in `src/router/AppRouter.jsx` that do not require authentication:

| # | URL Path | Page Name | Notes |
|---|----------|-----------|-------|
| 1 | `/` | Homepage (Marketing) | PublicRoute wrapper -- redirects if logged in |
| 2 | `/features` | Features Page | Marketing layout |
| 3 | `/pricing` | Pricing Page | Marketing layout |
| 4 | `/auth/login` | Login Page | PublicRoute wrapper -- may auto-redirect with dev bypass |
| 5 | `/auth/signup` | Signup Page | PublicRoute wrapper |
| 6 | `/auth/reset-password` | Reset Password Page | No auth wrapper |
| 7 | `/auth/update-password` | Update Password Page | No auth wrapper |
| 8 | `/auth/accept-invite` | Accept Invite Page | No auth wrapper, needs token param |
| 9 | `/preview/test-token` | Public Preview Page | Token-based access, no auth |

## Tasks

<task id="98-01-01">
<title>Start dev server and verify it is running</title>
<steps>
1. Run `npm run dev` in background using the Bash tool with `run_in_background: true`
2. Wait 5 seconds for the server to start
3. Use `browser_navigate` to go to `http://localhost:5173`
4. Verify the page loads by taking a `browser_snapshot` -- confirm it shows the BizScreen homepage content
5. If the server is not running or the page does not load, check the terminal output and retry
</steps>
<verify>The homepage at `http://localhost:5173` loads and the browser_snapshot shows page content (not a connection error)</verify>
</task>

<task id="98-01-02">
<title>Screenshot and snapshot all public routes</title>
<steps>
For each of the 9 public routes listed above, perform these steps in order:

**Route 1: Homepage `/`**
1. browser_navigate to `http://localhost:5173/`
2. Wait for the page to fully load (wait for text "BizScreen" or similar heading)
3. browser_take_screenshot with filename `screenshots/98-01-homepage.png`
4. browser_snapshot -- save the output mentally, noting all buttons, links, and interactive elements

**Route 2: Features `/features`**
1. browser_navigate to `http://localhost:5173/features`
2. browser_take_screenshot with filename `screenshots/98-02-features.png`
3. browser_snapshot

**Route 3: Pricing `/pricing`**
1. browser_navigate to `http://localhost:5173/pricing`
2. browser_take_screenshot with filename `screenshots/98-03-pricing.png`
3. browser_snapshot

**Route 4: Login `/auth/login`**
1. browser_navigate to `http://localhost:5173/auth/login`
2. NOTE: Dev bypass may cause auto-redirect to `/app`. If this happens, the screenshot will show the dashboard instead. Take the screenshot regardless to document the behavior.
3. browser_take_screenshot with filename `screenshots/98-04-login.png`
4. browser_snapshot

**Route 5: Signup `/auth/signup`**
1. browser_navigate to `http://localhost:5173/auth/signup`
2. browser_take_screenshot with filename `screenshots/98-05-signup.png`
3. browser_snapshot

**Route 6: Reset Password `/auth/reset-password`**
1. browser_navigate to `http://localhost:5173/auth/reset-password`
2. browser_take_screenshot with filename `screenshots/98-06-reset-password.png`
3. browser_snapshot

**Route 7: Update Password `/auth/update-password`**
1. browser_navigate to `http://localhost:5173/auth/update-password`
2. browser_take_screenshot with filename `screenshots/98-07-update-password.png`

**Route 8: Accept Invite `/auth/accept-invite`**

2. browser_take_screenshot with filename `screenshots/98-09-public-preview.png`
3. browser_snapshot
</steps>
