# Plan 99-03: Logout Flow & Auth Transition States

## Goal

Walk through the complete logout flow with screenshots, capture all remaining auth transition states (post-logout landing, authentication loading spinner, auth callback page), and compile a summary of all auth flow screenshots taken across plans 99-01 through 99-03.

## Context

- The dev server should already be running from prior plans. Restart if needed.
- For logout testing, the dev server needs `VITE_DEV_BYPASS_AUTH=true` so we have an authenticated session to sign out from
- After logout, switch back to `VITE_DEV_BYPASS_AUTH=false` to verify the post-logout state shows the login page
- All screenshots go to `./screenshots/` with prefix `99-`
- Use MCP Playwright tools: browser_navigate, browser_take_screenshot, browser_snapshot, browser_click
- The sign-out button is in the app header (a `LogOut` icon button with title="Sign out" next to the user avatar)
- After sign out, `signOut()` in AuthContext calls `supabase.auth.signOut()` and clears the user state, which triggers `RequireAuth` to redirect to `/auth/login`
- The auth callback page at `/auth/callback` handles OAuth/email-link redirect flows

## Auth Routes Involved

| # | URL Path | Page Name | Notes |
|---|----------|-----------|-------|
| 1 | `/app` | Dashboard (authenticated) | Starting point for logout flow |
| 2 | `/auth/login` | Login Page | Post-logout landing page |
| 3 | `/auth/callback` | Auth Callback | Handles OAuth/email-link redirects |
