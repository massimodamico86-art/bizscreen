# Plan 99-02: Signup & Password Reset Flows

## Goal

Walk through the complete signup registration flow and the password reset flow step-by-step, capturing screenshots of every form state, validation feedback, error states, and success confirmation screens.

## Context

- The dev server should already be running from plan 99-01. If not, start it with `VITE_DEV_BYPASS_AUTH=false npm run dev` from `/Users/massimodamico/bizscreen`
- **CRITICAL**: Ensure `VITE_DEV_BYPASS_AUTH=false` is set so auth forms render instead of redirecting to `/app`
- All screenshots go to `./screenshots/` with prefix `99-`
- Use MCP Playwright tools: browser_navigate, browser_take_screenshot, browser_snapshot, browser_click, browser_type, browser_fill_form
- The signup page at `/auth/signup` has fields: Full name (id="fullName"), Business name (id="businessName"), Email (id="email"), Password (id="password") with PasswordStrengthIndicator, plus Terms of Service links and "Sign in" link
- The reset password page at `/auth/reset-password` has a single email field and "Send reset link" button, plus "Back to login" link
- The update password page at `/auth/update-password` shows different states: loading (checking session), no-session error ("Invalid or expired link"), form (with password + confirm password), and success state
- The accept invite page at `/auth/accept-invite` shows: no-token error, loading, invite details, or success

## Auth Routes Involved

| # | URL Path | Page Name | Notes |
|---|----------|-----------|-------|
| 1 | `/auth/signup` | Signup Page | Full registration form with password strength indicator |
| 2 | `/auth/reset-password` | Reset Password Page | Email-only form to request reset link |
| 3 | `/auth/update-password` | Update Password Page | New password form (requires valid session from email link) |
| 4 | `/auth/accept-invite` | Accept Invite Page | Team invitation acceptance (requires token param) |
