# Plan 99-01: Login Flow Walkthrough

## Goal

Walk through the complete login flow step-by-step, capturing screenshots of the empty login form, filled form, invalid credential errors, empty field validation, loading states, and successful login landing on the dashboard.

## Context

- The dev server runs on `http://localhost:5173` (Vite default port)
- **CRITICAL**: The `.env.local` file has `VITE_DEV_BYPASS_AUTH=true`, which causes the login page to auto-redirect to `/app` before the form renders. You MUST start the dev server with the bypass disabled: `VITE_DEV_BYPASS_AUTH=false npm run dev`
- All screenshots go to `./screenshots/` with prefix `99-`
- Use MCP Playwright tools: browser_navigate, browser_take_screenshot, browser_snapshot, browser_click, browser_type, browser_fill_form
- The login page is at `/auth/login` and uses `AuthLayout` wrapper with email + password fields
- The login form has: email input (id="email"), password input (id="password"), show/hide password toggle, "Sign in" submit button, "Forgot password?" link, "Create an account" link
- Error messages appear in a red-bordered div with class `bg-red-50`
- The loading state shows a spinner with "Signing in..." text
