**2. Create `src/components/data-sources/DataPreviewTable.jsx`:**

A scrollable table that displays all fields and rows from a data source. This is the admin-side preview (not the player widget). Used after connecting a Google Sheets URL or uploading CSV so user can confirm data looks right.

Props:
- `fields` (array of field objects with `name`, `label`, `dataType`)
- `rows` (array of row objects with `values` map)
- `visibleColumns` (optional array of field names — if provided, only show these columns)
- `columnOrder` (optional array of field names — if provided, use this order)
- `maxHeight` (optional string, default '400px')

Implementation:
- Compute displayed fields: use `columnOrder` (or `fields.map(f => f.name)`) filtered by `visibleColumns` (if provided), mapped to field objects.
- Render a `div` with `overflow: auto`, `maxHeight`, border `border border-gray-700 rounded-lg`.
- Inside, render an HTML `<table>` with `className="w-full text-sm"`:
  - `<thead>` with sticky header: `className="sticky top-0 bg-gray-800 z-10"`. Each `<th>` shows `field.label || field.name`, left-aligned, `px-3 py-2 text-xs font-medium text-gray-400 border-b border-gray-700`.
  - `<tbody>`: Map rows. Each `<tr>` gets alternating colors: even rows `bg-gray-900/50`, odd rows `bg-transparent`. Hover: `hover:bg-gray-800/50`. Each `<td>`: `px-3 py-2 text-gray-300 border-b border-gray-800`. Show `row.values?.[field.name] ?? '—'`.
- Below table, show row count: `<p className="text-xs text-gray-500 mt-2">{rows.length} row(s)</p>`.
- If no fields or no rows, show a subtle empty state: "No data available" in gray text.
- Export as named export: `export function DataPreviewTable(...)`.

**3. Create `src/components/data-sources/ColumnPicker.jsx`:**

Column visibility toggles and reorder controls. User can show/hide columns and change their order.

Props:
- `fields` (array of field objects — all available fields from the data source)
- `visibleColumns` (array of field names currently visible, or null for "all visible")
- `columnOrder` (array of field names in display order, or null for source order)
- `onChange({ visibleColumns, columnOrder })` — callback when user changes visibility or order

Implementation:
- Derive working state: `effectiveOrder` = `columnOrder || fields.map(f => f.name)`. `effectiveVisible` = `visibleColumns || fields.map(f => f.name)` (convert null to "all selected").
- Render a vertical list of columns in `effectiveOrder`. Each item is a row with:
  - Checkbox (Tailwind styled): checked if field name is in `effectiveVisible`. Toggle adds/removes from visible list. Use `<input type="checkbox">` with `className="rounded border-gray-600 text-blue-500 focus:ring-blue-500 bg-gray-800"`.
  - Field label: `field.label || field.name`, `text-sm text-gray-300`.
  - Move up/down buttons: `ArrowUp` and `ArrowDown` icons from lucide-react (small, `w-3.5 h-3.5`, gray). Disabled at boundaries. On click, swap adjacent items in order array and call `onChange`.
- "Select All" / "Deselect All" toggle at top: text button, toggles all fields visible/hidden.
- When checkbox toggled: compute new `visibleColumns` array (if all are checked, pass null for "show all" optimization). Call `onChange({ visibleColumns: newVisible, columnOrder: effectiveOrder })`.
- When order changed: Call `onChange({ visibleColumns: effectiveVisible.length === fields.length ? null : effectiveVisible, columnOrder: newOrder })`.
- Export as named export: `export function ColumnPicker(...)`.
