**1. Extend IndexedDB cache in `src/player/cacheService.js`:**

CRITICAL: Keep ALL existing store creation logic unchanged (scenes, media, deviceState, offlineQueue). The existing stores use `!db.objectStoreNames.contains()` checks which handle upgrades correctly.
- Add `DATA_SOURCES: 'dataSources'` to the STORES constant object
- Add two new exported functions:
  - `cacheDataSource(dataSourceId, data)` — puts `{ id: dataSourceId, fields: data.fields, rows: data.rows, cachedAt: new Date().toISOString(), lastAccessedAt: new Date().toISOString() }` into the dataSources store
  - `getCachedDataSource(dataSourceId)` — gets entry by id from dataSources store, returns entry or null. If found, update `lastAccessedAt` (touch for LRU)

**2. Create `src/player/components/widgets/DataTableWidget.jsx`:**

Follow the ClockWidget pattern: export a named function `DataTableWidget` that accepts `{ props = {} }`.

Props (destructured from `props`):
- `dataSourceId` (required)
- `visibleColumns` (array of field names, optional — null means show all)
- `columnOrder` (array of field names, optional — null means use source order)
- `refreshIntervalMinutes` (number, default 15)
- `pageIntervalSeconds` (number, default 10)
- `rowsPerPage` (number, default 8)
- `showHeader` (boolean, default true)
- `alternateRowColors` (boolean, default true)
- Theme overrides: `headerBgColor`, `headerTextColor`, `evenRowBgColor`, `oddRowBgColor`, `textColor` (all string|null, null = inherit defaults)

Implementation:
- State: `data` (object with fields/rows), `currentPage` (number, starts 0)
- Use a `dataRef` to hold current data for stable timer reads
- **Initial load effect** (on `dataSourceId` change): Try `getDataSource(dataSourceId)` from `dataSourceService`. On success, setData + cache to IndexedDB via `cacheDataSource`. On error, silently fall back to `getCachedDataSource`. Never show error UI.
- **Refresh timer effect** (on `dataSourceId`, `refreshIntervalMinutes`): Set an interval at `refreshIntervalMinutes * 60 * 1000` that re-fetches data using the same try/cache-fallback pattern. Reset `currentPage` to 0 on successful fresh data. Clean up interval on unmount.
- **Column filtering**: Compute `visibleFields` by taking `columnOrder` (or `fields.map(f => f.name)` if null), filtering by `visibleColumns` (if provided), then mapping to field objects. Filter out any names that don't match actual fields.
- **Pagination effect** (on `totalPages`, `pageIntervalSeconds`): If `totalPages > 1`, set interval to advance page: `setCurrentPage(prev => (prev + 1) % totalPages)`. Clean up on unmount. If `totalPages <= 1`, no timer.
- **Render**: Full-bleed within zone (100% width/height, flex column). Per discretion recommendation: full-bleed style, no card wrapper.
  - Header row: flex row, bold text, background color `headerBgColor || 'rgba(59,130,246,0.15)'`, text color `headerTextColor || '#ffffff'`, responsive font `clamp(0.7rem, 2vw, 1rem)`. Each field gets `flex: 1, padding: '0 0.5rem'`, shows `field.label || field.name`.
  - Data rows: flex column with `flex: 1`. Each row is flex row with `flex: 1, alignItems: 'center'`. Alternate row colors: even rows get `evenRowBgColor || 'rgba(255,255,255,0.03)'`, odd rows get `oddRowBgColor || 'transparent'`. Text color `textColor || '#ffffff'`, font `clamp(0.6rem, 1.8vw, 0.9rem)`. Use `formatValue` from dataSourceService if available, otherwise just render `row.values?.[field.name]`.
  - Page indicator: Only show when `totalPages > 1`. Subtle: `textAlign: center, fontSize: 0.6rem, color: rgba(255,255,255,0.3)`. Display `{currentPage + 1} / {totalPages}`.
  - If no data or no visible fields, return `null` (silent — no error display per locked decision).

Import `getDataSource` and optionally `formatValue` from `'../../../services/dataSourceService'`. Import `cacheDataSource` and `getCachedDataSource` from `'../../cacheService'`. Use `createScopedLogger('DataTableWidget')` for internal logging only (no user-facing errors).
