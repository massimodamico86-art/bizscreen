**IndexedDB cache extension (`src/player/cacheService.js`):**

Bump `DB_VERSION` from 2 to 3. Add `RSS_FEEDS: 'rssFeeds'` to the `STORES` object. In the `upgrade` callback, add a version guard:
```js
if (oldVersion < 3) {
  if (!db.objectStoreNames.contains(STORES.RSS_FEEDS)) {
    db.createObjectStore(STORES.RSS_FEEDS, { keyPath: 'feedUrl' });
  }
}
```

Add two exported functions:
- `cacheRssFeed(feedUrl, feedData)`: Stores `{ feedUrl, ...feedData, cachedAt: Date.now() }` in the `rssFeeds` store
- `getCachedRssFeed(feedUrl)`: Retrieves cached feed data by feedUrl, returns null if not found

Follow the exact same pattern as `cacheDataSource`/`getCachedDataSource`.

**RssTickerWidget (`src/player/components/widgets/RssTickerWidget.jsx`):**

Follow DataTableWidget pattern for data loading/caching. Props accepted via `{ props = {} }`:
- `feedUrl` (string, required) -- RSS feed URL
- `refreshIntervalMinutes` (number, default 15) -- how often to refetch
- `speed` (number, default 50) -- pixels per second for scroll speed
- `separator` (string, default '|') -- character between items
- `backgroundColor` (string, default 'rgba(0,0,0,0.85)') -- ticker background
- `textColor` (string, default '#ffffff') -- text color
- `fontSize` (string, default 'clamp(0.8rem, 2vw, 1.2rem)') -- font size

Data loading pattern (same as DataTableWidget):
1. `useEffect` on `feedUrl` change: call `fetchRssFeed(feedUrl)` from rssFeedService
2. On success: set items state, cache via `cacheRssFeed(feedUrl, result)`
3. On error: silently fall back to `getCachedRssFeed(feedUrl)`, render nothing if no cache
4. Refresh timer: `setInterval` at `refreshIntervalMinutes * 60 * 1000`

Rendering:
- Outer div: `overflow: hidden`, full width/height, `backgroundColor`, flex center vertically
- Inner `<style>` tag with `@keyframes ticker-scroll { from { transform: translateX(0); } to { transform: translateX(-50%); } }`
- Content div: `display: flex`, `whiteSpace: nowrap`, `willChange: 'transform'`, animation using calculated duration
- **Seamless loop**: Render items TWICE (duplicate the array) so content wraps seamlessly. End position `translateX(-50%)` means exactly one copy scrolls off
- Each item: `<span>` with padding `0 2rem`, showing `item.title`
- Between items: separator span styled with `opacity: 0.4`, `padding: 0 1rem`
- Duration calculation: use `useRef` on content div, measure `scrollWidth`, compute `duration = scrollWidth / (speed * 2)` (because we duplicated content). Recalculate when items change
- If no items: render nothing (silent)
- Add `@media (prefers-reduced-motion: reduce)` to disable animation

**RssCardWidget (`src/player/components/widgets/RssCardWidget.jsx`):**

Follow DataTableWidget pattern for data loading/caching. Props accepted via `{ props = {} }`:
- `feedUrl` (string, required) -- RSS feed URL
- `refreshIntervalMinutes` (number, default 15) -- how often to refetch
- `maxCards` (number, default 4) -- max visible cards
- `cardRotateSeconds` (number, default 8) -- seconds per card rotation (for single-card carousel mode)
- `layout` (string, default 'grid') -- 'grid' or 'carousel'
- `showImages` (boolean, default true) -- show article images
- `showDate` (boolean, default true) -- show publication date
- `backgroundColor` (string, default 'transparent')
- `cardBgColor` (string, default 'rgba(255,255,255,0.05)')
- `textColor` (string, default '#ffffff')
- `accentColor` (string, default '#3b82f6')

Data loading: Same pattern as RssTickerWidget (fetch, cache, offline fallback).

Rendering:
- **Grid layout** (`layout === 'grid'`): CSS grid, `gridTemplateColumns: repeat(auto-fit, minmax(200px, 1fr))`, gap `0.75rem`, overflow hidden. Show up to `maxCards` items.
- **Carousel layout** (`layout === 'carousel'`): Single card at a time, auto-rotate every `cardRotateSeconds` via `setInterval` cycling `currentIndex`. Fade transition via opacity.
- Each card:
  - Container: `cardBgColor` background, `borderRadius: 0.5rem`, overflow hidden, flex column
  - Image area (if `showImages` and `item.imageUrl`): `<img>` with `object-fit: cover`, height ~40% of card. On error, hide the image (use `onError` to set a `failedImages` state set)
  - Content area: padding `0.75rem`
    - Title: `textColor`, bold, `fontSize: clamp(0.7rem, 1.5vw, 1rem)`, max 2 lines with `-webkit-line-clamp: 2`
    - Description excerpt: lighter color (50% opacity), `fontSize: clamp(0.6rem, 1.2vw, 0.85rem)`, max 3 lines with `-webkit-line-clamp: 3`
    - Date (if `showDate` and `item.pubDate`): `accentColor`, small font, bottom of card. Format with simple date display (just show the date string as-is from the feed -- no additional date library needed since it's display-only)
- If no items: render nothing (silent)
