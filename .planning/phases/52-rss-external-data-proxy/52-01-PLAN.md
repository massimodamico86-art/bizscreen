**Edge Function (`supabase/functions/rss-proxy/index.ts`):**

Follow the exact pattern from `supabase/functions/unsplash-proxy/index.ts`:
- Import `createClient` from `npm:@supabase/supabase-js@2`, `XMLParser` from `npm:fast-xml-parser@5`, `sanitizeHtml` from `npm:sanitize-html@2`, and `corsHeaders` from `../_shared/cors.ts`
- CORS preflight handler (OPTIONS)
- JWT auth verification (same pattern as unsplash-proxy: extract Bearer token, verify via `supabaseAdmin.auth.getUser()`, extract tenantId)
- Single action: `fetch` -- accepts `{ action: 'fetch', feedUrl: string }`
- Validate feedUrl: must be non-empty string, must start with `http://` or `https://`

`handleFetchFeed` flow:
1. **Cache check**: Query `rss_feed_cache` for `feed_url = feedUrl` where `expires_at > now()`. If cache hit, return cached items with `meta.cached: true`
2. **Fetch RSS feed**: Use `fetch(feedUrl)` with 10-second timeout via `AbortController`. Include conditional GET headers if cache has `etag` or `last_modified` values. If response is 304 Not Modified, extend cache TTL and return cached data
3. **Parse XML**: Use `XMLParser` with `{ ignoreAttributes: false, attributeNamePrefix: '@_' }`. Handle both RSS 2.0 (`parsed.rss.channel.item`) and Atom (`parsed.feed.entry`) formats. Ensure single-item feeds are normalized to arrays
4. **Extract feed items**: For each item, extract:
   - `title`: sanitize with `allowedTags: []` (text only)
   - `description`: sanitize with `allowedTags: ['b', 'i', 'em', 'strong', 'br', 'p']`
   - `link`: extract URL (Atom uses `@_href` on link element)
   - `pubDate`: from `pubDate` (RSS) or `published`/`updated` (Atom)
   - `imageUrl`: check in order: (a) `media:content` or `media:thumbnail` `@_url`, (b) `enclosure` with image MIME type `@_url`, (c) first `<img src="...">` regex match in description/content:encoded, (d) null. Resolve relative URLs against feed base URL
5. **Sanitize**: Use `sanitize-html` as described above. For `content:encoded`, prefer it over `description` for richer content
6. **Limit**: Max 50 items per feed
7. **Cache write**: Upsert to `rss_feed_cache` with 15-minute TTL (`expires_at`), store `etag` and `last_modified` from response headers
8. **Return**: `{ ok: true, data: { feedTitle, items, itemCount }, meta: { cached: false } }`

Error handling: Return structured `{ ok: false, error: { code, message } }` responses for invalid params (400), auth failures (401), feed fetch failures (502), parse failures (422), and internal errors (500).

**Migration (`supabase/migrations/144_create_rss_feed_cache.sql`):**

```sql
CREATE TABLE IF NOT EXISTS rss_feed_cache (
  feed_url TEXT PRIMARY KEY,
  feed_title TEXT,
  items JSONB NOT NULL DEFAULT '[]',
  item_count INTEGER NOT NULL DEFAULT 0,
  etag TEXT,
  last_modified TEXT,
  fetched_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  expires_at TIMESTAMPTZ NOT NULL DEFAULT (now() + interval '15 minutes')
);

-- Index for TTL cleanup
CREATE INDEX IF NOT EXISTS idx_rss_feed_cache_expires ON rss_feed_cache (expires_at);

-- RLS: service role only (Edge Function uses service role key)
ALTER TABLE rss_feed_cache ENABLE ROW LEVEL SECURITY;
-- No RLS policies needed -- only service role accesses this table
```

**rssFeedService.js (`src/services/rssFeedService.js`):**

Create a new service module following the pattern of existing services:
- Import `supabase` from `../supabase` and `createScopedLogger` from `./loggingService.js`
- Create scoped logger: `createScopedLogger('RssFeedService')`

Export `fetchRssFeed(feedUrl)`:
- Calls `supabase.functions.invoke('rss-proxy', { body: { action: 'fetch', feedUrl } })`
- Handles the response: if `data.ok` is true, return `data.data` (contains `feedTitle`, `items`, `itemCount`)
- If `data.ok` is false or error exists, log and throw with the error message
- This is the single entry point for all RSS feed fetching from the client

Export `validateRssUrl(url)`:
- Simple validation: must be non-empty string, must start with `http://` or `https://`
- Returns `{ valid: boolean, error?: string }`
