<objective>
Build a public REST API gateway as a Supabase Edge Function that validates existing API tokens, applies rate limiting, and proxies requests to tenant-scoped PostgreSQL RPCs for screens, playlists, and media CRUD.

Purpose: Enterprise customers need programmatic access to manage their digital signage fleet. The token management infrastructure already exists (api_tokens table, apiTokenService.js, validate_api_token RPC, DeveloperSettingsPage with token creation UI). This plan builds the missing gateway that connects tokens to data operations.

Output: Working REST API with 9 endpoints (CRUD for screens/playlists/media with presigned URL upload flow), rate limiting per token, tenant isolation, and API documentation tab in developer settings.
</objective>

**Migration (161_api_gateway_rpcs.sql):**

Create tenant-scoped SECURITY DEFINER RPCs. Every RPC takes `p_tenant_id UUID` as first parameter -- this comes from the validated token, NEVER from the API request body.

1. `api_list_screens(p_tenant_id UUID, p_limit INT DEFAULT 50, p_offset INT DEFAULT 0)` -- Returns JSONB array of screens (id, name, device_type, orientation, is_online, current_playlist_id, created_at) from tv_devices WHERE owner_id = p_tenant_id. Include total_count for pagination.

2. `api_get_screen(p_tenant_id UUID, p_screen_id UUID)` -- Returns single screen record or null.

3. `api_list_playlists(p_tenant_id UUID, p_limit INT DEFAULT 50, p_offset INT DEFAULT 0)` -- Returns JSONB array of playlists (id, name, description, item_count, created_at, updated_at) from playlists WHERE owner_id = p_tenant_id.

4. `api_get_playlist(p_tenant_id UUID, p_playlist_id UUID)` -- Returns single playlist with its items (joined from playlist_items).

5. `api_update_playlist(p_tenant_id UUID, p_playlist_id UUID, p_name TEXT DEFAULT NULL, p_description TEXT DEFAULT NULL)` -- Updates playlist fields where non-null. Returns updated record. Fails if playlist not owned by tenant.

6. `api_list_media(p_tenant_id UUID, p_limit INT DEFAULT 50, p_offset INT DEFAULT 0)` -- Returns JSONB array of media assets (id, name, type, file_url, thumbnail_url, file_size, created_at).

7. `api_update_screen_assignment(p_tenant_id UUID, p_screen_id UUID, p_playlist_id UUID)` -- Updates tv_devices.current_playlist_id. Validates both screen and playlist belong to tenant.

8. `api_create_media_record(p_tenant_id UUID, p_name TEXT, p_type TEXT, p_file_url TEXT, p_file_size BIGINT DEFAULT 0)` -- Inserts a media_assets record. Returns the created record with id. Called AFTER the client has completed the direct S3 upload. The `p_file_url` should be the public/CDN URL corresponding to the `file_key` returned by the presigned URL step.

NOTE: Presigned URL generation is handled directly in the Edge Function (TypeScript), NOT as a PostgreSQL RPC, because it requires AWS Signature V4 HMAC computation with S3 credentials -- logic that belongs in the application layer, not in PL/pgSQL.

All RPCs: SECURITY DEFINER, SET search_path = public, GRANT EXECUTE to service_role only (Edge Function uses service_role_key).

**Edge Function (api-gateway/index.ts):**

Follow the existing weather-proxy pattern. Single Deno.serve() handler:

1. **CORS preflight**: Return 200 with corsHeaders on OPTIONS.

2. **Token extraction**: Get Authorization header, verify it starts with `Bearer biz_`. Extract raw token.

3. **Token validation**: SHA-256 hash the raw token using Web Crypto API. Call `validate_api_token({ p_token_hash: hash })`. If not valid, return 401.

4. **Rate limiting**: Call `check_rate_limit({ p_identifier: tokenId, p_action: 'api_request', p_max_requests: 100, p_window_minutes: 15 })`. If not allowed, return 429 with Retry-After header.

5. **URL routing**: Parse req.url pathname after `/api-gateway/`. Match against route table:
```typescript
const ROUTES: Record<string, RouteConfig> = {
  'GET /v1/screens':           { rpc: 'api_list_screens',           scope: 'screens:read' },
  'GET /v1/screens/:id':       { rpc: 'api_get_screen',             scope: 'screens:read' },
  'GET /v1/playlists':         { rpc: 'api_list_playlists',         scope: 'playlists:read' },
  'GET /v1/playlists/:id':     { rpc: 'api_get_playlist',           scope: 'playlists:read' },
  'PUT /v1/playlists/:id':     { rpc: 'api_update_playlist',        scope: 'playlists:write' },
  'GET /v1/media':             { rpc: 'api_list_media',             scope: 'media:read' },
  'POST /v1/media':            { handler: 'generatePresignedUrl',    scope: 'media:write' },
  'POST /v1/media/confirm':    { rpc: 'api_create_media_record',    scope: 'media:write' },
  'PUT /v1/screens/:id/assignment': { rpc: 'api_update_screen_assignment', scope: 'screens:read' },
};
```

6. **Scope check**: Verify token scopes include the required scope. Return 403 if not.

7. **Parameter extraction**: For list endpoints, read `limit` and `offset` from query params. For `:id` routes, extract UUID from path. For PUT/POST, parse JSON body.

8. **RPC call**: Call the mapped RPC with `p_tenant_id` from validated token's `owner_id`. CRITICAL: Never pass tenant_id from request body. Include route-specific params.

9. **POST /v1/media presigned URL flow**: When the matched route is `POST /v1/media`, the Edge Function handles it directly (no RPC call). Parse `{ filename, content_type }` from the JSON body. Generate an S3 object key: `uploads/{tenant_id}/{crypto.randomUUID()}/{filename}`. Generate a presigned S3 PUT URL using AWS Signature V4 with the S3 credentials from Deno.env (`S3_BUCKET`, `S3_REGION`, `S3_ACCESS_KEY_ID`, `S3_SECRET_ACCESS_KEY`). This follows the same pattern used by the existing `/api/media/presign` endpoint (see `vite.config.js` dev middleware and `src/services/s3UploadService.js`). Return `{ upload_url, file_key, expires_in: 3600 }` so the external client can upload the binary file directly to S3 via PUT. After upload completes, the client calls `POST /v1/media/confirm` with `{ name, type, file_url, file_size }` which calls `api_create_media_record` RPC to create the `media_assets` row.

    Helper function needed for presigned URL generation:
    - `generatePresignedUrl(bucket, region, key, contentType, accessKeyId, secretAccessKey, expiresIn)` -- Implements AWS Signature V4 presigned URL using Web Crypto API for HMAC-SHA256. Returns the presigned PUT URL string.

10. **Response formatting**: Wrap RPC results in `{ data, meta: { total_count } }` for list endpoints, `{ data }` for single resource. Return 404 if single resource not found.

Helper functions needed:
- `sha256(message: string): Promise<string>` -- Web Crypto SHA-256
- `jsonResponse(body, status, extraHeaders?)` -- JSON response with corsHeaders
- `matchRoute(method, pathname): { rpc, scope, params } | null` -- URL pattern matcher
- `generatePresignedUrl(bucket, region, key, contentType, accessKeyId, secretAccessKey, expiresIn): Promise<string>` -- AWS Signature V4 presigned PUT URL using Web Crypto HMAC-SHA256

Import pattern: `import { createClient } from 'npm:@supabase/supabase-js@2'; import { corsHeaders } from '../_shared/cors.ts';`

Use `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` from Deno.env.
