  <name>Task 1: Create MenuBoardWidget with themed rendering, pagination, and realtime</name>
  <files>src/player/components/widgets/MenuBoardWidget.jsx</files>
  <action>
    Create `src/player/components/widgets/MenuBoardWidget.jsx` following the DataTableWidget pattern:

    **Props (received via `{ props }` wrapper):**
    - `menuBoardId` (string, required) — the menu board UUID
    - `theme` (string, default 'dark') — 'dark', 'light', or 'custom'
    - `textColor` (string, default '#ffffff')
    - `accentColor` (string, default '#f59e0b')
    - `showImages` (boolean, default true)
    - `showDescriptions` (boolean, default true)
    - `pageIntervalSeconds` (number, default 10)
    - `currencyCode` (string, default 'USD') — override; falls back to board's currency_code

    **Imports:**
    - useState, useEffect, useRef, useCallback from react
    - getMenuBoard, subscribeToMenuBoard, formatMenuPrice, DIETARY_TAGS from menuBoardService
    - SyncStatusIndicator from ./SyncStatusIndicator.jsx
    - useWidgetData from ../../hooks/useWidgetData.js

    **Data Fetching (following DataTableWidget):**
    - Use `useWidgetData` hook with key `menu-board:${menuBoardId}`, fetchFn calling `getMenuBoard(menuBoardId)`, interval 5 minutes (menu data doesn't need frequent polling since realtime handles updates, but periodic refresh ensures eventual consistency).
    - Fallback: if no orchestrator data, show a "No menu board configured" placeholder.

    **Realtime Subscription (MENU-07):**
    - In a separate useEffect, call `subscribeToMenuBoard(menuBoardId, handleRealtimeUpdate)`.
    - `handleRealtimeUpdate` triggers a re-fetch of the full menu board data (simplest approach -- refetch on any change). Set a local `refreshTrigger` state counter; increment it on realtime event. Include `refreshTrigger` in the useWidgetData key to force re-fetch: `menu-board:${menuBoardId}:${refreshTrigger}`.
    - Clean up subscription in useEffect cleanup.

    **Rendering:**
    - Compute `displayItems`: flatten all visible categories with their available items. Each category becomes a "category header" entry, each item becomes an "item" entry. Filter out `is_available === false` items and `is_visible === false` categories.
    - Compute `itemsPerPage` based on a reasonable estimate: ~8-10 items per page (configurable via a constant, or derive from container height if available). Default to 8.
    - Compute `totalPages = Math.ceil(displayItems.length / itemsPerPage)`.

    **Auto-Pagination (MENU-05, following DataTableWidget):**
    - State: `currentPage` (number), `displayedPage` (number), `opacity` (number for fade).
    - Timer effect: if `totalPages > 1`, setInterval to advance `currentPage` every `pageIntervalSeconds * 1000`. Wrap around with modulo.
    - Fade effect: when `currentPage` changes, set opacity to 0, after 300ms set `displayedPage = currentPage` and opacity to 1. Use the exact same dual-page state pattern from DataTableWidget.
    - Reset `currentPage` to 0 when data changes (total items count changes).
    - Clamp currentPage to valid range: `Math.min(currentPage, Math.max(0, totalPages - 1))`.

    **Theme Styles:**
    - Define theme presets:
      ```javascript
      const THEMES = {
        dark: { bg: '#1a1a2e', headerBg: '#16213e', text: '#ffffff', accent: '#f59e0b', itemBg: '#1a1a2e', altItemBg: '#16213e' },
        light: { bg: '#ffffff', headerBg: '#f8f9fa', text: '#1f2937', accent: '#f59e0b', itemBg: '#ffffff', altItemBg: '#f8f9fa' },
      };
      ```
    - For 'custom' theme, use the provided textColor and accentColor props.
    - Apply theme colors via inline styles (player widgets use inline styles, not Tailwind).

    **Layout (inline styles for player):**
    - Outer container: full width/height, background color from theme, padding, flex column layout, font-family from board's `font_family` or 'system-ui'.
    - For each item on the current page slice:
      - **Category headers**: full-width row with accent-colored left border (4px), category name in larger font, accent color. Optional category description below in smaller muted text.
      - **Menu items**: horizontal flex row with:
        - Item image (if `showImages` and item has `image_url`): small square thumbnail (48px), rounded corners, object-fit cover. Use `<img>` tag with inline styles.
        - Item name (bold, theme text color) + dietary badges inline (small colored rectangles with emoji text, 2px border-radius, matching DIETARY_TAGS colors).
        - Item description (if `showDescriptions`): below name, smaller font, muted color (70% opacity).
        - Prices: right-aligned. For each price column defined on the board, show the formatted price using `formatMenuPrice(item.prices[column.key], effectiveCurrency)`. Use the board's `currency_code` as default, overridden by the widget prop `currencyCode`. Multiple columns show as separate price entries with small column label above.
      - Alternating subtle background on item rows for readability.
    - **Page indicator** (if totalPages > 1): small dots at the bottom center, active dot highlighted with accent color.
    - **SyncStatusIndicator** in a subtle position (bottom-right corner) showing `lastFetchedAt`.
    - All font sizes should use `clamp()` for responsive scaling on different screen sizes (TV screens vary widely).

    **DietaryBadge inline component:**
    - Small inline component within MenuBoardWidget (not a separate file -- it's player-specific and tiny):
    ```javascript
    function DietaryBadge({ tag }) {
      const info = DIETARY_TAGS.find(t => t.key === tag);
      if (!info) return null;
      return (
        <span style={{
          display: 'inline-flex', alignItems: 'center', justifyContent: 'center',
          fontSize: 'clamp(0.45rem, 1vw, 0.6rem)', fontWeight: 700,
          color: '#fff', backgroundColor: info.color,
          borderRadius: '2px', padding: '0 3px', lineHeight: 1.4, marginLeft: '3px',
        }}>
          {info.emoji}
        </span>
      );
    }
    ```

    **Edge Cases:**
    - No menuBoardId: show placeholder text "No menu board configured"
    - Empty menu (no categories/items): show "Menu is empty"
    - All items unavailable: show "No items available" (all filtered out)
    - Missing price for a column: show empty string (no "$0.00")
    - Image load failure: hide the img element (onError handler sets display:none)
