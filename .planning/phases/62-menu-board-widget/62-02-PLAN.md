    Create `src/components/menu-boards/` directory with four components:

    **DietaryTagPicker.jsx** — Simple tag selection grid:
    - Import `DIETARY_TAGS` from `menuBoardService.js`
    - Props: `selectedTags` (string[]), `onChange` (fn)
    - Render a flex-wrap grid of tag buttons. Each button shows the tag emoji + label.
    - Selected tags have solid background with tag's color; unselected are outlined/gray.
    - Clicking toggles tag in/out of the array. Call `onChange(newTags)`.

    **MenuItemRow.jsx** — Single item row in the editor:
    - Props: `item`, `priceColumns`, `currencyCode`, `onUpdate`, `onDelete`, `onToggleAvailability`, `dragHandleProps` (from useSortable)
    - Layout: horizontal row with drag handle (GripVertical icon), name input, description input (smaller), price inputs (one per price column), image URL input, dietary tag badges (clickable to open DietaryTagPicker popover/inline), availability toggle switch, delete button (Trash2 icon).
    - Name and description are inline-editable text inputs that call `onUpdate` on blur.
    - Price inputs are number inputs (type="number", step="0.01") for each price column.
    - Availability toggle is a small switch/checkbox that calls `onToggleAvailability(item.id, !item.is_available)`.
    - Items that are unavailable (`is_available === false`) should have reduced opacity (opacity-50) on the row.
    - DietaryTagPicker renders inline below the item when the tag button is clicked (toggle show/hide).
    - Use `useSortable` from `@dnd-kit/sortable` for drag-and-drop. Apply `CSS.Transform.toString(transform)` to the row's style. Spread `attributes` and `listeners` on the drag handle.

    **CategorySection.jsx** — Sortable category with nested sortable items:
    - Props: `category`, `priceColumns`, `currencyCode`, `onUpdateCategory`, `onDeleteCategory`, `onAddItem`, `onUpdateItem`, `onDeleteItem`, `onToggleItemAvailability`, `onReorderItems`, `dragHandleProps`
    - Layout: category header row with drag handle, editable name input, visibility toggle (Eye/EyeOff icon), delete button (Trash2).
    - Below header: list of MenuItemRow components wrapped in a `DndContext` + `SortableContext` (verticalListSortingStrategy) for item reordering within this category.
    - "Add Item" button at bottom of item list (Plus icon + "Add Item" text).
    - `onDragEnd` handler: compute new order via `arrayMove` from `@dnd-kit/sortable`, call `onReorderItems(reorderedItems)`.
    - Use `PointerSensor` with `activationConstraint: { distance: 5 }` to prevent accidental drags on input clicks.
    - Use `useSortable` for category-level drag (for reordering categories).

    **MenuBoardEditorModal.jsx** — Full editor modal:
    - Props: `isOpen`, `onClose`, `menuBoard` (null for create, object for edit), `showToast`, `onSaved`
    - State: local state for the board's fields (name, description, theme, currencyCode, priceColumns, accentColor, textColor, showImages, showDescriptions, pageIntervalSeconds), plus `categories` array (each with nested items).
    - On mount (if editing): call `getMenuBoard(menuBoard.id)` to fetch full data with categories and items. Populate local state. Map snake_case DB fields to camelCase state.
    - **Board Settings Section** (top of modal): name input (required), description textarea, theme select (dark/light/custom), currency code input (text, default 'USD'), accent color input (color picker or text), page interval seconds input (number).
    - **Price Columns Section**: render list of price column entries (label + key). "Add Column" button. Each column has a label input and a remove button. Default column has key "default" and label "Price". When adding, generate key from label (lowercase, replace spaces with hyphens). Minimum 1 column (can't remove the last one).
    - **Categories Section**: wrapped in `DndContext` + `SortableContext` for category-level reordering. Render each category as a `CategorySection`. "Add Category" button at bottom. `onDragEnd` computes new order via `arrayMove`, calls `reorderCategories`.
    - **Save logic**: On save button click:
      - If creating: call `createMenuBoard(boardFields)`, then for each category call `createCategory(boardId, catFields)`, then for each item call `createMenuItem(catId, boardId, itemFields)`.
      - If editing: call `updateMenuBoard(id, boardFields)` for board changes. For categories/items, diff against original and call create/update/delete as needed. For simplicity, a valid approach is to track additions/deletions/updates in state and batch them on save.
      - Simpler approach for edit: on each user action (add/delete/rename category, add/delete/update item), call the service function immediately (optimistic update of local state). This avoids complex diffing. Show toast on error.
    - **Delete confirmation**: category and item deletes show a simple confirm before executing.
    - Use design-system components: `Modal` for the outer container, `Button` for actions.
    - Use `useLogger('MenuBoardEditorModal')` for error logging.
    - Error handling: wrap service calls in try/catch, log error, show toast with error message.
