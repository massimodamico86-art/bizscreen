   - Calls `loginAndPrepare(page)` from helpers
   - Uses `await use(page)` for the test
   - No explicit cleanup needed (browser context handles it)

3. Create `freshPage` fixture for unauthenticated tests that:
   - Creates a new browser context with empty storage state
   - Creates a new page in that context
   - Uses `await use(page)`
   - Closes the context in cleanup

4. Re-export `expect` from `@playwright/test`

The fixtures should follow this pattern from the research:
```javascript
import { test as base, expect } from '@playwright/test';
import { loginAndPrepare } from '../helpers.js';

export const test = base.extend({
  authenticatedPage: async ({ page }, use) => {
    await loginAndPrepare(page);
    await use(page);
  },

  freshPage: async ({ browser }, use) => {
    const context = await browser.newContext({
      storageState: { cookies: [], origins: [] }
    });
    const page = await context.newPage();
    await use(page);
    await context.close();
  },
});

export { expect };
```

This provides test-scoped isolation - each test using `freshPage` gets a completely clean context.
