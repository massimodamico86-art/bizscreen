  <action>
    Replace the existing Danger Zone section (currently a button that just shows a toast) with a proper multi-step deletion flow. This section is inside the `compliance` tab (activeTab === 'compliance') and is only visible to super_admin users.

    **Add state for deletion flow:**
    - `showDeleteConfirm` boolean (false) — controls visibility of the confirmation panel
    - `deleteConfirmText` string ('') — tracks what user types in confirmation input
    - `deletionReason` string ('') — optional reason for audit trail
    - `deleting` boolean (false) — loading state during deletion

    **Replace the Danger Zone button behavior:**
    - Instead of the single button that shows a toast, clicking "Delete All Tenant Data" sets `showDeleteConfirm` to true, expanding an inline confirmation panel below (NOT a modal — keep it inline for gravity).

    **Confirmation panel UI (shown when showDeleteConfirm is true):**
    1. Red-bordered panel with AlertTriangle icon
    2. Warning text: "This action will permanently delete all data for this tenant including screens, playlists, layouts, media, campaigns, schedules, and team member associations. This cannot be undone."
    3. Show the data summary counts (from existing `dataSummary` state) so admin sees exactly what will be deleted: "X screens, Y playlists, Z layouts, W media files, etc."
    4. Optional textarea for deletion reason (for audit trail)
    5. Confirmation input: Label reads "Type DELETE MY DATA to confirm". Input is a text field. The typed value is stored in `deleteConfirmText`.
    6. Two buttons side by side:
       - "Cancel" — resets `showDeleteConfirm`, `deleteConfirmText`, `deletionReason` to defaults
       - "Permanently Delete All Data" — red button, DISABLED unless `deleteConfirmText` exactly equals 'DELETE MY DATA' (case-sensitive). onClick calls the deletion handler.

    **Deletion handler (`handleDeleteTenantData`):**
    - Set `deleting` to true
    - Get current user's email from `userProfile.email`
    - Call `requestDataDeletion(userProfile.id, userProfile.email, deletionReason || null)` from complianceService (already imported via the existing imports — verify `requestDataDeletion` is imported, add to import if missing)
    - On success: show success toast "Tenant data deletion initiated", reset all deletion state, reload data summary (call `loadData()`)
    - On error: show error toast with error message, set `deleting` to false
    - Wrap in try/catch

    **Import:** Ensure `requestDataDeletion` is imported from `../services/complianceService`. Currently the page imports `downloadExport`, `getDataSummary`, `exportAsCSV`, `hasEnterpriseFeatures` — add `requestDataDeletion` to that import.

    Keep the existing `userProfile?.role === 'super_admin'` guard around the entire Danger Zone section.
