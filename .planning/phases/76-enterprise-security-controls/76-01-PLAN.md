  <files>src/services/passwordService.js, src/pages/EnterpriseSecurityPage.jsx</files>
  <action>
    **passwordService.js — Add tenant-level policy CRUD:**
    - Add `getPasswordPolicy(tenantId)` — queries supabase `tenant_settings` table for key `password_policy`, returns object `{ minLength: 8, requireUppercase: true, requireNumbers: true, requireSpecial: true }` or defaults if no row exists. Use `.from('tenant_settings').select('value').eq('tenant_id', tenantId).eq('key', 'password_policy').maybeSingle()`. If table doesn't exist (PGRST error), return defaults silently.
    - Add `savePasswordPolicy(tenantId, policy)` — upserts to `tenant_settings` with key `password_policy` and value as the policy object. Use `.from('tenant_settings').upsert({ tenant_id: tenantId, key: 'password_policy', value: policy, updated_at: new Date().toISOString() }, { onConflict: 'tenant_id,key' })`. If table doesn't exist, log error and return `{ success: false }`.
    - Add `getSessionPolicy(tenantId)` — queries `tenant_settings` for key `session_policy`, returns `{ sessionTimeout: 24, jwtExpiry: 1 }` (hours) or defaults.
    - Add `saveSessionPolicy(tenantId, policy)` — upserts `session_policy` to tenant_settings.
    - Export all four functions.

    **EnterpriseSecurityPage.jsx — Wire Security Policy tab:**
    - Import `getPasswordPolicy`, `savePasswordPolicy`, `getSessionPolicy`, `saveSessionPolicy` from passwordService.
    - Add state: `securityPolicy` object with `{ minLength: 8, requireUppercase: true, requireNumbers: true, requireSpecial: true, sessionTimeout: 24, jwtExpiry: 1 }` and `savingPolicy` boolean.
    - In `loadData()`, call `getPasswordPolicy` and `getSessionPolicy` and merge results into `securityPolicy` state.
    - **Password Policy section** (currently lines 763-787): Replace the two hardcoded `<select>` elements with:
      1. Minimum Length dropdown: options 8, 10, 12, 16 characters. Controlled by `securityPolicy.minLength`. onChange updates state.
      2. Three individual toggle switches for: "Require uppercase letters", "Require numbers", "Require special characters". Each controlled by respective `securityPolicy.requireX` booleans. Use the same toggle switch pattern already in the SSO section (lines 533-541).
    - **Session Security section** (currently lines 790-813): Replace the static `<span>` badges with:
      1. Session Timeout dropdown: options 1h, 4h, 8h, 12h, 24h, 72h, 168h (1 week). Controlled by `securityPolicy.sessionTimeout`.
      2. JWT Token Expiry dropdown: options 15min, 30min, 1h, 2h, 4h, 8h. Use minute values stored as hours (0.25, 0.5, 1, 2, 4, 8). Controlled by `securityPolicy.jwtExpiry`.
    - Add a "Save Security Settings" button at the bottom of the Security Policy tab. onClick calls `savePasswordPolicy` and `saveSessionPolicy` with current state, shows success/error toast. Use `savingPolicy` state for loading indicator.
    - Also update `PASSWORD_REQUIREMENTS` export in passwordService.js to have a note that these are defaults; runtime policy from tenant_settings overrides them. The `validatePassword` function should accept an optional `policy` parameter that overrides the hardcoded `PASSWORD_REQUIREMENTS`. If `policy.minLength` is provided, use that instead of `PASSWORD_REQUIREMENTS.minLength`. Same for requireUppercase, requireNumber, requireSpecial.
