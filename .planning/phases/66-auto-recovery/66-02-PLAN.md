A minimal, self-contained React component displayed when all recovery attempts are exhausted. Uses inline styles only (no Tailwind) since the player bundle is separate.

Props: `{ screenId, screenName, crashCount }`

Renders a dark full-screen overlay (`position: fixed, inset: 0, backgroundColor: '#0f172a'`) centered content with:
- Red circle icon (5rem) with a warning triangle SVG (stroke white)
- "Recovery Failed" heading (1.5rem, white)
- Message: "This screen has encountered repeated errors and automatic recovery has been disabled." (color: #94a3b8)
- Screen name line: "Screen: {screenName || screenId || 'Unknown'}" (color: #64748b, 0.875rem)
- Attempts line: "Attempts: {crashCount} | Please restart the device manually" (color: #64748b, 0.75rem)

Follow the exact design from 66-RESEARCH.md Pattern 4.

Add imports:

After the `usePlayerHeartbeat` call (which returns `{ heartbeatRef, screenshotInProgressRef, lastActivityRef, lastScreenshotTimeRef }`), destructure `screenshotInProgressRef` from the return value. Currently ViewPage does not destructure the return — change `usePlayerHeartbeat(screenId, loadContentRef, contentContainerRef)` to:
```javascript
const { screenshotInProgressRef } = usePlayerHeartbeat(screenId, loadContentRef, contentContainerRef);
```

Add `useAutoRecovery` hook call after the heartbeat hook:
```javascript
const { isExhausted, crashCount, triggerRecovery } = useAutoRecovery({
  screenId,
  contentContainerRef,
  loadContentRef,
  loading,
  content,
});
```

Update the `useStuckDetection` call to:
1. Add `contentContainerRef` parameter
2. Add `loading` parameter
3. Update `onVideoStuck` callback: instead of just trying to restart video and then calling `advanceToNextRef.current?.()` on failure, add a call to `triggerRecovery('video_stuck')` when the video recovery fails:
```javascript
onVideoStuck: () => {
  logger.warn('Video stuck detected, attempting recovery...');
  try {
    videoRef.current.currentTime = 0;
    videoRef.current.play().catch(err => {
      logger.error('Video play failed', { error: err });
      triggerRecovery('video_stuck');
    });
  } catch (err) {
    logger.error('Video recovery failed', { error: err });
    triggerRecovery('video_stuck');
  }
},
```
4. Update `onPageStuck` callback: replace `window.location.reload()` with `triggerRecovery('page_inactive')`:
```javascript
onPageStuck: () => {
  if (screenshotInProgressRef.current) {
    logger.debug('Screenshot in progress, deferring recovery by one cycle');
    return;
  }
  logger.warn('Player inactive for too long, triggering recovery...');
  triggerRecovery('page_inactive');
},
```
5. Add `onBlankScreen` callback:
```javascript
onBlankScreen: () => {
  if (screenshotInProgressRef.current) {
    logger.debug('Screenshot in progress, deferring blank screen recovery');
    return;
  }
  logger.warn('Blank screen detected, triggering recovery...');
  triggerRecovery('blank_screen');
},
```

Add exhaustion check: BEFORE the loading state check (the `if (loading)` block), add:
```javascript
// Recovery exhausted - show static fallback screen
if (isExhausted) {
  return (
    <RecoveryFallbackScreen
      screenId={screenId}
      screenName={content?.screen?.name}
      crashCount={crashCount}
    />
  );
}
```

This must come before the `if (loading)` check so that an exhausted device never shows loading spinner or attempts to render content.
