**1. Create `src/player/hooks/useAutoRecovery.js`:**

Create a new hook that orchestrates auto-recovery with a localStorage-persisted crash counter.

Constants:
- `RECOVERY_KEYS = { crashCount: 'player_recovery_count', lastRecoveryAt: 'player_last_recovery_at', recoveryPhase: 'player_recovery_phase' }`
- `MAX_RECOVERY_ATTEMPTS = 6`

Helper functions (module-level, not exported):
- `getCrashCount()` — synchronous localStorage read, returns parseInt or 0
- `incrementCrashCount()` — increments count, writes timestamp, returns new count
- `resetCrashCount()` — removes all 3 recovery keys from localStorage
- `getRecoveryAction(count)` — returns action string:
  - count >= 6 → `'exhausted'`
  - count >= 5 → `'cached_fallback'`
  - count >= 2 → `'hard_reload'`
  - else → `'soft_reload'`

Export `useAutoRecovery({ screenId, contentContainerRef, loadContentRef, loading, content })`:
- Use `useLogger('useAutoRecovery')`
- State: `isExhausted` (boolean, useState)
- Ref: `mountTimeRef` (Date.now())

On mount effect (synchronous crash count check):
- Read `getCrashCount()`. If >= MAX_RECOVERY_ATTEMPTS, log error and `setIsExhausted(true)`.

Content success effect (dependency: `loading`, `content`):
- When `!loading && content && contentContainerRef?.current?.children?.length > 0`:
  - If `getCrashCount() > 0`, log info "Content displayed successfully, resetting crash counter" with previousCount, then call `resetCrashCount()`.

`triggerRecovery` callback (useCallback, deps: [screenId, loadContentRef, logger]):
- Takes `reason` string parameter.
- Calls `incrementCrashCount()`, gets new count and action from `getRecoveryAction(count)`.
- Logs warn with reason, crashCount, action.
- Stores `action` in localStorage under `RECOVERY_KEYS.recoveryPhase`.
- Switch on action:
  - `'soft_reload'`: try `await loadContentRef.current?.(screenId, true)`, catch silently (next detection cycle will retry).
  - `'hard_reload'`: `window.location.reload()`
  - `'cached_fallback'`: try `await getCachedContent(\`content-${screenId}\`)` from playerService. If cached exists, call `await loadContentRef.current?.(screenId, false)`. If no cache or error, fall through to `window.location.reload()`.
  - `'exhausted'`: `setIsExhausted(true)`

Return: `{ isExhausted, crashCount: getCrashCount(), triggerRecovery }`

Import `getCachedContent` from `../../services/playerService`.
Import `useLogger` from `../../hooks/useLogger.js`.

Inside `checkStuck()`, AFTER the existing video and inactivity checks, add blank screen detection:
```
// Blank screen detection (Phase 66)
if (contentContainerRef?.current && !loading) {
  const elapsed = now - mountTimeRef.current;
  if (elapsed > STUCK_DETECTION.blankScreenGraceMs) {
    const container = contentContainerRef.current;
    const isBlank = container.children.length === 0 ||
                    container.getBoundingClientRect().width === 0 ||
                    container.getBoundingClientRect().height === 0;
    if (isBlank) {
      blankCheckCountRef.current++;
      if (blankCheckCountRef.current >= STUCK_DETECTION.blankConfirmChecks) {
        logger.warn('Blank screen detected', {
          confirmations: blankCheckCountRef.current,
        });
        if (onBlankScreen) {
          onBlankScreen({ confirmations: blankCheckCountRef.current });
        }
        blankCheckCountRef.current = 0;
      }
    } else {
      blankCheckCountRef.current = 0;
    }
  }
}
```

Update the useEffect dependency array to include `contentContainerRef`, `loading`, and `onBlankScreen`.

Update JSDoc to document the new parameters.
