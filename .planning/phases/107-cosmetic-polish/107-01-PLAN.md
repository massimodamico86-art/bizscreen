The Templates page has two filter rows:

At 375px, these filter tabs take ~50% of screen width alongside the template grid. Fix by making them collapsible on mobile:

1. Add a `showMobileFilters` state variable (default false).

2. Add a filter toggle button that is ONLY visible on small screens (`sm:hidden`):
   ```jsx
   <button
     onClick={() => setShowMobileFilters(prev => !prev)}
     className="sm:hidden flex items-center gap-2 px-4 py-2 mb-4 bg-gray-100 text-gray-700 rounded-lg w-full justify-center"
   >
     <Filter size={16} />
     {showMobileFilters ? 'Hide Filters' : 'Show Filters'}
     {(activeCategory !== 'all' || activeType !== 'all' || showFavorites) && (
       <span className="ml-1 bg-blue-100 text-blue-700 px-2 py-0.5 rounded-full text-xs">Active</span>
     )}
   </button>
   ```
   Import `Filter` from lucide-react.

3. Wrap both filter rows (Category and Type) in a container div:
   ```jsx
   <div className={`${showMobileFilters ? 'block' : 'hidden'} sm:block`}>
     {/* Category Filter Tabs */}
     ...
     {/* Type Filter with Favorites */}
     ...
   </div>
   ```

4. On the Category filter tabs div, change `flex flex-wrap gap-2 mb-4` to `flex flex-wrap gap-2 mb-4 overflow-x-auto pb-1` to handle horizontal scrolling if many categories.

This approach: filters are always visible on sm+ (640px+) screens but hidden behind a toggle on mobile. Active filter indicator shows the user that filters are applied even when collapsed.

The Pricing page uses `grid md:grid-cols-3 gap-8` (line 121) which forces 3 columns at 768px, making cards too tight with aggressive text wrapping.

Fix the responsive grid breakpoints:

1. Change the pricing cards grid from:
   `grid md:grid-cols-3 gap-8`
   to:
   `grid md:grid-cols-2 lg:grid-cols-3 gap-8`

   This gives:
   - Mobile (< 768px): 1 column (full width)
   - Tablet (768-1023px): 2 columns (comfortable spacing)
   - Desktop (1024px+): 3 columns (current layout)

2. When in 2-column layout, the 3rd card (Pro) will be alone on its row. To center it, add `md:last:col-span-2 md:last:max-w-md md:last:mx-auto lg:last:col-span-1 lg:last:max-w-none` to each card div to handle the orphan card gracefully. Actually, a simpler approach: wrap the grid in a container and let the 3rd card naturally flow.

   Actually, the simplest clean fix: Change to `grid sm:grid-cols-2 lg:grid-cols-3 gap-6 lg:gap-8`. This gives:
   - < 640px: 1 column
   - 640-1023px: 2 columns with gap-6
   - 1024px+: 3 columns with gap-8

   The "Most Popular" (Starter) card in the center works well in both 2-col and 3-col layouts.

3. Also reduce padding on cards for tablet: Change `p-8` on each card to `p-6 lg:p-8` for slightly tighter padding on smaller screens.
