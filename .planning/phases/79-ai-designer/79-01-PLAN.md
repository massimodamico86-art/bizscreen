<task type="auto">
  <name>Task 1: Create ai-designer Supabase Edge Function</name>
  <files>supabase/functions/ai-designer/index.ts</files>
  <action>
Create a Deno-based Supabase Edge Function at `supabase/functions/ai-designer/index.ts` that:

1. Accepts POST requests with JSON body: `{ prompt, messages, orientation, brandContext, imageBase64 }`
   - `prompt` (string, required): User's text description of the desired layout
   - `messages` (array, optional): Conversation history for refinement `[{ role: 'user'|'assistant', content: string }]`
   - `orientation` (string, optional): '16:9' | '9:16' | '4:3' | '1:1', defaults to '16:9'
   - `brandContext` (object, optional): `{ colors: { primary, secondary, accent }, fonts: { heading, body }, logoUrl }`
   - `imageBase64` (string, optional): Base64-encoded reference image

2. Reads `ANTHROPIC_API_KEY` from Deno.env (set as Supabase secret).

3. Constructs a Claude API call (model: claude-sonnet-4-20250514) with a system prompt that:
   - Instructs Claude to generate a JSON array of layout elements matching the types from `src/components/layout-editor/types.js`
   - Element types: text, image, shape, widget. Each has `id`, `type`, `position: { x, y, width, height }` (0-1 fractions), `layer`, `props`.
   - For text: props include `text`, `fontSize`, `fontFamily`, `fontWeight`, `align`, `color`, `backgroundColor`, `padding`, `borderRadius`
   - For image: props include `url` (use placeholder URL like `https://placehold.co/800x600/HEXCOLOR/white?text=Description`), `fit`, `borderRadius`, `opacity`
   - For shape: props include `shapeType` ('rectangle'|'circle'|'line'), `fill`, `stroke`, `strokeWidth`, `borderRadius`, `opacity`
   - For widget: include `widgetType` ('clock'|'date'|'weather'|'qr'), props per widget type
   - Background: `{ type: 'solid'|'gradient', color, from, to, direction }`
   - If brand context provided, use brand colors/fonts in the design
   - Orientation affects aspect ratio understanding (portrait = tall layouts, landscape = wide layouts)
   - Generate IDs using pattern `{type}-{timestamp}-{random9chars}` (match `generateElementId` pattern)

4. For follow-up refinement calls (messages array provided), pass the full conversation history so Claude modifies the existing layout rather than generating from scratch.

5. Parses Claude's response, extracts the JSON layout data, and returns `{ elements: [...], background: {...}, name: string }`.

6. Error handling: returns `{ error: string }` with appropriate HTTP status codes (400 for bad input, 500 for API errors, 503 for rate limits).

7. CORS headers: Allow origin `*` for development, include `Content-Type` in allowed headers.

Follow the same Deno patterns as `supabase/functions/rss-proxy/index.ts` and `supabase/functions/weather-proxy/index.ts`. Use `fetch` directly to call Anthropic API (no SDK needed in Deno) at `https://api.anthropic.com/v1/messages` with headers `x-api-key`, `anthropic-version: 2023-06-01`, `content-type: application/json`.
  </action>
  <verify>
Review the edge function code for: correct Deno syntax, proper Anthropic API call structure, JSON parsing of response, CORS headers, error handling. Verify the system prompt produces output matching the layout editor element types from types.js.
  </verify>
  <done>Edge function accepts a prompt and optional conversation history, calls Claude API, returns a valid layout element array with background and name.</done>

<task type="auto">
  <name>Task 2: Create aiDesignerService client-side service</name>
  <files>src/services/aiDesignerService.js</files>
  <action>
Create `src/services/aiDesignerService.js` with these exports:

1. `generateLayout({ prompt, orientation, brandContext, imageBase64 })` — Calls the ai-designer edge function via `supabase.functions.invoke('ai-designer', { body: { prompt, orientation, brandContext, imageBase64 } })`. Returns `{ elements, background, name }`. Validates that returned elements have valid types and positions.

2. `refineLayout({ prompt, messages, orientation, brandContext })` — Same as generateLayout but passes the full conversation history (messages array) so Claude refines the existing layout. The messages array should include the original prompt, Claude's original response (as JSON string of elements), and the new refinement prompt.

3. `buildConversation(history)` — Helper that takes an array of `{ role, content }` entries and formats them for the API call. Manages truncation if conversation gets too long (keep last 10 exchanges max).

4. `validateLayoutElements(elements)` — Validates that each element has required fields (id, type, position with x/y/width/height). Clamps position values to 0-1 range. Filters out elements with invalid types. Returns cleaned array.

5. `EXAMPLE_PROMPTS` — Array of 6 example prompts for the UI: "Restaurant menu board with daily specials", "Gym class schedule with motivational quotes", "Retail store welcome screen with promotions", "Hotel lobby information display", "Coffee shop menu with seasonal drinks", "Corporate lobby visitor welcome screen".

Use `createScopedLogger('AiDesignerService')` for logging. Import `supabase` from `../supabase`. Follow the existing service pattern (named exports, async functions, try/catch with structured logging).

Handle edge function errors gracefully: if the function returns an error, throw with a user-friendly message. If the response elements fail validation, log a warning and return the cleaned subset.
