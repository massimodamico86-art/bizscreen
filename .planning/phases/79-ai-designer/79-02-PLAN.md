  <action>
Create `src/components/layout-editor/AiDesignerPanel.jsx` — a collapsible side panel for the AI Designer feature.

**Props:**
- `onApplyLayout({ elements, background, name })` — callback to apply generated layout to canvas
- `orientation` — current canvas orientation ('16:9', '9:16', etc.)
- `onOrientationChange` — callback to change canvas orientation
- `showToast` — toast notification callback

**UI Structure:**

1. **Header:** Purple gradient icon (Sparkles from lucide-react) + "AI Designer" title + close button (X icon).

2. **Prompt Input Section:**
   - Textarea (3-4 rows) with placeholder "Describe the layout you want..."
   - Below the textarea: "Try:" section with 3-4 clickable example prompts from `EXAMPLE_PROMPTS` (show as small rounded chips/buttons). Clicking one fills the textarea.

3. **Orientation Selector:**
   - Row of 4 small icon buttons for orientation: Landscape (16:9), Portrait (9:16), Standard (4:3), Square (1:1)
   - Highlight current selection with purple border/bg
   - Use Monitor, Smartphone, ScreenShare, Square icons from lucide-react

4. **Generate Button:**
   - Full-width purple gradient button "Generate Layout"
   - Disabled when textarea is empty or generating
   - Shows Loader2 spinner + "Generating..." text during generation

5. **Loading State:**
   - While generating: show a pulsing animation with 3 steps: "Analyzing prompt...", "Designing layout...", "Generating elements..."
   - Steps cycle every 2 seconds using setInterval

6. **Result Display:**
   - After generation: show a brief success message "Layout applied!" with checkmark
   - Message auto-dismisses after 3 seconds

7. **Feature Gate:**
   - Wrap the entire panel content in a check for `Feature.AI_ASSISTANT` using `useFeatureFlag`
   - If not available, show a compact upsell message: "AI Designer is available on Pro plans" with upgrade link

**Behavior:**
- On submit: call `generateLayout({ prompt, orientation })` from aiDesignerService
- On success: call `onApplyLayout({ elements, background, name })` to push elements to canvas
- On error: show toast with error message
- Clear prompt after successful generation (user can type follow-up in plan 03)

Use `useLogger('AiDesignerPanel')` for structured logging. Import Button from design-system. Use Tailwind classes matching the dark theme of the layout editor (gray-800/900 backgrounds, gray-300/400 text, purple-400/500 accents).

**YodeckLayoutEditorPage.jsx:**
1. Create a `handleApplyAiLayout` callback that:
   - Receives `{ elements, background, name }` from the AI panel
   - Replaces the current layout elements with the AI-generated ones: `handleLayoutUpdate({ elements, background, name: name || layout.name })`
   - This uses the existing undo/redo history (pushHistory is called inside handleLayoutUpdate), so the user can undo the AI generation
   - Deselects any currently selected element: `setSelectedElementId(null)`
   - Shows toast: "AI layout applied — use Ctrl+Z to undo"
2. Pass new props to LeftSidebar:
   - `onApplyLayout={handleApplyAiLayout}`
   - `orientation={orientation}`
   - `onOrientationChange={handleOrientationChange}`
   - `showToast={showToast}`

This integration means the AI Designer panel appears as a sidebar tab. When the user generates a layout, it replaces the current canvas elements (with undo support), and the user can immediately edit any generated element using the existing layout editor tools.
