Extend `src/services/aiDesignerService.js` with:

1. **`buildBrandContext(branding)` export** — Takes the branding object from BrandingContext and extracts relevant fields for the AI:
   ```
   {
     colors: {
       primary: branding.primary_color || '#F26F26',
       secondary: branding.secondary_color || '#1E40AF',
       accent: branding.accent_color || branding.primary_color,
     },
     fonts: {
       heading: branding.heading_font || 'Inter',
       body: branding.body_font || 'Inter',
     },
     logoUrl: branding.logo_url || null,
   }
   ```
   Returns null if branding is null/undefined (brand toggle off).

2. **`convertImageToBase64(file)` export** — Takes a File object from file input, reads it using FileReader, returns a promise resolving to a base64 string (data URL format `data:image/...;base64,...`). Validates file is an image (type starts with 'image/') and under 4MB. Throws with user-friendly message if invalid.

3. **Update `generateLayout`** — Accept optional `imageBase64` parameter and pass to edge function body.

4. **Update `refineLayout`** — Accept optional `imageBase64` parameter. When building the messages array for refinement, include the previous layout JSON as an assistant message so Claude has context of what it previously generated. Format: messages array with `[{ role: 'user', content: originalPrompt }, { role: 'assistant', content: JSON.stringify(previousElements) }, { role: 'user', content: refinementPrompt }]`.

Follow existing service patterns: createScopedLogger, named exports, JSDoc comments.

Extend `src/components/layout-editor/AiDesignerPanel.jsx` with three new features:

**1. Conversation History & Refinement:**
- Add state: `conversationHistory` (array of `{ role: 'user'|'assistant', content: string, timestamp: Date }`), `currentElements` (the last generated elements array for passing to refineLayout).
- After successful generation, push `{ role: 'user', content: prompt }` and `{ role: 'assistant', content: 'Generated layout' }` to history. Store the elements in `currentElements`.
- Show conversation history above the prompt input as a scrollable list of message bubbles:
  - User messages: right-aligned, purple-500/20 bg, text-gray-200
  - Assistant messages: left-aligned, gray-700/50 bg, text-gray-300, show "Layout generated" or "Layout updated"
- When `conversationHistory.length > 0`, the submit button label changes to "Refine Layout" and calls `refineLayout` instead of `generateLayout`, passing the conversation messages and currentElements.
- Add a "New Design" button (small, outline style) above the conversation that clears history and currentElements to start fresh.

**2. Image Upload:**
- Add a small image upload area below the textarea: a dashed border box with ImagePlus icon + "Add reference image" text.
- Hidden file input (`accept="image/*"`). Clicking the box triggers the file input.
- On file select: call `convertImageToBase64(file)` from the service. Store base64 in state `referenceImage`.
- Show a thumbnail preview of the uploaded image (40x40 rounded) with an X button to remove.
- Pass `imageBase64: referenceImage` to generateLayout/refineLayout calls.
- After generation, clear the reference image (it was a one-time reference).

**3. Brand Toggle:**
- Import `useBranding` from `../../contexts/BrandingContext`.
- Add a toggle switch below the orientation selector: "Use my brand" with a small color swatch preview showing the brand primary color.

- When toggled on and branding data is available, call `buildBrandContext(branding)` and pass as `brandContext` to generateLayout/refineLayout.
- When no branding data is available (branding context returns null/empty), show the toggle as disabled with tooltip "Set up brand colors in Settings first".

- All new elements fit within the existing panel scroll area
