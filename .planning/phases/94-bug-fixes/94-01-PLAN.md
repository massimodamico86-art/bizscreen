<objective>
Fix the dashboard retry loop (BUG-01) and error toast flooding (BUG-03).

Purpose: DashboardPage currently polls every 30s with `setInterval` and calls `showToast` on every failure -- producing ~200 errors/sec when Supabase is unreachable. **Create `src/hooks/useRetryWithBackoff.js`:**

Create a custom hook with this API:

```js
/**
 * useRetryWithBackoff - Manages data fetching with exponential backoff retry.
 *
 * @param {Function} fetchFn - Async function to call (must throw on error)
 * @param {Object} options
 * @param {number} options.maxRetries - Max automatic retries before stopping (default 5)
 * @param {number} options.baseDelay - Base delay in ms (default 2000)
 * @param {number} options.maxDelay - Max delay cap in ms (default 30000)
 * @param {number} options.pollInterval - Normal polling interval in ms when healthy (default 30000)
 * @param {boolean} options.enabled - Whether to start fetching (default true)
 *
 * @returns {{ loading, error, retryCount, maxedOut, retry: () => void }}
 */
export function useRetryWithBackoff(fetchFn, options = {}) { ... }
```

Implementation details:
- Uses `useState` for `loading`, `error`, `retryCount`, `maxedOut` (boolean -- true when retryCount >= maxRetries).
- Uses `useRef` for timer ID and mounted flag.
- On mount (and when `enabled` transitions to true), calls `fetchFn()`:
  - Success: reset `retryCount` to 0, clear `error`, schedule next poll at `pollInterval`.
  - Failure: increment `retryCount`, set `error` to the caught error message. If `retryCount < maxRetries`, schedule next attempt at `min(baseDelay * 2^retryCount, maxDelay)`. If `retryCount >= maxRetries`, set `maxedOut = true` and do NOT schedule another attempt.
- Exposes a `retry()` function that resets `retryCount` to 0, clears `maxedOut`, clears `error`, and triggers a fresh fetch cycle. This is the manual retry button handler.
- Cleans up timers on unmount.
- Does NOT call `showToast` -- leaves that to the consumer.

**Update the `showToast` function in `src/App.jsx` (currently at line ~252):**

Replace the simple `showToast` with a deduplication-aware version:

1. Add a `useRef` for tracking recently shown toasts:
```jsx
const recentToastsRef = useRef(new Map()); // key: "type:message" -> timestamp
```

2. Replace `showToast`:
```jsx
const TOAST_THROTTLE_MS = 5000; // Suppress duplicates within 5 seconds

const showToast = useCallback((message, type = 'success') => {
  const key = `${type}:${message}`;
  const now = Date.now();
  const lastShown = recentToastsRef.current.get(key);

  // Deduplicate: skip if same message+type shown within throttle window
  if (lastShown && (now - lastShown) < TOAST_THROTTLE_MS) {
    return;
  }

  recentToastsRef.current.set(key, now);
  setToast({ message, type });
  setTimeout(() => setToast(null), 3000);

  // Housekeeping: prune old entries every 20 calls to prevent memory leak
  if (recentToastsRef.current.size > 50) {
    const cutoff = now - TOAST_THROTTLE_MS;
    for (const [k, ts] of recentToastsRef.current) {
      if (ts < cutoff) recentToastsRef.current.delete(k);
    }
  }
}, []);
```

3. Add `useRef` and `useCallback` to the React import at the top of App.jsx if not already imported (check first -- `useCallback` is likely already there, `useRef` may or may not be).

4. The `TOAST_THROTTLE_MS` constant should be defined outside the component (module-level) for clarity.
