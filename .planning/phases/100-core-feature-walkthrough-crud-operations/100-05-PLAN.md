# Plan 100-05: Interactive Controls Exerciser - Toggles, Dropdowns, Tabs & Modals

## Goal

Exercise every toggle, dropdown, tab, and modal on each major entity page -- screenshotting both/all states -- to fulfill CRUD-05 and ensure no interactive control is left untested.

## Context

- The dev server runs on `http://localhost:5173` with `VITE_DEV_BYPASS_AUTH=true`
- All screenshots go to `./screenshots/` with prefix `100-`
- Use MCP Playwright tools: browser_navigate, browser_take_screenshot, browser_snapshot, browser_click, browser_type
- Navigation uses `window.__setCurrentPage('pageId')` for authenticated pages
- This plan runs AFTER all entity CRUD plans (100-01 through 100-04) are complete, so entity data exists for testing interactive controls
- **Key interactive control types to exercise:**
  - **Tabs:** Media type tabs, campaign status tabs, template category tabs
  - **Dropdowns/Selects:** Sort order, location filter, view mode
  - **Toggles:** Grid/list view, show/hide panels, active/inactive states
  - **Modals:** All create/edit/delete modals (already captured in CRUD plans -- this plan captures additional modals like assign playlist to screen, insert content, analytics, kiosk mode)
  - **Action menus:** Three-dot menus on entity rows/cards with full option lists
  - **Drawers/Panels:** Screen detail drawer, playlist item detail, filter sidebars
  - **Tooltips/Popovers:** Hover states on buttons and badges
