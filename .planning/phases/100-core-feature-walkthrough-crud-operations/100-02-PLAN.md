# Plan 100-02: Schedules, Campaigns & Scenes CRUD Walkthrough

## Goal

Walk through the complete CRUD lifecycle for Schedules, Campaigns, and Scenes entities -- create, browse/filter, edit, and delete -- capturing screenshots at every step.

## Context

- The dev server runs on `http://localhost:5173` with `VITE_DEV_BYPASS_AUTH=true`
- All screenshots go to `./screenshots/` with prefix `100-`
- Use MCP Playwright tools: browser_navigate, browser_take_screenshot, browser_snapshot, browser_click, browser_type, browser_fill_form
- Navigation uses `window.__setCurrentPage('pageId')` for authenticated pages
- **Schedules page** (`schedules`): Has create modal (name, description), schedule list with search, schedule editor page (`schedule-editor-{id}`), action menu with edit/delete/duplicate/pause/activate
- **Campaigns page** (`campaigns`): Feature-gated (campaigns feature flag, may show FeatureUpgradePrompt). Has campaign list with status badges (Draft/Scheduled/Active/Completed), create from template flow via TemplatePickerModal, action menu with edit/delete/duplicate/pause/activate
- **Scenes page** (`scenes`): Card grid layout with business type badges, create button leading to scene creation flow, scene detail page (`scene-detail-{id}`), scene editor page (`scene-editor-{id}`), publish to screens, duplicate, delete with confirmation
- Feature-gated pages (campaigns) may show an upgrade prompt instead of content -- screenshot whatever state appears

<title>Screenshot Schedule CRUD lifecycle (create, browse, edit, delete)</title>
<steps>
**Step 1: Browse schedules list (initial state)**
1. Navigate to schedules page using `browser_evaluate` with `window.__setCurrentPage('schedules')`

10. Fill in: name = "QA Test Schedule", description = "Schedule created during QA audit"
11. Take screenshot of the filled modal: `screenshots/100-33-schedules-create-modal-filled.png`
12. Click create/submit
13. Wait for result -- may navigate to schedule editor or return to list
14. Take screenshot of result: `screenshots/100-34-schedules-create-result.png`

**Step 4: Schedule editor**
15. If now on the schedule editor page, take screenshot: `screenshots/100-35-schedules-editor.png`
16. Take `browser_snapshot` of the editor to document all controls (time slots, playlist assignment, day/week controls)
17. Navigate back to schedules list

**Step 5: Action menu on a schedule**
18. Find a schedule in the list, click the action menu (three dots)
19. Take screenshot showing all menu options: `screenshots/100-36-schedules-action-menu.png`

**Step 6: Delete a schedule**
20. Click delete from the action menu (or find a delete path)
21. Take screenshot of the delete confirmation: `screenshots/100-37-schedules-delete-confirm.png`
