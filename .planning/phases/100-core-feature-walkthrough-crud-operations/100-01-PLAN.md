# Plan 100-01: Screens & Playlists CRUD Walkthrough

## Goal

Walk through the complete CRUD lifecycle for Screens and Playlists entities -- create, browse/filter, edit, and delete -- capturing screenshots at every step to prove each operation is functional.

## Context

- The dev server runs on `http://localhost:5173` with `VITE_DEV_BYPASS_AUTH=true` (needs authenticated session for CRUD)
- All screenshots go to `./screenshots/` with prefix `100-`
- Use MCP Playwright tools: browser_navigate, browser_take_screenshot, browser_snapshot, browser_click, browser_type, browser_fill_form
- Navigation uses `window.__setCurrentPage('pageId')` for authenticated pages (exposed in dev mode)
- **Screens page** (`screens`): Has Add Screen modal (OTP-based pairing), screen list with search/filter, Edit Screen modal, delete with confirmation, screen detail drawer, status badges, action menu (reboot, reload, etc.)
- **Playlists page** (`playlists`): Has Create Playlist modal (name, description, template selection), playlist list with search, playlist editor page (`playlist-editor-{id}`), delete with usage check, duplicate
- **CRITICAL**: The app uses mock/demo data in dev mode. Screens and playlists may already have sample data pre-populated. Work with whatever state exists.
- Add `.gitignore` negation for `!screenshots/100-*` if not already present

## Tasks

<task id="100-01-01">
<title>Screenshot Screen CRUD lifecycle (create, browse, edit, delete)</title>
<steps>
**Setup:**
1. Ensure the dev server is running with `VITE_DEV_BYPASS_AUTH=true npm run dev` (run_in_background). If already running, navigate to verify.
2. Navigate to `http://localhost:5173/app` and wait for the dashboard to load
3. Add `.gitignore` negation for `!screenshots/100-*` if not already present

**Step 1: Browse screens list (initial state)**
4. Navigate to the screens page using `browser_evaluate` with `window.__setCurrentPage('screens')`
5. Wait for the page to load, take `browser_snapshot` to catalog all interactive elements
6. Take screenshot of the screens list in its initial state: `screenshots/100-01-screens-list-initial.png`

**Step 2: Search and filter screens**
7. If a search bar is visible, type a search term (e.g., "lobby" or the name of any visible screen) into the search field
8. Take screenshot showing search results: `screenshots/100-02-screens-search-results.png`
9. Clear the search field
10. If status filter buttons/tabs are visible (Online/Offline/All), click through each filter state and take screenshots:
    - `screenshots/100-03-screens-filter-online.png`
    - `screenshots/100-04-screens-filter-offline.png`
11. Return to "All" filter state

**Step 3: Create a new screen**
12. Click the "Add Screen" or "+" button to open the Add Screen modal
13. Take screenshot of the empty Add Screen modal: `screenshots/100-05-screens-add-modal-empty.png`
14. Take `browser_snapshot` to identify all form fields in the modal
15. Fill in the form fields (screen name: "QA Test Screen", location if available: "Test Location")
16. Take screenshot of the filled modal: `screenshots/100-06-screens-add-modal-filled.png`
17. Click the create/submit button
18. Wait for the result (may show OTP pairing code or success message)
19. Take screenshot of the result: `screenshots/100-07-screens-add-result.png`
20. Close the modal if still open and take screenshot of the updated screens list: `screenshots/100-08-screens-list-after-create.png`

**Step 4: Edit a screen**
21. Find a screen in the list (preferably the one just created, or any existing screen)
22. Click the edit button or action menu -> Edit for that screen
23. Take screenshot of the Edit Screen modal with pre-populated data: `screenshots/100-09-screens-edit-modal.png`
24. Change a field (e.g., rename to "QA Test Screen - Edited")
25. Take screenshot of the edited form: `screenshots/100-10-screens-edit-modal-changed.png`
26. Click save/update
27. Take screenshot showing the updated screen in the list: `screenshots/100-11-screens-list-after-edit.png`

**Step 5: Open screen detail drawer**
28. Click on a screen row to open the detail drawer/panel (if the UI supports it)
29. Take screenshot of the screen detail view: `screenshots/100-12-screens-detail-drawer.png`
30. Take `browser_snapshot` to document all elements in the detail view
31. Close the detail drawer

**Step 6: Delete a screen**
32. Find a screen to delete (preferably "QA Test Screen - Edited")
33. Click the delete button or action menu -> Delete
34. Take screenshot of the delete confirmation dialog: `screenshots/100-13-screens-delete-confirm.png`
35. Confirm the deletion
36. Take screenshot of the screens list after deletion: `screenshots/100-14-screens-list-after-delete.png`

**Step 7: Screen action menu**
37. If any screens remain in the list, click the action menu (three dots / more button) on a screen
38. Take screenshot showing the action menu with all available options (edit, delete, reboot, reload, etc.): `screenshots/100-15-screens-action-menu.png`
</steps>

- Edit flow shows pre-populated data and the result after saving
