# Plan 100-04: Data Sources, Menu Boards & Apps CRUD Walkthrough

## Goal

Walk through the complete CRUD lifecycle for Data Sources and Menu Boards, and browse the Apps marketplace -- create, browse/filter, edit, and delete -- capturing screenshots at every step.

## Context

- The dev server runs on `http://localhost:5173` with `VITE_DEV_BYPASS_AUTH=true`
- All screenshots go to `./screenshots/` with prefix `100-`
- Use MCP Playwright tools: browser_navigate, browser_take_screenshot, browser_snapshot, browser_click, browser_type, browser_fill_form
- Navigation uses `window.__setCurrentPage('pageId')` for authenticated pages
- **Data Sources page** (`data-sources`): Feature-gated (data-sources feature). List of data sources with field/row counts, create modal (internal table or CSV import), inline editing of fields and rows, CSV upload with auto-field detection, delete with confirmation, search
- **Menu Boards page** (`menu-boards`): Card grid of menu boards with theme indicators, create/edit via MenuBoardEditorModal (name, categories, menu items, theme), duplicate, delete with confirmation
- **Apps page** (`apps`): App marketplace grid with 146 interactive elements, category filtering, app detail modal with install/configure actions. This is a BROWSE-ONLY entity (apps are not user-created CRUD -- they are marketplace items). Focus on browsing, filtering, and detail views.
- This plan depends on 100-01 only to ensure `.gitignore` negation exists for `100-*` screenshots

<task id="100-04-01">
<title>Screenshot Data Sources CRUD lifecycle (create, browse, edit, delete)</title>
<steps>
**Step 1: Browse data sources list**
1. Navigate to data sources page using `window.__setCurrentPage('data-sources')`

10. Take `browser_snapshot` to identify form fields and creation options (internal table vs CSV import)

12. Fill in: name = "QA Test Data Source", select "Internal Table" if available
13. Take screenshot of the filled form: `screenshots/100-94-datasources-create-filled.png`
14. Click create/submit
15. Take screenshot of result: `screenshots/100-95-datasources-create-result.png`

**Step 4: Edit data source (inline editing)**
16. If the created data source now shows an inline editor for fields/rows, take screenshot: `screenshots/100-96-datasources-inline-editor.png`
17. Add a field or row if possible
18. Take screenshot after editing: `screenshots/100-97-datasources-after-edit.png`

**Step 5: Delete a data source**
19. Find a data source to delete
20. Click delete
21. Take screenshot of delete confirmation: `screenshots/100-98-datasources-delete-confirm.png`

23. Take screenshot after: `screenshots/100-99-datasources-after-delete.png`
