  <name>Task 1: Create shared cloud OAuth utility service</name>
  <files>src/services/cloud/cloudOAuthService.js</files>
  <action>
Create `src/services/cloud/cloudOAuthService.js` with shared utilities used by all 5 cloud providers.

Follow the canvaService.js pattern but extract reusable parts:

1. **PKCE utilities** (reuse from canvaService pattern):
   - `generateCodeVerifier()` — random 32-byte hex string via crypto.getRandomValues
   - `generateCodeChallenge(verifier)` — SHA-256 hash, base64url encode

2. **OAuth state management**:
   - `saveOAuthState(provider, { codeVerifier, state })` — stores to sessionStorage keyed by provider
   - `getOAuthState(provider)` — retrieves and removes (one-time use) from sessionStorage
   - `validateOAuthState(provider, receivedState)` — compares against saved state, returns { valid, codeVerifier }

3. **Token storage** (keyed by provider name):
   - `saveTokens(provider, { accessToken, refreshToken, expiresIn })` — stores to localStorage with keys like `cloud_{provider}_access_token`, `cloud_{provider}_refresh_token`, `cloud_{provider}_token_expiry`
   - `getTokens(provider)` — returns { accessToken, refreshToken, expiry } or null
   - `clearTokens(provider)` — removes all tokens for provider
   - `isTokenExpired(provider)` — checks if access token has expired (with 5-min buffer like canvaService)

4. **Provider registry constant**:
   ```js
   export const CLOUD_PROVIDERS = {
     GOOGLE_DRIVE: 'gdrive',
     DROPBOX: 'dropbox',
     ONEDRIVE: 'onedrive',
     SHAREPOINT: 'sharepoint',
     GOOGLE_PHOTOS: 'gphotos',
   };
   ```

Use `createScopedLogger('CloudOAuthService')` for logging. Export all functions as named exports.

<task type="auto">
  <name>Task 2: Create Google Drive and Dropbox service files</name>
  <files>src/services/cloud/googleDriveService.js, src/services/cloud/dropboxService.js</files>
  <action>
**Google Drive service** (`src/services/cloud/googleDriveService.js`):

Follow canvaService.js pattern. Use `createScopedLogger('GoogleDriveService')`.

Config constants:
- `VITE_GOOGLE_DRIVE_CLIENT_ID` (env var)
- OAuth URL: `https://accounts.google.com/o/oauth2/v2/auth`
- Token URL: `https://oauth2.googleapis.com/token`
- API URL: `https://www.googleapis.com/drive/v3`
- Scopes: `https://www.googleapis.com/auth/drive.readonly`

Functions (all named exports):
- `startGoogleDriveOAuth()` — generates PKCE + state via cloudOAuthService, saves to sessionStorage, redirects to Google OAuth with redirect_uri `${origin}/auth/cloud/callback?provider=gdrive`
- `handleGoogleDriveCallback(code, state)` — validates state via cloudOAuthService, exchanges code for tokens at token URL (POST with client_id, code, code_verifier, redirect_uri, grant_type=authorization_code), stores tokens via cloudOAuthService
- `isGoogleDriveConnected()` — checks token existence and expiry via cloudOAuthService
- `getGoogleDriveToken()` — returns valid access token, refreshes if needed (POST to token URL with grant_type=refresh_token), disconnects on failure
- `disconnectGoogleDrive()` — clears tokens via cloudOAuthService
- `listGoogleDriveFiles({ folderId = 'root', pageToken = null, query = '' })` — GET /files with fields=files(id,name,mimeType,thumbnailLink,size,modifiedTime),nextPageToken. Filter: `trashed=false and '${folderId}' in parents`. If query, add `and name contains '${query}'`. Support folders (mimeType=application/vnd.google-apps.folder). Returns `{ files: [...], nextPageToken }`.
- `getGoogleDriveDownloadUrl(fileId)` — returns `https://www.googleapis.com/drive/v3/files/${fileId}?alt=media` with auth header (for fetch-based download)
- Internal `gdriveFetch(url, options)` — wrapper adding Authorization Bearer header from getGoogleDriveToken()

**Dropbox service** (`src/services/cloud/dropboxService.js`):

Use `createScopedLogger('DropboxService')`.

Config constants:
- `VITE_DROPBOX_APP_KEY` (env var)
- OAuth URL: `https://www.dropbox.com/oauth2/authorize`
- Token URL: `https://api.dropboxapi.com/oauth2/token`
- API URL: `https://api.dropboxapi.com/2`
- Content URL: `https://content.dropboxapi.com/2`

Functions (all named exports):
- `startDropboxOAuth()` — PKCE + state, redirect to Dropbox OAuth. Dropbox uses `token_access_type=offline` for refresh tokens. Redirect URI: `${origin}/auth/cloud/callback?provider=dropbox`
- `handleDropboxCallback(code, state)` — validate state, exchange code for tokens
- `isDropboxConnected()` — check token via cloudOAuthService
- `getDropboxToken()` — get/refresh token
- `disconnectDropbox()` — clear tokens
- `listDropboxFiles({ path = '', cursor = null, query = '' })` — If query: POST to /files/search_v2 with `{ query, options: { path } }`. If no query: POST to /files/list_folder with `{ path: path || '', include_media_info: true }` or /files/list_folder/continue with cursor. Returns `{ files: [...], cursor, hasMore }`. Map entries to `{ id, name, isFolder, size, modified, thumbnailUrl, path }`.
- `getDropboxDownloadUrl(path)` — POST to content URL /files/get_temporary_link with `{ path }` returns `{ link }`. Return the link URL.
- Internal `dropboxFetch(url, body, options)` — wrapper adding Authorization Bearer header

Both services must import from `./cloudOAuthService.js` for PKCE, state, and token operations. No new npm dependencies required.
