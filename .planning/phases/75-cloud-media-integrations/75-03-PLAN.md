Create `src/components/media/CloudFilePicker.jsx` — a shared modal for browsing and importing files from any cloud provider.

**Props:**
- `open` (boolean) — controls visibility
- `onClose` (function) — close handler
- `provider` (string) — one of 'gdrive', 'dropbox', 'onedrive', 'sharepoint', 'gphotos'
- `onImport` (function) — called with array of imported media asset objects after successful import
- `showToast` (function) — toast notification handler

**Component structure:**

1. **Provider config map** at top of file:
```js
const PROVIDER_CONFIG = {
  gdrive: {
    name: 'Google Drive',
    listFiles: (args) => import('../services/cloud/googleDriveService.js').then(m => m.listGoogleDriveFiles(args)),
    getDownloadUrl: (id) => import('../services/cloud/googleDriveService.js').then(m => m.getGoogleDriveDownloadUrl(id)),
    isConnected: () => import('../services/cloud/googleDriveService.js').then(m => m.isGoogleDriveConnected()),
    disconnect: () => import('../services/cloud/googleDriveService.js').then(m => m.disconnectGoogleDrive()),
  },
  // ... same pattern for dropbox, onedrive, sharepoint, gphotos
};
```

Use lazy dynamic imports so provider services are only loaded when needed (not bundled together).

2. **State:**
- `files` — array of file objects from provider API
- `loading` — boolean for API calls
- `importing` — boolean for import in progress
- `selectedFiles` — Set of selected file IDs
- `breadcrumbs` — array of { id, name } for folder navigation (start with [{ id: 'root', name: provider.name }])
- `searchQuery` — string for search filter
- `nextPageToken` / `hasMore` — pagination
- `error` — error message string

For SharePoint, add `sites` state and `selectedSite` state — show site picker first, then file browser.
For Google Photos, add `albums` state and `selectedAlbum` state — show album grid first, then media items.

3. **UI layout** (use Tailwind classes matching YodeckAddMediaModal design):

**Header:** Provider name + icon, search input field, breadcrumb trail, close button (X icon from lucide-react)

**File grid:** Grid of file/folder cards (4 columns on desktop, 2 on mobile):
- Folders: Folder icon + name, click navigates into folder (push to breadcrumbs, fetch children)
- Files: Thumbnail (from provider's thumbnailLink/thumbnailUrl) or type icon fallback, name, size (human-readable), checkbox for selection
- Clicking a file toggles selection (checkbox)
- Support multi-select

**Footer:** "Import X files" button (disabled when none selected, shows count), Cancel button. Import button uses orange brand color `bg-[#f26f21]`.

**Load more:** If hasMore/nextPageToken, show "Load more" button at bottom of grid.

4. **Import flow** (handleImport function):
- Set `importing = true`
- For each selected file:
  a. Get download URL from provider service
  b. Fetch the file content as a blob using the download URL (need provider auth token in header — use provider service's internal fetch wrapper or pass token)
  c. Determine media type from file's mimeType
  d. Upload blob to S3 via the presign endpoint pattern OR create the asset with the download URL as the url field (simpler — use `createMediaAsset` with the cloud file's direct URL for image/video types; the URL may be temporary but is valid for the import action)
  e. Call `createMediaAsset({ name: file.name, type: mediaType, url: downloadUrl, thumbnailUrl: file.thumbnailUrl, mimeType: file.mimeType, fileSize: file.size })`
- On success: call `onImport(importedAssets)`, show toast "Imported X files successfully!", close modal
- On error: show toast with error message, keep modal open

**Important:** For the download URL approach, use the provider's authenticated fetch to get the file content as a blob, then create a temporary object URL or use S3 presigned upload. Since files need to persist in BizScreen beyond the temporary cloud URL lifetime, the preferred approach is:
  a. Get download URL from provider
  b. Create media asset record with name, type, mimeType, fileSize (store a reference that the file came from cloud import)
  c. In a real production flow, the server would download and re-host the file. For this client-side implementation, save the direct URL — it will work while the session is active. Add a TODO comment noting that production should re-host files.

3. Replace the `handleCloudProvider` function in `UploadTabContent` — instead of accepting it as a callback prop with "coming soon" toast, pass a new `onCloudProvider` prop. The new handler should:
```js
const handleCloudProvider = async (providerId) => {
  const providerMap = {
    gdrive: { isConnected: isGoogleDriveConnected, startOAuth: startGoogleDriveOAuth },
    dropbox: { isConnected: isDropboxConnected, startOAuth: startDropboxOAuth },
    onedrive: { isConnected: isOneDriveConnected, startOAuth: startOneDriveOAuth },
    sharepoint: { isConnected: isSharePointConnected, startOAuth: startSharePointOAuth },
    gphotos: { isConnected: isGooglePhotosConnected, startOAuth: startGooglePhotosOAuth },
  };
  const provider = providerMap[providerId];
  if (!provider) return;

  if (provider.isConnected()) {
    // Already connected — open file picker directly
    setCloudPickerProvider(providerId);
    setCloudPickerOpen(true);
  } else {
    // Need OAuth — save return state so we know to open picker after callback
    sessionStorage.setItem('cloud_import_return', providerId);
    try {
      await provider.startOAuth();
    } catch (err) {
      showToast?.(`Failed to connect: ${err.message}`, 'error');
    }
  }
};
```

3. Add a `useEffect` (or inline in existing init logic) that handles the cloud callback:
```js
useEffect(() => {
  if (!isCloudCallback) return;

  const params = new URLSearchParams(window.location.search);
  const provider = params.get('provider');
  const code = params.get('code');
  const state = params.get('state');
  const error = params.get('error');

  if (error) {
    showToast?.(`Cloud connection failed: ${params.get('error_description') || error}`, 'error');
    window.history.replaceState({}, '', '/app');
    return;
  }

  if (!code || !provider) {
    window.history.replaceState({}, '', '/app');
    return;
  }

  // Dynamic import of the correct provider's callback handler
  const callbackHandlers = {
    gdrive: () => import('./services/cloud/googleDriveService').then(m => m.handleGoogleDriveCallback(code, state)),
    dropbox: () => import('./services/cloud/dropboxService').then(m => m.handleDropboxCallback(code, state)),
    onedrive: () => import('./services/cloud/oneDriveService').then(m => m.handleOneDriveCallback(code, state)),
    sharepoint: () => import('./services/cloud/sharePointService').then(m => m.handleSharePointCallback(code, state)),
    gphotos: () => import('./services/cloud/googlePhotosService').then(m => m.handleGooglePhotosCallback(code, state)),
  };

  const handler = callbackHandlers[provider];
  if (handler) {
    handler()
      .then(() => {
        showToast?.(`Connected to cloud storage!`, 'success');
        // Navigate to media library — the YodeckAddMediaModal will detect the return provider
        window.history.replaceState({}, '', '/app');
        setCurrentPage('media');
      })
      .catch((err) => {
        showToast?.(`Connection failed: ${err.message}`, 'error');
        window.history.replaceState({}, '', '/app');
      });
  } else {
    window.history.replaceState({}, '', '/app');
  }
}, [isCloudCallback]);
```

This uses dynamic imports to keep the main App.jsx bundle small. The callback is handled inline (no separate page needed like Canva) since it just exchanges the code and redirects to media.
