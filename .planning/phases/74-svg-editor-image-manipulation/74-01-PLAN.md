In FabricSvgEditor.jsx, implement `handleAlignObject` that reads the selected object and canvas dimensions, then repositions based on alignment type. The PositionPanel already passes alignment types via `onAlign` prop. Create the handler:

```javascript
const handleAlignObject = useCallback((alignment) => {
  const canvas = fabricCanvasRef.current;
  const obj = canvas?.getActiveObject();
  if (!canvas || !obj) return;

  const objWidth = (obj.width || 0) * (obj.scaleX || 1);
  const objHeight = (obj.height || 0) * (obj.scaleY || 1);

  switch (alignment) {
    case 'left':
      obj.set('left', 0);
      break;
    case 'center-h':
      obj.set('left', canvasWidth / 2 - objWidth / 2);
      break;
    case 'right':
      obj.set('left', canvasWidth - objWidth);
      break;
    case 'top':
      obj.set('top', 0);
      break;
    case 'center-v':
      obj.set('top', canvasHeight / 2 - objHeight / 2);
      break;
    case 'bottom':
      obj.set('top', canvasHeight - objHeight);
      break;
  }

  obj.setCoords();
  canvas.renderAll();
  setHasUnsavedChanges(true);
}, [canvasWidth, canvasHeight]);
```

Wire `handleAlignObject` to the PositionPanel's `onAlign` prop in the JSX where PositionPanel is rendered (look for `activePanel === 'position'`). The prop is already passed as `onAlign` -- verify it's connected to `handleAlignObject` (it may currently be undefined or a no-op).
