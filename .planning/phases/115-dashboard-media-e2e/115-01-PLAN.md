  <action>
Create a NEW spec file `tests/e2e/dashboard-screenshots.spec.js` (do NOT modify the existing `dashboard.spec.js`). Use the existing helpers (`loginAndPrepare`, `navigateToSection`, `waitForPageReady`, `dismissAnyModals`).

The spec must use the `chromium` project only (skip for admin/superadmin). Skip if `TEST_USER_EMAIL` not set.

Use `beforeEach` to call `loginAndPrepare` with client credentials and navigate to `/app`.

Create the following test cases, each producing a screenshot via `page.screenshot({ path: 'screenshots/115-XX-description.png', fullPage: false })`:

**DASH-01: Stat cards**
- Navigate to dashboard (default after login)
- Wait for `#main-content` to show stat card labels: "Total Screens", "Playlists", "Media Assets", "Apps"
- Screenshot: `screenshots/115-01-dashboard-stat-cards.png`

**DASH-02: Sidebar navigation to all primary pages**
- From dashboard, iterate through sidebar nav items by clicking each in sequence: Media (expand > All Media), Playlists, Screens, Schedules, Scenes, Templates (if nav button exists), Apps (if nav button exists), Dashboard (return)
- For each page: wait for heading or main content to load, take screenshot: `screenshots/115-02-nav-{pagename}.png`
- Use `navigateToSection` for known sections. For others, click sidebar button directly by text match.
- After each navigation, verify `main` content area is visible (no crash).

**DASH-03: Breadcrumb navigation**
- Navigate to a sub-page that shows breadcrumbs (e.g., Media > All Media)
- Look for breadcrumb text (a nav or text element containing "/" or ">" separator with page names)
- Screenshot the breadcrumb area: `screenshots/115-03-breadcrumb-path.png`

**DASH-04: Welcome vs Dashboard differentiation**
- Navigate to `/app/welcome` and take screenshot: `screenshots/115-04-welcome-page.png`
- Navigate to `/app` (dashboard) and take screenshot: `screenshots/115-04-dashboard-page.png`
- Assert that both pages loaded without error (no "Something Went Wrong")

**DASH-05: Notification bell dropdown**
- On dashboard, locate the notification bell icon button (look for bell icon in header/topbar area, likely `button` with bell SVG or aria-label containing "notification")
- Click the bell to open dropdown
- Wait for dropdown/popover to appear
- Screenshot: `screenshots/115-05-notification-dropdown.png`
- Close dropdown by clicking elsewhere

Each test should be wrapped in `test.describe('Dashboard Screenshots', ...)`. Use `test.slow()` for the navigation test since it visits many pages.
