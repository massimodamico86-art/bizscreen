<task type="auto">
  <name>Task 2: Integrate UnifiedOnboardingController in DashboardPage</name>
  <files>src/pages/DashboardPage.jsx</files>
  <action>
Add conditional rendering of UnifiedOnboardingController based on feature flag.

1. Import the controller and config:
```javascript
import { UnifiedOnboardingController } from '../components/onboarding/UnifiedOnboardingController';
import { config } from '../config/env';
```

2. Add state for showing unified onboarding:
```javascript
const [showUnifiedOnboarding, setShowUnifiedOnboarding] = useState(false);
```

3. Add effect to check if unified onboarding should show (after initial load):
```javascript
// Check for unified onboarding (feature flag)
useEffect(() => {
  if (config.useUnifiedOnboarding && !loading) {
    // Import state check function
    import('../services/onboardingService').then(({ getUnifiedOnboardingState }) => {
      getUnifiedOnboardingState().then(state => {
        if (state.canResume && !state.isComplete) {
          setShowUnifiedOnboarding(true);
        }
      });
    });
  }
}, [loading]);
```

4. Add handler for onboarding completion:
```javascript
const handleUnifiedOnboardingComplete = useCallback(() => {
  setShowUnifiedOnboarding(false);
  // Refresh dashboard data
  fetchData?.();
}, [fetchData]);
```

5. In the render, add conditional rendering BEFORE existing onboarding modals:
```jsx
{/* Unified Onboarding Controller (feature flagged) */}
{config.useUnifiedOnboarding && showUnifiedOnboarding && (
  <UnifiedOnboardingController onComplete={handleUnifiedOnboardingComplete} />
)}
```

IMPORTANT: The unified controller should take precedence over existing onboarding when enabled. Add early return or condition to skip old onboarding logic when unified is active.

When feature flag is disabled, existing onboarding logic remains unchanged.
  </action>
