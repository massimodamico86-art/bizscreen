**New file: src/player/hooks/useWorkingHours.js**

Create a hook `useWorkingHours(workingHours, timezone)`:
- State: `isWithinHours` (boolean, default true).
- If `workingHours` is null/undefined, always return true (screen is always on).
- On mount and every 60 seconds, call `isWithinWorkingHours(workingHours, timezone)` from workingHoursService and update state.
- Use `useEffect` with `setInterval(check, 60000)` and cleanup.
- Import `isWithinWorkingHours` from `../../services/workingHoursService`.
- Return `isWithinHours`.
- Log state transitions (entering/leaving working hours) using `createScopedLogger('Player:WorkingHours')`.

**New file: src/player/hooks/useBackgroundAudio.js**

Create a hook `useBackgroundAudio({ audioUrl, volume, isPlaying })`:
- Creates and manages an `Audio` object (not a DOM element -- use `new Audio()`).
- `audioRef = useRef(null)` -- holds the Audio instance.
- On mount, create `audioRef.current = new Audio()` with `loop = true`, `preload = 'auto'`.
- When `audioUrl` changes: update `audioRef.current.src`. If switching from one URL to another, pause old, set new src, play if isPlaying.
- When `volume` changes: set `audioRef.current.volume = (volume || 100) / 100`.
- When `isPlaying` changes: call `.play()` or `.pause()`. Handle play() promise rejection (autoplay policy) silently with a debug log.
- On unmount: pause and clean up.
- If `audioUrl` is null/falsy, do nothing (no audio element interaction).
- Return `{ isAudioPlaying }` for status display (optional).

This component approach is used so it can be placed in the ViewPage render tree and will mount/unmount with the player lifecycle.
