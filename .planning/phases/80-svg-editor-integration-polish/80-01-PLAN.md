Three targeted fixes in FabricSvgEditor.jsx:

**Fix A — Consume openInNewTab in handleSaveHyperlink (ISSUE-2, part 2)**

At ~line 786, `handleSaveHyperlink` currently ignores the second argument and hardcodes `_blank`:

```jsx
const handleSaveHyperlink = useCallback((url) => {
  if (!selectedObject) return;
  selectedObject.set({ hyperlink: url, hyperlinkTarget: '_blank' });
```

Change to accept and use the `openInNewTab` parameter:

```jsx
const handleSaveHyperlink = useCallback((url, openInNewTab) => {
  if (!selectedObject) return;
  selectedObject.set({
    hyperlink: url,
    hyperlinkTarget: openInNewTab !== false ? '_blank' : '_self',
  });
```

This maps `openInNewTab=true` to `_blank` (current default, backward compatible) and `openInNewTab=false` to `_self`. Using `!== false` ensures backward compatibility if called with only one argument.

**Fix B — Sync canvasObjects after name edit in handleUpdateObject**

At ~line 2482, `handleUpdateObject` sets the property and calls `renderAll()` but never calls `syncCanvasObjects()`. When the user renames an element in ElementSettingsPanel, the LayersPanel `canvasObjects` array is stale — the old name shows.

After the `setUpdateCounter(c => c + 1)` line (~line 2492), add a call to `syncCanvasObjects()`:

```jsx
const handleUpdateObject = useCallback((property, value) => {
  const canvas = fabricCanvasRef.current;
  if (!canvas || !selectedObject) return;

  selectedObject.set(property, value);
  canvas.renderAll();
  setHasUnsavedChanges(true);

  // Force re-render by incrementing counter
  setUpdateCounter(c => c + 1);

  // Sync layers panel state (name, visibility, lock changes)
  syncCanvasObjects();
}, [selectedObject, syncCanvasObjects]);
```

Note: `syncCanvasObjects` must be added to the dependency array.

**Fix C — Close activePanel on selection:cleared**

At ~line 228, the `selection:cleared` handler only sets `setSelectedObject(null)`. The `activePanel` (which may be `'settings'`, `'effects'`, etc.) stays open even though no object is selected, showing stale data.

Add `setActivePanel(null)` inside the `selection:cleared` handler:

```jsx
canvas.on('selection:cleared', () => {
  setSelectedObject(null);
  setActivePanel(null);
});
```

This ensures ElementSettingsPanel (and any other object-specific panel) closes when the user clicks the canvas background or presses Escape to deselect.
