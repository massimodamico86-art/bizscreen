**Step 1 - Add cardLift preset to motion.js:**

Add a new `cardLift` export to `src/design-system/motion.js` after the existing `scaleHover` preset (around line 102). This preset is specifically for template cards and interactive surfaces that need a "lift off the page" feel:

```js
/**
 * Card lift animation
 * Use for template cards and premium interactive surfaces
 * Combines subtle scale with y-translate and shadow deepening
 */
export const cardLift = {
  whileHover: {
    y: -4,
    scale: 1.02,
    boxShadow: '0 20px 25px -5px rgb(0 0 0 / 0.1), 0 8px 10px -6px rgb(0 0 0 / 0.1)',
  },
  transition: { duration: duration.fast, ease: easing.smooth },
};
```

Also export `cardLift` from the design system barrel file `src/design-system/index.js` alongside the existing motion exports.

**Step 2 - Rewrite TemplateCard with Framer Motion and large thumbnails:**

Replace the current TemplateCard implementation in `src/design-system/components/TemplateCard.jsx`:

1. **Add Framer Motion import:** `import { motion } from 'framer-motion';`
2. **Import motion presets:** `import { cardLift, duration, easing } from '../motion';`
3. **Wrap the card root in `motion.div`** and spread `cardLift` onto it. Remove the existing CSS `hover:shadow-elevated transition-all duration-200` classes from the card (the Framer Motion `whileHover` replaces them). Keep the cursor-pointer conditional logic.
4. **Change thumbnail height from `aspect-video` to `h-60`** (240px). The `aspectClasses` lookup should be replaced with height classes:
   - `landscape`: `h-60` (240px)
   - `portrait`: `h-80` (320px)
   - `square`: `h-60` (240px)
   - default (no orientation): `h-60`
5. **Add progressive image loading** (follow the `OptimizedImage` pattern):
   - Add `const [imageLoaded, setImageLoaded] = useState(false);` state
   - Show an `animate-pulse` placeholder div (`absolute inset-0 bg-gray-200 animate-pulse`) when `!imageLoaded`
   - Add `loading="lazy"` and `onLoad={() => setImageLoaded(true)}` to the img tag
   - Apply opacity transition: `transition-opacity duration-300 ${imageLoaded ? 'opacity-100' : 'opacity-0'}`
6. **Update content padding** from `p-3` to `p-4` for visual balance with larger thumbnails.
7. **Do NOT change the hover overlay behavior** (the dark overlay with action buttons on hover stays as-is -- it works well on top of the lift).
8. **Do NOT change the forwardRef pattern** or component API (props stay the same).

**Step 3 - Update TemplateCardSkeleton to match new dimensions:**

Update `TemplateCardSkeleton` to match the enhanced card:
1. Change `aspect-video` to `h-60` for the thumbnail placeholder (must match loaded card exactly to prevent jump)
2. Update content padding from `p-3` to `p-4` to match the real card
3. Keep the `animate-pulse` shimmer pattern as-is (CSS pulse is the project standard, used in 34+ files)

**Anti-patterns to avoid:**
- Do NOT use `transition-all` on the card wrapper (Framer Motion handles transforms; CSS `transition-all` would conflict and cause jank)
- Do NOT animate border-color as part of the hover lift (only transform + shadow)
- Do NOT remove the `forwardRef` wrapping -- it's needed for parent measurement/refs
