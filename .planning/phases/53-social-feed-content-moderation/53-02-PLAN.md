Controls to include (all stored in design_json props, NOT in social_feed_settings table -- consistent with RSS widget pattern):

1. **Provider selector** -- `<select>` with options from `PROVIDER_LABELS` (imported from `../../services/social`). Default: 'instagram'. Prop key: `provider`.

3. **Layout selector** -- Button group (like RssWidgetControls layout buttons) with options from `LAYOUT_LABELS`: carousel, grid, list, single, masonry. Default: 'carousel'. Prop key: `layout`.

4. **Filter Mode selector** -- `<select>` with options from `FILTER_MODE_LABELS`: all, approved, hashtag, latest. Default: 'all'. Prop key: `filterMode`.

5. **Hashtags input** -- Only shown when `filterMode === 'hashtag'`. Text input for comma-separated hashtags. Prop key: `hashtags` (store as comma-separated string; the player/DB will parse). Placeholder: "#food, #restaurant".

6. **Max Posts slider** -- Range input, min 1, max 20, default 6. Prop key: `maxItems`.

7. **Rotation Speed slider** -- Only shown when layout is 'carousel' or 'single'. Range min 3, max 30, default 5. Label shows "{value}s". Prop key: `rotationSpeed`.

8. **Display toggles** -- Four checkboxes (same pattern as RssWidgetControls checkboxes):
   - Show Caption (prop: `showCaption`, default true)
   - Show Likes (prop: `showLikes`, default true)

   - Show Date (prop: `showDate`, default true)
