  <name>Task 1: Create ModerationPage with approve/reject queue</name>
  <files>src/pages/ModerationPage.jsx</files>
  <action>
Create `src/pages/ModerationPage.jsx` as a full page component receiving `{ showToast, onNavigate }` props (standard page pattern in this codebase).

Import from services:
- `getModerationQueue, moderatePost, getConnectedAccounts` from `../services/socialFeedSyncService`
- `PROVIDER_LABELS, PROVIDER_COLORS` from `../services/social`

Page structure:

1. **Header** -- Title "Content Moderation" with description "Review and approve social media posts before they appear on screens." Include a note: "Posts without moderation records are shown by default. Set a widget's filter mode to 'Approved Only' to require explicit approval."

2. **Account filter** -- `<select>` at top to filter by social account (populated via `getConnectedAccounts()` on mount). "All Accounts" as default option. Store selected `accountId` in state.

3. **Status filter tabs** -- Three tabs: "All", "Pending" (no moderation record, i.e. `moderation` is null or empty array), "Approved", "Rejected". Store `statusFilter` in state.

4. **Posts grid** -- Load posts via `getModerationQueue(accountId || null)` on mount and when accountId changes. Display posts in a responsive grid (Tailwind `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4`).

Each post card shows:
- Thumbnail image (post.media_url or post.thumbnail_url) -- 200px height, object-cover
- Provider badge (small colored pill using PROVIDER_COLORS)
- Author name (post.author_name)
- Caption text (post.content_text, line-clamp-3)
- Posted date (formatted with toLocaleDateString)
- Engagement stats (likes, comments) if available
- **Moderation status badge**: Green "Approved" / Red "Rejected" / Gray "Pending" based on `post.moderation` array (the query returns moderation as a joined array -- if empty or null, pending; else check `post.moderation[0].approved`)
- **Action buttons**:
  - "Approve" button (green, `CheckCircle` icon from lucide-react) -- calls `moderatePost(post.id, true)`, then refreshes the list
  - "Reject" button (red, `XCircle` icon from lucide-react) -- calls `moderatePost(post.id, false)`, then refreshes the list
  - If already approved/rejected, show the opposite action button (e.g., if approved, show "Reject" to change decision)

5. **Client-side status filtering** -- After loading posts from `getModerationQueue`, filter locally based on statusFilter tab:
- "All": show all posts
- "Pending": posts where `!post.moderation || post.moderation.length === 0`
- "Approved": posts where `post.moderation?.[0]?.approved === true`
- "Rejected": posts where `post.moderation?.[0]?.approved === false`

6. **Empty state** -- If no posts, show "No social feed posts to moderate. Connect a social account and sync posts to get started." with a button to navigate to 'social-accounts'.

7. **Loading state** -- Show a spinner while loading.

Use the standard page layout pattern from this codebase: white background, max-w container, proper spacing. Use `showToast` for success/error feedback on approve/reject actions. Wrap moderatePost calls in try/catch.

Export as default.
