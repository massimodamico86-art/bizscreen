<task type="auto">
  <name>Task 1: Create gate script and add JSON reporter</name>
  <files>scripts/e2e-gate.cjs, playwright.config.js, package.json</files>
  <action>
  1. **Create `scripts/e2e-gate.cjs`** -- a Node.js script (CommonJS with `.cjs` extension, following the `seed-ci-test-user.cjs` precedent since project is `"type": "module"`) that uses no dependencies beyond Node built-ins:
     - Defines constants: `THRESHOLD = 0.9`, `MAX_RUNS = 3`, `RESULTS_FILE = 'test-results/e2e-results.json'`
     - For each run (1 to MAX_RUNS):
       a. Sets `PLAYWRIGHT_JSON_OUTPUT_FILE` env var to `test-results/e2e-results.json`
       b. Runs `npx playwright test --reporter=json,html` via `child_process.execSync` with `stdio: 'inherit'`
       c. After each run (whether it succeeds or fails), reads and parses the JSON results file
       d. Walks the nested structure recursively: `report.suites[] > suites[] > specs[] > tests[]`
       e. For each `test` object, reads `test.status`:
          - `"expected"` -> passed
          - `"flaky"` -> passed (flaky counts as passed per user decision)
          - `"unexpected"` -> failed
          - `"skipped"` -> skipped (excluded from rate calculation)
       f. Computes: `passRate = passed / (passed + failed)` -- skipped excluded from both
       g. Prints run summary: passed, failed, skipped, flaky, pass rate, threshold
       h. If passRate >= THRESHOLD, prints success message and exits 0
       i. If passRate < THRESHOLD and more runs remain, prints retry message
     - After all runs exhausted, prints failure message with best result across all runs, exits 1
     - Tracks skipped count as separate metric (prints it but does not affect gate)
     - Handles edge case: if JSON file does not exist after a run (e.g., crash), treat as 0% pass rate for that run
     - Uses `process.argv` to accept optional `--threshold=X` override (default 0.9) and `--max-runs=N` (default 3) for local testing flexibility

  2. **Update `playwright.config.js`** -- Add JSON reporter to the reporter array:
     ```js
     reporter: [
       ['list'],
       ['html', { open: 'never', outputFolder: 'playwright-report' }],
       ['json', { outputFile: 'test-results/e2e-results.json' }],
     ],
     ```
     Note: The gate script also sets `PLAYWRIGHT_JSON_OUTPUT_FILE` env var as a belt-and-suspenders approach, but the config ensures JSON output even when running `npm run test:e2e` directly.

  3. **Update `package.json`** -- Add script entry:
     ```json
     "test:e2e:gate": "node scripts/e2e-gate.cjs"
     ```
     Place it after the existing `test:e2e` entry.

  Important: The file MUST be named `scripts/e2e-gate.cjs` (not `.js`) since the project is `"type": "module"` and the script uses CommonJS `require()`. This follows the existing `seed-ci-test-user.cjs` precedent.
  </action>
