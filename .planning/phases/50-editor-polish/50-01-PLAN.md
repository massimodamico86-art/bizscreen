Create a new dark-themed keyboard shortcuts overlay component. This is a standalone component -- FabricSvgEditor will wire it in Plan 02.

**Props:** `{ isOpen, onClose }`

**Shortcut data** (hardcoded constant SHORTCUTS array):
```
Category "General":
  Ctrl+S  -> Save design
  Ctrl+Z  -> Undo
  Ctrl+Y  -> Redo
  Ctrl+Shift+Z -> Redo (alt)
  Ctrl+P  -> Toggle preview
  Ctrl+D  -> Duplicate selection
  Delete  -> Delete selection
  Escape  -> Exit preview / deselect
  ?       -> Show keyboard shortcuts
```

**Visual design:**
- Use AnimatePresence wrapping the overlay so it animates in/out
- Backdrop: fixed inset-0, bg-black/50, z-[10000], onClick calls onClose
- Panel: centered dark card (bg-gray-800 border border-gray-700 rounded-xl shadow-2xl), max-w-md, using the `modal.content` animation preset from `../../design-system/motion`
- Header: "Keyboard Shortcuts" title (text-white font-semibold text-lg) + close button (X icon from lucide-react) in top-right
- Body: each shortcut as a flex row with description on left (text-gray-300 text-sm) and kbd tags on right
- Kbd tag styling: `px-2 py-1 text-xs bg-gray-700 text-gray-200 rounded border border-gray-600 font-mono min-w-[28px] text-center`
- Multiple keys joined with "+" rendered as separate kbd tags with a `+` span between them
- Add padding p-6, gap-3 between shortcut rows

**Keyboard handling:** The component itself does NOT handle the `?` key -- that is done in FabricSvgEditor's keydown handler (Plan 02). But DO handle Escape key inside the overlay to call onClose via a useEffect.

**Mac support:** Display "Cmd" instead of "Ctrl" when `navigator.platform` includes "Mac" (check once on mount with useState).

Import `motion, AnimatePresence` from framer-motion, `modal` from `../../design-system/motion`, `X` from lucide-react.

Export as default.
