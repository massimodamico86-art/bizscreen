**Exported function `searchPhotos(query, options)`:**
- JSDoc with @param and @returns:
  - `@param {string} query` - Search term (e.g., "coffee shop")
  - `@param {object} [options]` - Search options
  - `@param {number} [options.page=1]` - Page number (1-based)
  - `@param {number} [options.perPage=20]` - Results per page (1-30)
  - `@param {string} [options.orientation]` - Filter: 'landscape', 'portrait', or 'squarish'
  - `@returns {Promise<{photos: Array, pagination: {total: number, total_pages: number, page: number, per_page: number}, cached: boolean}>}`
- Validate query is a non-empty string; throw Error if not
- Call `supabase.functions.invoke(FUNCTION_NAME, { body: { action: 'search', query, page: options.page || 1, per_page: options.perPage || 20, orientation: options.orientation } })`
- Handle the response:
  - If `error` from invoke, log and throw with message `Unsplash search failed: ${error.message}`
  - If `data.ok === false`, check for rate limiting (`data.error.code === 'RATE_LIMITED'`):
    - Log warning with retry_after_seconds
    - Throw Error with user-friendly message: `Too many requests. Please try again in ${Math.ceil(data.error.retry_after_seconds / 60)} minutes.`
  - For other errors: throw with `data.error?.message || 'Unknown proxy error'`
  - On success: return `{ photos: data.data.photos, pagination: data.data.pagination, cached: data.meta?.cached || false }`

**Exported function `trackDownload(photoId, downloadTrackingUrl)`:**
- JSDoc explaining this MUST be called when a user selects/inserts a photo into their design (Unsplash TOS requirement)
- `@param {string} photoId` - Unsplash photo ID (for logging)
- `@param {string} downloadTrackingUrl` - The download_tracking_url from search results
- Fire-and-forget pattern: call `supabase.functions.invoke(FUNCTION_NAME, { body: { action: 'track-download', download_location: downloadTrackingUrl } })` inside a `.then().catch()` chain (do NOT await)
- On success: `logger.debug('Download tracked for photo:', photoId)`
- On error: `logger.warn('Download tracking failed for photo:', photoId, err.message)` -- log but never throw (must not disrupt user workflow)

**Module-level JSDoc comment at top of file:**
```
/**
 * Unsplash Proxy Service
 *
 * Client-side interface for the unsplash-proxy Edge Function.
 * All Unsplash API calls are proxied server-side to keep the API key secure.
 *
 * Usage:
 *   import { searchPhotos, trackDownload } from '../services/unsplashProxyService.js';
 *   const { photos, pagination } = await searchPhotos('coffee shop', { orientation: 'landscape' });
 *   // When user selects a photo:
 *   trackDownload(photo.id, photo.download_tracking_url);
 *
 * TOS Compliance:
 *   - Display photo.attribution.html near the photo
 *   - Use photo.urls.* directly (hotlink from Unsplash CDN, never re-host)
 *   - Call trackDownload() when user inserts a photo into their design
 */
```

**IMPORTANT:** This service does NOT import or reference UNSPLASH_ACCESS_KEY. It does NOT make direct calls to api.unsplash.com. All Unsplash interaction goes through the Edge Function proxy.
