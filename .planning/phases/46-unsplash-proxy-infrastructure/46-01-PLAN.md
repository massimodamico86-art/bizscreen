**Migration 142 -- Cache table (`supabase/migrations/142_create_unsplash_cache.sql`):**
- Create table `unsplash_search_cache` with columns:
  - `id` UUID primary key (gen_random_uuid())
  - `cache_key` TEXT NOT NULL UNIQUE -- deterministic key from normalized search params
  - `query` TEXT NOT NULL -- original search query for debugging
  - `page` INTEGER NOT NULL DEFAULT 1
  - `per_page` INTEGER NOT NULL DEFAULT 20
  - `orientation` TEXT -- nullable (landscape, portrait, squarish, or null)
  - `total_results` INTEGER NOT NULL DEFAULT 0
  - `total_pages` INTEGER NOT NULL DEFAULT 0
  - `results` JSONB NOT NULL DEFAULT '[]'::jsonb -- shaped photo results array
  - `unsplash_rate_limit` INTEGER -- upstream rate limit total from Unsplash headers
  - `unsplash_rate_remaining` INTEGER -- upstream remaining from Unsplash headers
  - `cached_at` TIMESTAMPTZ NOT NULL DEFAULT now()
  - `expires_at` TIMESTAMPTZ NOT NULL DEFAULT (now() + interval '24 hours')
- Create index `idx_unsplash_cache_key` on `cache_key` for fast lookups
- Create index `idx_unsplash_cache_expires` on `expires_at` for cleanup queries
- Add a comment on the table explaining its purpose

**Migration 143 -- Rate limit table and function (`supabase/migrations/143_create_unsplash_rate_limits.sql`):**
- Create table `unsplash_rate_limits` with columns:
  - `id` UUID primary key (gen_random_uuid())
  - `tenant_id` UUID NOT NULL
  - `window_start` TIMESTAMPTZ NOT NULL
  - `request_count` INTEGER NOT NULL DEFAULT 1
  - UNIQUE constraint on (tenant_id, window_start)
- Create index `idx_unsplash_rate_tenant` on (tenant_id, window_start)
- Create PL/pgSQL function `check_unsplash_rate_limit(p_tenant_id UUID, p_max_requests INTEGER DEFAULT 100)` that:
  - Computes `v_window_start` as `date_trunc('hour', now())`
  - Upserts into `unsplash_rate_limits`: INSERT with request_count=1, ON CONFLICT increment
  - Returns TABLE(allowed BOOLEAN, current_count INTEGER, max_allowed INTEGER, retry_after_seconds INTEGER)
  - `retry_after_seconds` = seconds until current hour window expires
  - The function is atomic (handles concurrent requests correctly via ON CONFLICT)

NOTE: These tables are internal infrastructure -- no RLS policies needed since they are only accessed by the Edge Function using the service role key. Add a comment to each table stating this.

This is BizScreen's first Edge Function. Create the directory structure and files.

**CORS module (`supabase/functions/_shared/cors.ts`):**
- Export `corsHeaders` object with:
  - `Access-Control-Allow-Origin`: `*`
  - `Access-Control-Allow-Headers`: `authorization, x-client-info, apikey, content-type`
  - `Access-Control-Max-Age`: `86400`
- The `_shared/` prefix is the Supabase convention to prevent this from being deployed as a standalone function.

**Edge Function (`supabase/functions/unsplash-proxy/index.ts`):**

Structure the handler using Deno.serve() with these sections:

1. **Imports and constants:**
   - Import `createClient` from `npm:@supabase/supabase-js@2`
   - Import `corsHeaders` from `../_shared/cors.ts`
   - Constants: `UNSPLASH_BASE = 'https://api.unsplash.com'`, `CACHE_TTL_HOURS = 24`, `RATE_LIMIT_PER_HOUR = 100`, `APP_NAME = 'bizscreen'`

2. **Main handler (Deno.serve):**
   - Handle OPTIONS preflight with corsHeaders
   - Extract Authorization header; return 401 if missing
   - Create supabaseAdmin client using `Deno.env.get('SUPABASE_URL')` and `Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')`
   - Verify user JWT via `supabaseAdmin.auth.getUser(token)` (strip 'Bearer ' prefix)
   - Extract tenant_id: use `user.user_metadata?.tenant_id || user.id` (in this schema, user.id IS the tenant_id for most clients; matches existing pattern in feedbackService.js and usageService.js)
   - Parse JSON body, extract `action` field
   - Route to `handleSearch` or `handleDownloadTracking` based on action
   - Wrap everything in try/catch, return standardized error envelope on failure

3. **handleSearch function:**
   - Accept params: `query` (required), `page` (default 1), `per_page` (default 20, max 30), `orientation` (optional)
   - Validate: query must be non-empty string, per_page clamped to 1-30, page >= 1
   - **Rate limit check:** Call `supabaseAdmin.rpc('check_unsplash_rate_limit', { p_tenant_id: tenantId, p_max_requests: RATE_LIMIT_PER_HOUR })`. If `allowed` is false, return 429 with `{ ok: false, error: { code: 'RATE_LIMITED', message: 'Too many requests...', retry_after_seconds } }`
   - **Cache check:** Build cache key by normalizing params: `JSON.stringify({ query: query.toLowerCase().trim().replace(/\s+/g, ' '), page, per_page, orientation: orientation || null })`. Query `unsplash_search_cache` where `cache_key = key AND expires_at > now()`. If hit, return cached results with `meta: { cached: true, cache_age_seconds: ... }`
   - **Unsplash API call:** If cache miss, call `GET {UNSPLASH_BASE}/search/photos` with URLSearchParams for query, page, per_page, content_filter='high', and optional orientation. Authorization header: `Client-ID {UNSPLASH_ACCESS_KEY}`. Handle non-OK responses (throw with status and remaining header info).
   - **Shape results:** Map each photo through a `shapePhoto` function that returns:
     - `id`, `description` (photo.description || photo.alt_description), `color`, `width`, `height`
     - `urls`: { thumb, small, regular, full, raw } from photo.urls
     - `attribution`: { photographer: { name, username, profile_url with UTM }, unsplash_url with UTM, photo_url with UTM, html (pre-formatted attribution string) }
     - `download_tracking_url`: photo.links.download_location
   - **Cache write:** Upsert into `unsplash_search_cache` with all fields. Use ON CONFLICT (cache_key) DO UPDATE to refresh stale entries.
   - **Return response:** `{ ok: true, data: { photos: [...], pagination: { total, total_pages, page, per_page } }, meta: { cached: false, upstream_rate_remaining: Number(response.headers.get('X-Ratelimit-Remaining')) } }`

4. **handleDownloadTracking function:**
   - Accept params: `download_location` (required)
   - Validate download_location is a string starting with 'https://api.unsplash.com/'
   - Call `GET {download_location}` with `Authorization: Client-ID {UNSPLASH_ACCESS_KEY}` header
   - This is fire-and-forget from the client's perspective, but the Edge Function should still await it
   - Return `{ ok: true, data: { tracked: true } }`
   - On error, log but still return success (download tracking failure should not block the user)

5. **Helper function `jsonResponse(body, status = 200)`:**
   - Returns `new Response(JSON.stringify(body), { status, headers: { ...corsHeaders, 'Content-Type': 'application/json' } })`

**IMPORTANT anti-patterns to avoid:**
- Do NOT use `unsplash-js` -- it is archived. Use raw fetch.
- Do NOT expose UNSPLASH_ACCESS_KEY in any way in responses.
- Do NOT use `Deno.env.get('SUPABASE_ANON_KEY')` for database operations -- use service role key for cache/rate-limit tables.
- Do NOT use GET query params with functions.invoke -- the function receives POST body.
- Do NOT cache without normalizing the query (lowercase, trim, collapse spaces).
