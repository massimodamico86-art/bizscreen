   - All four fields (fullName, businessName, email, password) are wired to `formData` state
   - `signUp({ email, password, fullName, businessName })` from authService called on submit
   - `isEmailConfirmationPending(user)` check triggers `emailSent` state showing confirmation view
   - `PasswordStrengthIndicator` `onValidationChange` sets `isPasswordValid`; submit button disabled when invalid
   - No console errors on load

3. **src/auth/ResetPasswordPage.jsx** — Verify:
   - `requestPasswordReset(email)` from authService is called correctly
   - Success state shows "Check your email" heading with the submitted email address
   - "Back to login" link navigates to `/auth/login`
   - No console errors on load

4. **src/auth/UpdatePasswordPage.jsx** — Verify:
   - `getSession()` call on mount correctly sets `hasSession` state
   - When `hasSession` is false, shows "Invalid or expired link" with request-new-link button pointing to `/auth/reset-password`
   - `validatePassword(password)` is checked before submit; passwords-match check is also present
   - `updatePassword(password)` from authService called and success triggers `navigate('/app')` after 2s
   - `PasswordStrengthIndicator` `onValidationChange` sets `isPasswordValid`; submit disabled when invalid
   - No console errors on load

5. **src/auth/AcceptInvitePage.jsx** — Verify:
   - `getInviteDetails(token)` is called on mount; missing token shows error state immediately
   - When user is not logged in, both CTA buttons (Create Account & Join, Sign in to existing account) carry `returnTo` param pointing back to `/auth/accept-invite?token=...`
   - When user IS logged in, `acceptInvite(token)` is called; success sets `success` state and redirects to `/app` after 2s
   - Error from `acceptInvite` is shown in the inline error div (`.bg-red-50`)
   - `getRoleDisplayName(inviteDetails?.role)` call does not crash when role is undefined

6. **src/pages/LoginPage.jsx** and **src/pages/ForgotPasswordPage.jsx** — These are legacy components used in the old SPA flow (before the router-based auth pages).    - `src/pages/ForgotPasswordPage.jsx` uses `<Alert>` and `<Button>` — same issue. Add the same imports.

Fix all issues found during the audit. The primary auth pages in `src/auth/` should have no functional issues; the legacy pages in `src/pages/` need their missing imports fixed.
