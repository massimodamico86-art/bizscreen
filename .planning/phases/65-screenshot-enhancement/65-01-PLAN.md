Add a 5-minute periodic screenshot capture to the heartbeat hook. This runs alongside the existing 30-second heartbeat, NOT as a separate setInterval (to avoid race conditions).

Implementation:
1. Add a constant: `const SCREENSHOT_INTERVAL = 5 * 60 * 1000; // 5 minutes`
2. Add a ref: `const lastScreenshotTimeRef = useRef(0);`
3. Inside `sendBeat()`, AFTER the existing `needs_screenshot_update` block (which handles on-demand captures), add a periodic capture check:
   - Calculate `timeSinceLastScreenshot = Date.now() - lastScreenshotTimeRef.current`
   - If `timeSinceLastScreenshot >= SCREENSHOT_INTERVAL` AND `!screenshotInProgressRef.current`:
     - Log: `logger.info('Periodic screenshot capture (5-min interval)')`
     - Set `screenshotInProgressRef.current = true`
     - Call `captureAndUploadScreenshot(screenId, container)` where `container = contentContainerRef?.current || document.body`
     - On success: set `lastScreenshotTimeRef.current = Date.now()` and call `cleanupOldScreenshots(screenId, 5)`
     - On failure: log the error but do not throw
     - In finally: set `screenshotInProgressRef.current = false`
4. IMPORTANT: Also update `lastScreenshotTimeRef.current = Date.now()` inside the EXISTING `needs_screenshot_update` block (after its successful capture) so on-demand captures reset the periodic timer. This prevents duplicate captures when an on-demand request fires right before the 5-minute mark.

Do NOT create a separate setInterval for screenshots -- keep all timing within the heartbeat sendBeat function. The heartbeat fires every 30 seconds, so the 5-minute check will trigger every ~10th heartbeat.

   - If `statusResult?.success !== undefined`: Check the result to detect recovery:
     - If `wasOfflineRef.current === true` and the heartbeat succeeded (statusResult is truthy / not an error):
       - Log: `logger.info('Device recovered from offline, capturing recovery screenshot')`
       - Set `wasOfflineRef.current = false`
       - Trigger a screenshot capture (same pattern: check `screenshotInProgressRef`, capture, update `lastScreenshotTimeRef`)
   - In the outer catch block of `sendBeat()` (heartbeat error): set `wasOfflineRef.current = true`

**B. Initial content load capture:**
1. The `lastScreenshotTimeRef` is initialized to `0`, which means on the very first heartbeat, `timeSinceLastScreenshot` will be >= SCREENSHOT_INTERVAL. This naturally triggers a capture on the first successful heartbeat -- effectively an initial capture.

However, the player may not have rendered content yet on the very first heartbeat (30s is typically enough, but to be safe): Add a small guard -- only do the periodic/initial capture if `contentContainerRef?.current` exists (content has mounted). If content hasn't mounted yet, skip the capture and the timer will try again next heartbeat.

**Integration notes:**
- Place the recovery capture BEFORE the periodic capture check in sendBeat() so that recovery captures also reset `lastScreenshotTimeRef` and prevent the periodic check from immediately firing again.
- The recovery capture should also call `cleanupOldScreenshots(screenId, 5)` after success.

Return `lastScreenshotTimeRef` from the hook alongside the existing returned refs.
