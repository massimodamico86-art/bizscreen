**2. Create `supabase/migrations/146_screen_group_tags_gin_index.sql`:**

```sql
-- GIN index on screen_groups.tags for efficient array containment queries
-- Supports @> operator used by Supabase .contains() for tag filtering
CREATE INDEX IF NOT EXISTS idx_screen_groups_tags ON screen_groups USING GIN(tags);
```

Follow the precedent from `094_svg_templates.sql` line 35.

**3. Extend `src/services/screenGroupService.js`:**

Add two new functions:

a) `fetchAllGroupTags()` - Query all distinct tags across screen groups for the current tenant. Use `supabase.from('screen_groups').select('tags')`, then flatten and deduplicate client-side (tags is TEXT[], so each row returns an array). Return a sorted unique array of strings. Coalesce null tags to empty array.

b) `fetchScreenGroupsWithScenesByTag(tag)` - Extend the existing `fetchScreenGroupsWithScenes` data by applying a client-side tag filter. Since `fetchScreenGroupsWithScenes` uses an RPC that doesn't support `.contains()`, fetch all groups via the RPC, then filter client-side: `groups.filter(g => g.tags?.includes(tag))`. This is acceptable because screen_groups is a small table (typically <100 rows per tenant).

Add both functions to the named exports AND to the default export object at the bottom of the file.
