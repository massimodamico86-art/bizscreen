  <action>
Create `CountdownWidget.jsx` in `src/player/components/widgets/`. Follow the ClockWidget pattern for the 1-second setInterval tick loop.

**Exported `calculateCountdown` pure function:**
- Parameters: `(mode, targetDate, targetTime, timezone, now)`
- `mode`: `'oneTime'` or `'daily'`
- `targetDate`: ISO 8601 string (e.g., `'2026-06-15T09:00:00'`) for oneTime mode
- `targetTime`: `'HH:mm'` string (e.g., `'17:00'`) for daily mode
- `timezone`: IANA timezone string or `'device'` sentinel
- `now`: `Date` object (current time)
- Resolve `'device'` sentinel to `Intl.DateTimeFormat().resolvedOptions().timeZone`
- Use `TZDate` from `@date-fns/tz` and `differenceInSeconds` from `date-fns` for timezone-safe calculation
- **oneTime mode:** Create target as `new TZDate(targetDate, tz)`. If `differenceInSeconds(target, now) <= 0`, return `{ expired: true, days: 0, hours: 0, minutes: 0, seconds: 0 }`
- **daily mode:** Parse `targetTime` into hours/minutes. Create today's target in the resolved timezone using TZDate constructor with explicit year/month/date/h/m/s/ms/tz. If target has passed (`differenceInSeconds <= 0`), immediately create tomorrow's target (increment date by 1). This handles the locked decision: "Immediate reset when countdown reaches zero -- starts counting to same time tomorrow instantly". Return decomposed remaining time.
- Decompose total seconds into `{ expired, days, hours, minutes, seconds }` using integer division (86400, 3600, 60)

**CountdownWidget component:**
- Receives `{ props = {} }` and destructures: `mode` (default `'oneTime'`), `targetDate`, `targetTime`, `timezone` (default `'device'`), `label` (default `''`), `locale` (default `'en'`), `textColor` (default `'#ffffff'`), `showTargetDate` (default `false`), `showUrgency` (default `true`), `unitLabelStyle` (default `'short'`)
- 1-second `setInterval` updating `now` state (same pattern as ClockWidget)
- Call `calculateCountdown(mode, targetDate, targetTime, timezone, now)` on every render (not useEffect -- it's a pure calculation from current state)
- **Expiry behavior (oneTime):** Track expiry timestamp with `useRef`. When `expired` first becomes true, record `Date.now()`. If expired for > 60 seconds, return `null` (auto-hide -- "stale content is bad" principle). Otherwise show frozen zeros.
- **Urgency visual cue:** When total remaining is under 3600 seconds (1 hour) AND `showUrgency` is true, add a subtle warm background to segments: `rgba(239, 68, 68, 0.15)`.

**Visual layout -- segmented boxes:**
- Outer container: `flex flex-col items-center justify-center w-full h-full gap-1`
- If `label` exists, render it above segments in `textColor` with `clamp()` font sizing: `fontSize: 'clamp(0.4rem, 1.2vw, 1rem)'`
- Segments row: `flex gap-1 sm:gap-2` wrapping 4 segments (days, hours, minutes, seconds)
- Each segment: rounded box with `bg-black/30` background (or urgency bg when applicable), padding, centered content
  - Number: large `clamp()` font (`clamp(0.5rem, 2vw, 1.5rem)`), bold, `textColor`, zero-padded to 2 digits (days can be more)
  - Label below number: smaller `clamp()` font (`clamp(0.2rem, 0.6vw, 0.5rem)`), `text-gray-400`
- **Unit labels by locale and style:**
  - Define a `UNIT_LABELS` constant object keyed by locale, each with `short` and `full` variants:
    - `en`: short `['D', 'H', 'M', 'S']`, full `['Days', 'Hours', 'Min', 'Sec']`
    - `es`: short `['D', 'H', 'M', 'S']`, full `['Días', 'Horas', 'Min', 'Seg']`
    - `pt`: short `['D', 'H', 'M', 'S']`, full `['Dias', 'Horas', 'Min', 'Seg']`
    - `it`: short `['G', 'O', 'M', 'S']`, full `['Giorni', 'Ore', 'Min', 'Sec']`
    - `fr`: short `['J', 'H', 'M', 'S']`, full `['Jours', 'Heures', 'Min', 'Sec']`
    - `de`: short `['T', 'S', 'M', 'S']`, full `['Tage', 'Std', 'Min', 'Sek']`
  - Fall back to `en` if locale not found
- **showTargetDate:** If true and not expired, render the target date/time below the segments using `Intl.DateTimeFormat` with the widget's `locale` and `timezone` (resolved). Format: `{ dateStyle: 'medium', timeStyle: 'short', timeZone: resolvedTz }`
- Export both `CountdownWidget` (named export) and `calculateCountdown` (named export for testability)
