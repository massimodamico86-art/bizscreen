Wire the hyperlink modal and click-to-open behavior into FabricSvgEditor:

1. **State**: Add `const [showHyperlinkModal, setShowHyperlinkModal] = useState(false);`

2. **Import**: Add `import HyperlinkModal from './HyperlinkModal.jsx';`

3. **Hyperlink handlers**:
   - `handleOpenLink`: `setShowHyperlinkModal(true)` — opens modal
   - `handleSaveHyperlink(url)`: Sets `selectedObject.hyperlink = url` and `selectedObject.hyperlinkTarget = '_blank'` as custom properties on the fabric object via `selectedObject.set({ hyperlink: url, hyperlinkTarget: '_blank' })`, then `fabricCanvasRef.current.renderAll()`, `setShowHyperlinkModal(false)`, `setHasUnsavedChanges(true)`. Force re-render with `setUpdateCounter(c => c + 1)`.
   - `handleRemoveHyperlink()`: Sets `selectedObject.hyperlink = ''` and `selectedObject.hyperlinkTarget = ''`, render, close modal, mark unsaved.

4. **Custom property serialization**: Fabric.js custom properties need to be included in JSON serialization. Find where `canvas.toJSON()` is called (for save and history) and ensure custom properties `hyperlink` and `hyperlinkTarget` are included. Fabric 6.x supports `canvas.toJSON(['hyperlink', 'hyperlinkTarget'])` — pass these property names. Search for all `toJSON()` calls and add these properties. Also search for `toObject()` calls.

5. **Preview mode click handler**: In preview mode, when user clicks on an object that has a `hyperlink` property, open the URL. Add a click handler on the canvas for preview mode:
   - In the canvas initialization or where preview mode events are set up, add: when `isPreviewMode` is true and user clicks an object, check `activeObject.hyperlink`. If truthy, call `window.open(activeObject.hyperlink, activeObject.hyperlinkTarget || '_blank')`.
   - Use `canvas.on('mouse:down', ...)` and check `isPreviewMode` ref.

6. **Wire props**:
   - TopToolbar: Add `onOpenLink={handleOpenLink}` prop
   - ContextMenu: Replace `onOpenLink={() => {}}` with `onOpenLink={handleOpenLink}`

7. **Render HyperlinkModal**: After the ContextMenu render block (around line 2993), add:
   ```jsx
   {showHyperlinkModal && selectedObject && (
     <HyperlinkModal
       isOpen={showHyperlinkModal}
       onClose={() => setShowHyperlinkModal(false)}
       currentUrl={selectedObject?.hyperlink || ''}
       onSave={handleSaveHyperlink}
       onRemove={handleRemoveHyperlink}
     />
   )}
   ```

8. **Visual indicator**: When an object has a hyperlink, show a small link icon overlay or underline indicator. For text objects with hyperlinks, you can add a subtle underline or color change. Keep it simple — just ensure the TopToolbar Link button shows active state (already handled in Task 1).
