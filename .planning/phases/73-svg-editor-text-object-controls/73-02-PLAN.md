3. **TopToolbar new props**: Pass these additional props to TopToolbar:
   - `onOpenSettings={() => setActivePanel(activePanel === 'settings' ? null : 'settings')}` — toggle settings panel
   - `onBringForward={handleBringForward}` — already exists as handler
   - `onSendBackward={handleSendBackward}` — already exists as handler
   - `onOpenLink={handleOpenLink}` — from plan 01
   - `onToggleAspectRatioLock={handleToggleAspectRatioLock}` — new handler
   - `isAspectRatioLocked={selectedObject?.lockUniScaling || false}`

4. **Aspect ratio lock handler**: Create `handleToggleAspectRatioLock`:
   ```javascript
   const handleToggleAspectRatioLock = useCallback(() => {
     const canvas = fabricCanvasRef.current;
     const activeObj = canvas?.getActiveObject();
     if (!activeObj) return;
     const newLock = !activeObj.lockUniScaling;
     activeObj.set('lockUniScaling', newLock);
     canvas.renderAll();
     setHasUnsavedChanges(true);
     setUpdateCounter(c => c + 1);
   }, []);
   ```
   `lockUniScaling` is a built-in fabric.js property that constrains scaling to maintain aspect ratio. When `true`, dragging a corner handle scales uniformly. When `false`, user can stretch freely.

5. **Serialize lockUniScaling**: Ensure `lockUniScaling` is included in the custom properties array where `toJSON(['hyperlink', 'hyperlinkTarget'])` was added in plan 01. Add `'lockUniScaling'` to that array so aspect ratio lock state persists through save/load.

6. **Default aspect ratio lock for images**: When adding new images (find `handleAddImage`, `handleAddImageFromUrl`, and the drop handler), set `lockUniScaling: true` as default for image objects. This is the expected behavior — images default to locked aspect ratio, user can unlock.
