<task type="auto">
  <name>Task 2: Create Sentry alert rules via API</name>
  <files></files>
  <action>
    Use the Sentry API to create two alert rules for the `javascript-react` project in the `bizscreen` organization. The SENTRY_AUTH_TOKEN is available as a GitHub secret but also needs to be available locally for API calls. Check if the user has it in their environment or .env.local.

    **Pre-check:** Verify API access:
    ```bash
    curl -s -H "Authorization: Bearer $SENTRY_AUTH_TOKEN" \
      "https://sentry.io/api/0/projects/bizscreen/javascript-react/" | head -20
    ```

    If the auth token is not available locally, create a `checkpoint:human-action` to get it. Otherwise proceed.

    **Alert Rule 1: "New Critical Error" (Issue Alert)**
    Create via Sentry API:
    ```
    POST /api/0/projects/bizscreen/javascript-react/rules/
    ```
    Configuration:
    - Name: "New Critical Error"
    - Conditions: A new issue is created, event level is error or fatal
    - Filters: The issue's category is equal to "error"
    - Actions: Send a Slack notification to the designated channel with the message "New critical error in BizScreen: {title}"
    - Action match: all (AND conditions)
    - Filter match: all
    - Frequency: 30 minutes (don't re-alert for same issue within 30 min)
    - Environment: production

    **Alert Rule 2: "Error Spike" (Metric Alert)**
    Create via Sentry API:
    ```
    POST /api/0/organizations/bizscreen/alert-rules/
    ```
    Configuration:
    - Name: "Error Spike"
    - Dataset: events (error events)
    - Query: is:unresolved
    - Aggregate: count()
    - Time window: 5 minutes
    - Threshold type: above
    - Triggers:
      - Critical: count > 10 in 5 minutes -> Slack notification "CRITICAL: Error spike in BizScreen ({value} errors in 5 min)"
      - Warning: count > 5 in 5 minutes -> Slack notification "WARNING: Elevated errors in BizScreen ({value} errors in 5 min)"
    - Resolve threshold: count < 3
    - Environment: production
    - Owner: Team or default

    NOTE: If the Sentry API does not support creating alert rules with Slack actions via token auth (some Sentry plans require specific permissions), fall back to providing the user with exact dashboard steps to create the rules manually. In that case, convert this task to checkpoint guidance.

    After creating, verify both rules exist:
    ```bash
    curl -s -H "Authorization: Bearer $SENTRY_AUTH_TOKEN" \
      "https://sentry.io/api/0/projects/bizscreen/javascript-react/rules/" | jq '.[].name'
    ```
  </action>
  <verify>
    Both alert rules visible in Sentry Dashboard -> Alerts -> Alert Rules:
    1. "New Critical Error" issue alert targeting Slack channel
    2. "Error Spike" metric alert targeting Slack channel
  </verify>
  <done>Two Sentry alert rules exist and are configured to send Slack notifications when error thresholds are exceeded.</done>
</task>
