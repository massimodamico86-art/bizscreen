**QRCodeWidget.jsx -- Fix import and add multi-type support:**

1. Add missing import at top: `import { QRCodeSVG } from 'qrcode.react';`

2. Add a `generateQRValue` helper function INSIDE the file (not exported) that generates the QR string based on `qrType`:
   - `'url'`: returns `props.url` directly
   - `'wifi'`: returns WiFi QR format string `WIFI:T:{encryption};S:{ssid};P:{password};H:{hidden ? 'true' : ''};;` with special characters escaped (backslash-escape `;`, `:`, `\`, `"`, `,` in ssid and password values)
   - `'text'`: returns `props.text` directly
   - Default: returns `props.url || props.text || ''`

3. Update the component to accept new props via destructuring from `props`:
   - `qrType` (default: `'url'`) -- the QR content type
   - `errorCorrection` (default: `'M'`) -- error correction level
   - `qrFgColor` (default: `'#000000'`) -- foreground color
   - `qrBgColor` (default: `'#ffffff'`) -- background color
   - Keep existing props: `url`, `label`, `cornerRadius`, `qrScale`, `textColor`
   - Add WiFi props (used when qrType is 'wifi'): `ssid`, `password`, `encryption` (default: 'WPA'), `hiddenNetwork` (default: false)
   - Add text prop: `text` (used when qrType is 'text')

4. Replace the hardcoded `url` usage in QRCodeSVG with `generateQRValue(props)`. The QR renders when the generated value is non-empty.

5. Update the label fallback logic: show `label`, or if empty show a truncated summary based on qrType:
   - URL: show truncated URL
   - WiFi: show SSID name
   - Text: show truncated text

**EditorCanvas.jsx -- Fix QRCodeSVG import:**

1. Add import at top of file: `import { QRCodeSVG } from 'qrcode.react';`
2. The inline QR preview case ('qr') already uses QRCodeSVG but it was never imported. No other changes needed to the preview logic for now -- it will continue to render the `url` prop as a basic preview. The canvas preview does not need to show all QR types since it is a simplified mock.

2. **QR Type Picker** (segmented control, 3 buttons):
   - URL | WiFi | Text (matching the 12h/24h segmented toggle pattern from clock controls)
   - Read `qrType` from props (default 'url')
   - On type change, call `onPropChange('qrType', newType)`
   - Active button: `bg-blue-600 text-white`, inactive: `bg-gray-700 text-gray-300`

3. **Type-specific content fields:**
   - **URL type**: Text input for `url` with placeholder "https://example.com". Below it, show a soft validation hint: if value is non-empty and doesn't match a basic URL pattern (`/^https?:\/\//`), show a small yellow text: "This doesn't look like a URL. QR will still work." using `<AlertTriangle className="w-3 h-3 inline" />` icon.
   - **WiFi type**: Three fields:
     - SSID text input (required feel, placeholder "Network name")
     - Password input with type toggle (eye/eye-off button to show/hide). Default type="password". Use `Eye` and `EyeOff` from lucide-react.
     - Encryption segmented control: WPA/WPA2 | Open (two buttons). Maps to `encryption` prop values 'WPA' and 'nopass'.
     - Hidden network checkbox: `<input type="checkbox" />` with label "Hidden network"
     - Below fields: human-readable summary in gray-500 text, e.g., "WiFi: MyNetwork (WPA)" or "WiFi: Open Network"
   - **Text type**: Textarea for `text` prop with placeholder "Enter any text..." and a character count display.

4. **Error Correction Level** selector:
   - Four buttons in a row: L | M | Q | H
   - Same segmented style as type picker
   - Label: "Error Correction" with small subtitle text "(Higher = more reliable scanning)"

5. **Color Pickers** (foreground and background):
