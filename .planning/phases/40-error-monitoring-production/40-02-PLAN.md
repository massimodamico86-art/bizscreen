  <action>The following configuration must be done in external dashboards. Claude cannot access these UIs.

**Step 1: Add GitHub Secrets (for CI source map uploads)**
Go to: GitHub -> Repository Settings -> Secrets and variables -> Actions
Add these three repository secrets:
- `SENTRY_AUTH_TOKEN` -- Create at Sentry Dashboard -> Settings -> Auth Tokens -> Create New Token. Required scopes: `project:releases` and `org:ci`
- `SENTRY_ORG` -- Your Sentry organization slug (from Sentry Dashboard -> Settings -> General Settings)
- `SENTRY_PROJECT` -- Your Sentry project slug (from Sentry Dashboard -> Settings -> Projects)

**Step 2: Connect Sentry Slack Integration**
Go to: Sentry Dashboard -> Settings -> Integrations -> Slack
- Click "Install" or "Configure" if already installed
- Authorize Sentry to post to your Slack workspace
- Select or create a channel for error alerts (suggested: #alerts or #errors)

**Step 3: Create Issue Alert -- "New Critical Error"**
Go to: Sentry Dashboard -> Alerts -> Create Alert -> Issues
- **When:** A new issue is created
- **If:** The issue's level is `error` or `fatal`
- **Then:** Send a Slack notification to your alerts channel
- **Action interval:** 5 minutes (prevents re-notification for same issue within 5 min)
- **Name:** "New Critical Error"

**Step 4: Create Metric Alert -- "Error Spike"**
Go to: Sentry Dashboard -> Alerts -> Create Alert -> Number of Errors
- **Time window:** 5 minutes
- **Warning threshold:** > 5 errors in 5 minutes
- **Critical threshold:** > 20 errors in 5 minutes
- **Actions:** Send Slack notification + email on Critical, Slack only on Warning
- **Name:** "Error Spike"

**Step 5: Configure Fingerprint Rules (optional, for alert fatigue prevention)**
Go to: Sentry Dashboard -> Project Settings -> Issue Grouping -> Fingerprint Rules
Add:
```
# Group Supabase connection errors together
error.type:SupabaseApiError error.value:"*connection*" -> supabase-connection-error
```
