<objective>
Wire payment method update so users can update their credit card directly from the subscription/billing page.

Purpose: Currently the AccountPlanPage has a "Manage Billing" button that opens the Stripe portal to the general dashboard. Users need a direct path to update their payment method without navigating the full portal. The Stripe Customer Portal supports flow_data to deep-link to payment method update.

Output: Updated billingService.js with `openPaymentMethodUpdate` function, AccountPlanPage with dedicated "Update Payment Method" button in the billing information card.
</objective>

  <files>src/services/billingService.js, src/pages/AccountPlanPage.jsx</files>
  <action>
    **billingService.js:**
    Add a new exported async function `openPaymentMethodUpdate()` that creates a Stripe Billing Portal session specifically for payment method update. It should:
    1. Get auth token via existing `getAuthToken()` helper
    2. POST to `/api/billing/portal` with body `{ flow_data: { type: 'payment_method_update' } }`
    3. Follow the same pattern as `openBillingPortal()` — check response, extract URL, redirect via `window.location.href`
    4. Throw meaningful errors on failure

    The Stripe Customer Portal API supports `flow_data` to direct users to a specific flow. The `payment_method_update` type opens the portal directly to the update payment method screen. Our existing `/api/billing/portal` endpoint should pass through `flow_data` if provided in the request body — if it doesn't, the portal will still open (just to the general dashboard), which is acceptable as a fallback. The key point is the frontend sends the correct request shape.

    **AccountPlanPage.jsx:**
    In the "Billing Information" card section (around line 380-408), update the `uiState.canManageBilling` branch to show TWO buttons instead of one:
    1. Keep existing "Manage Billing" button (calls `handleManageBilling` / `openBillingPortal`)
    2. Add a new "Update Payment Method" button with CreditCard icon that calls a new `handleUpdatePaymentMethod` async function

    The `handleUpdatePaymentMethod` function should:
    1. Set a new `updatingPayment` loading state to true
    2. Call `openPaymentMethodUpdate()` from billingService
    3. On error: log via logger, show toast, reset loading state
    4. Follow exact same pattern as `handleManageBilling`

    Add `updatingPayment` to the useState declarations (default false).
    Import `openPaymentMethodUpdate` alongside the existing billingService imports.

    Layout: Change the billing card from a single-row flex layout to show the description text on one line and both buttons below it. Use a flex row with gap-2 for the two buttons. The "Update Payment Method" button should be the primary action (variant default), and "Manage Billing" should be secondary (variant="secondary").

    Also update the description text from "Update payment method, view invoices, or cancel your subscription." to "Manage your payment method and subscription details." since the update payment action is now a dedicated button.
